Claude Code项目结构
这张图基本把ClaudeCode的项目工程化玩法讲清楚了：它不是单纯聊天💬，而是把AI🤖嵌进一个可落地的工作区里🧰✨。

左侧的目录树建议从my_project/开始📁：

-用CLAUDE.md做项目说明书/工作约定📝，把技术栈、目录规范、提交习惯、安全要求写清楚✅；
-.claude/里放settings.json、本地覆盖配置和commands/（比如review、deploy、test`这类一键指令）⚙️🚀，让常用流程变成可复用的操作♻️；
-skills/则像插件能力包🧩，按场景拆分成code-review、text-writer、security-audit、refactor等🔧，每个技能用SKILL.md描述输入输出和规则📌。

中间这列还强调🔍：
Hooks、MCP、Subagents、Plugins是扩展能力的关键🔑。
-Hook负责在“写入前/写入后/提交前”等时机自动触发检查⏱️✅；
-MCP用来连外部工具与服务（GitHub、Postgres、Playwright、Filesystem等）🔌🌐；
-多Agent并行能把“查问题—改代码—写测试—出报告”拆成流水线🏭➡️📄。

右侧给了最佳实践⭐：小步迭代🪜、技能文档要清晰📚、复杂任务模块化🧱、密钥用环境变量🔐、定期测试与审计🧪🛡️。

照着这个结构搭起来，你会发现ClaudeCode更像一个“可控、可追踪、可复用”的本地协作工程师👨‍💻🤝。
