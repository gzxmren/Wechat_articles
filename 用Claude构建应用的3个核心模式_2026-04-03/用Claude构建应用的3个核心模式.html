用Claude构建应用的3个核心模式

Anthropic联创Chris Olah说过一句话：生成式AI系统更像是在"培育"，而非"构建"。

研究员设定条件引导成长，但最终涌现的结构和能力并不总是可预测的。这给构建Claude应用带来了一个挑战：agent harness编码了对Claude能力的假设，但这些假设会随着Claude变强而过时。

Anthropic官方分享了三个模式，帮助团队跟上Claude的进化，同时平衡延迟和成本。

01 用Claude已经懂的工具

建议用Claude理解深刻的工具构建应用。

2024年底，Claude 3.5 Sonnet在SWE-bench Verified上达到49%——当时的最优成绩——只用了两个工具：bash工具和文本编辑器工具。Claude Code就基于这两个工具构建。

Bash不是为agent设计的，但Claude知道怎么用它，而且随着时间推移越用越好。

Claude用这些通用工具组合出不同模式：


  • Agent Skills
    ：技能扩展

  • Programmatic tool calling
    ：代码驱动的工具调用

  • Memory tool
    ：持久化记忆

这些全是从bash和文本编辑器组合出来的。

02 问"什么可以停止做"

Agent harness编码了Claude"做不到什么"的假设。随着Claude变强，这些假设需要不断测试。

让Claude自己编排动作

一个常见假设：每个工具结果都要经过Claude的上下文窗口，才能指导下一个动作。

问题：处理工具结果很慢、很贵、往往没必要。如果结果只需要传给下一个工具，或者Claude只关心其中一小部分，那让它经过上下文就是浪费。

给Claude一个代码执行工具（bash或REPL），让它自己决定哪些结果传递、过滤、直接管道到下一个调用。只有代码执行的输出才进入上下文。

效果：在BrowseComp评测上，让Opus 4.6自己过滤工具输出，准确率从45.3%提升到61.6%。

让Claude自己管理上下文

常见假设：系统提示应该手动精心设计，包含任务特定指令。

问题：预加载指令不扩展。每个token消耗Claude的注意力预算，预加载很少使用的指令是浪费。

Skills方案：每个skill的YAML前置信息是一个简短描述，预加载到上下文。完整的skill可以由Claude按需调用读取工具渐进式加载。

让Claude自己持久化上下文

长期运行的agent可能超过单个上下文窗口的限制。常见假设：记忆系统应该依赖模型外部的检索基础设施。

Anthropic的做法：给Claude简单的方式自己选择持久化什么内容。


  • Compaction
    ：让Claude总结过去上下文，保持长期任务的连续性

  • Memory folder
    ：让Claude把上下文写到文件，需要时再读

一个有趣的案例
：在宝可梦游戏中，Sonnet 3.5把记忆当转录，写下NPC说的每句话。Opus 4.6则写出战术笔记，从自己的失败中提炼教训。 

03 谨慎设置边界

Agent harness提供结构，强制UX、成本或安全边界。

设计上下文最大化缓存命中

Messages API是无状态的。Claude看不到之前轮次的对话历史。Agent harness需要在每轮把新上下文和所有过去动作、工具描述、指令一起打包给Claude。

提示可以根据设置的断点缓存。缓存token的成本只有基础输入token的10%。

几条原则：

原则
说明
静态优先
稳定内容（系统提示、工具）放前面
用消息更新
追加<system-reminder>而非编辑提示
别换模型
缓存是模型特定的，切换会打断
动态工具
用声明式工具设置边界

Claude不一定知道应用的安全边界或UX界面。它发出工具调用，由harness处理。

Bash工具给Claude广泛的编程能力，但只给harness一个命令字符串——每个动作都是同样的形状。

把动作提升为专用工具，给harness一个动作特定的钩子，带类型化参数，可以拦截、门控、渲染、审计。


  • 安全边界
    ：外部API调用等难逆操作，可以用用户确认门控

  • UX展示
    ：工具可以渲染为模态框，向用户清晰展示问题

  • 可观测性
    ：类型化工具给harness结构化参数，可以记录、追踪、回放

04 一个反复出现的模式

在Anthropic为长期任务构建的agent中，Sonnet 4.5会在感知到上下文限制接近时提前收尾——所谓的"上下文焦虑"。

他们添加了重置机制来清除上下文窗口。但到了Opus 4.5，这个行为消失了。之前为补偿而建的重置机制变成了harness里的死代码。

随着Claude变强，定期问自己：什么可以停止做？


 因为那些为补偿Claude弱点而加的结构，可能正在拖慢它的表现。
