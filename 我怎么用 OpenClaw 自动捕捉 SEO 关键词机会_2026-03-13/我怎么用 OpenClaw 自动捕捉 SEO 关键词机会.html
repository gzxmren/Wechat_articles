我做独立产品的 SEO 已经快一年了。

最开始那段时间，关键词研究全靠手动：打开 Google Search Console，导出数据，再去 Ahrefs 或者 Semrush 里查竞争度，然后整理成表格……一套流程下来，半天就没了。

问题不是麻烦，而是频率。

关键词机会是动态的——一个新工具出来了，大家都在搜相关词；某个话题突然火了，长尾词冒出来一批；竞争对手刚写了一篇文章，你还没发现。等你下周做关键词研究，机会早就被别人占了。

后来我在 OpenClaw 里配了一个专门做关键词捕捉的 Agent，叫它"Keyword Scout"。现在的流程是：它帮我持续监控关键词动态，发现新机会就告诉我，我只需要做判断，不需要做发现。

OpenClaw 自带 web_search（接 Brave Search API）、memory 系统和 Cron 调度器，这三个工具组合在一起，刚好能把关键词研究这件事完整自动化。这篇文章把怎么搭这个系统讲清楚。

底层逻辑：关键词机会从哪里来？

在讲怎么搭系统之前，先说清楚一件事：关键词机会是什么？

不是你已经排名的词，也不是竞争激烈的大词，而是这三类：

1. 新兴词：某个话题刚开始热，竞争还很小，但搜索量在涨

2. 竞争对手的漏网词：他们在某个关键词上有流量，但内容不够好，你可以写更好的

3. 你的相关词：你的产品 / 内容已经在某些词上有曝光，但排名不稳，需要加强

这三类机会，传统的做法要靠付费工具 + 定期手动检查。但如果有 AI 帮你盯着，频率可以提高到每天，发现效率大幅提升。

去 brave.com/search/api/ 注册，选 Data for Search 套餐，生成 API Key。注意不要选 "Data for AI"，那个不兼容 web_search 工具。

第一步：告诉 OpenClaw 你在做什么

Keyword Scout 能给出有针对性的建议，前提是 AI 知道你的产品背景。如果每次都要在对话里重新解释一遍，效率会很低。

更好的做法是写一个产品背景文件，放在 OpenClaw 的 memory 目录里，让 Agent 每次都能读到。

新建 memory/product-context.md：

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(line## 我的产品
**一句话介绍：** （你的产品是什么，解决什么问题）
**核心用户：** （主要面向哪类人）
**主要竞品：** （列出 3-5 个直接竞品）
**我们的差异点：**- （比竞品 A 强在哪里）- （比竞品 B 强在哪里）
**SEO 重点领域：**- （你想在哪些关键词上占位，列出大方向）
**已有内容：** （定期更新已写的关键词页面列表）

把这个文件填好。以后 cron prompt 里加一行 读取 memory/product-context.md 了解产品背景，AI 的所有建议都会基于这个上下文，不会给你乱出题。

真实效果：一次完整的发现过程

说一个具体案例。

大概两个月前，OpenClaw 的周报里出现了这样一条：

新发现：某个长尾词竞争极低搜索结果里前三名都是泛泛而谈的内容，没有专项教程。搜索量信号显示这个词近期有增长趋势，建议优先写。

这个词我之前完全没想到。

但 OpenClaw 在扫描竞品替代词的时候，看到有个评测文章提到了这个细分场景，然后顺着搜了一下，发现是空白。

按这个建议写了一篇针对性的文章，两周后开始有排名，现在稳定在 Google 第二页，还在涨。

这不是偶然，是系统性发现的结果。

进阶：让 OpenClaw 做竞品内容分析

关键词机会找到了，还有一步很有价值：分析竞品在这个词上写了什么，找出你能做得更好的角度。

在 OpenClaw 里运行这个 prompt：

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(line读取 memory/product-context.md。
帮我分析 [竞品名] 在关键词 [目标词] 上的内容策略：
1. 搜索 'site:[竞品域名] [目标词]'，看他们写了哪些页面
2. 从搜索摘要里，分析他们内容的侧重点（功能介绍？教程？对比？）
3. 找出他们内容没有覆盖的角度，这些角度可能是我们的差异化机会
整理成一份竞品内容地图。

这种分析用付费工具做要 30 分钟，OpenClaw 3 分钟就能给出初稿。精度比不上专业 SEO 工具，但对于发现方向性机会，完全够用。

把关键词机会转化成内容计划

发现机会是第一步，真正有用的是把它转化成行动。

每周让 OpenClaw 根据追踪档案生成内容建议：

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(line读取 memory/product-context.md 和 memory/keyword-tracker.md。
根据当前的机会评级和竞争格局，推荐本周最值得写的 3 个关键词，每个附上：- 建议的文章标题- 核心角度（和竞品的差异化点）- 预估写作难度（1-5分）
保存到 memory/content-suggestions-[日期].md

有了这份每周建议，内容计划就从"我不知道写什么"变成了"我从这几个里选一个"。

一个意外的发现：中文关键词机会

有一次扫描出了一个有趣的结果：

OpenClaw 在搜索英文关键词的时候，发现一篇中文内容排进了 Google 英文搜索前十。这说明这个关键词领域竞争极低，连中文页面都能占到排名。

这类"英文市场竞争极低"的词，对于我们这种有中文背景的团队是额外的机会——写出来的内容在质量上有自然优势。

这类洞察不是我主动想到的，是 OpenClaw 在执行任务过程中顺带发现的，属于"计划外收获"——这也是自动化监控比手动搜索有优势的地方。

这套系统最核心的价值，不是省时间，而是提高了发现频率。

每周手动做一次关键词研究，发现的是上周的机会。每天 OpenClaw 自动扫描，发现的是今天的机会。时效性完全不同。

几个关键设计原则回顾：

  • 产品背景沉淀在 memory：写好 product-context.md，AI 每次都有上下文，建议质量更高
  • 没有变化就不发消息：避免通知疲劳，只在有实际发现时打扰你
  • 追踪档案持续迭代：不只是发现，还要记录、对比、看趋势
  • 计划外收获才是亮点：AI 在执行任务过程中顺带发现的意外机会，是纯手动做不到的
  • 转化成行动：每周建议落到具体文章标题，不让机会停在"记录"阶段

整套系统搭下来大概需要一个小时：配好 Brave API、写好产品背景文件、设置两个 cron 任务。之后就是持续接收 OpenClaw 的发现，做选题决策。
