023步搞定环境，5分钟跑起来

先说环境搭建。我把踩过的坑都给你填平了，跟着做就行。

📍 第一步：安装OpenClaw

如果你还没装OpenClaw，先去官网装好。官网文档写得挺清楚，这里不赘述。

📍 第二步：安装acpx插件（关键！）

这是OpenClaw和Claude Code/Codex之间的"通讯器"，必须装。打开终端，依次执行这几条命令：

openclaw plugins install acpx

openclaw config set plugins.entries.acpx.enabled true

openclaw config set acp.enabled true

openclaw config set acp.backend acpx

openclaw config set acp.allowedAgents '["claude","codex","gemini","pi"]'

openclaw gateway restart

📍 第三步：配置API Key

Claude Code需要Anthropic的API Key，Codex需要OpenAI的API Key。设置成环境变量：

export ANTHROPIC_API_KEY=你的AnthropicKey

export OPENAI_API_KEY=你的OpenAIKey

03实测：让AI帮我写代码

环境配好了，现在到了最爽的部分——指挥AI干活。

场景1：一次性任务（快速搞定）

比如你想让Claude Code帮你重构一段代码。这种"干完就走"的任务，用oneshot模式：

/acp spawn claude --mode oneshot --thread off

然后在聊天框里直接描述需求，Claude Code会在后台运行，做完之后把结果发给你。

场景2：持续会话（深度协作）

如果你要做一个复杂的项目，需要多轮沟通、反复修改，那就用persistent模式：

/acp spawn claude --mode persistent --thread auto

这个命令会创建一个独立的会话线程。之后你说的每一句话，都会发给同一个Claude Code实例。它会记住上下文，就像你在和一个真正的程序员结对编程。

场景3：最省心的方式（推荐）

其实上面两种方法都有点"技术宅"，需要记命令。最简单的方式是直接说话："用claude code帮我写一个爬虫，抓取知乎热榜标题"。OpenClaw会自动识别你要调用Claude Code，然后帮你启动ACP会话。

04Claude Code vs Codex，怎么选？

很多主公会问：Claude Code和Codex都是AI编程助手，我该用哪个？我两个都测了，说下我的感受：

维度Claude CodeCodex
代码质量非常高，擅长架构设计也很高，偏向快速实现
上下文理解200K tokens，大项目也能hold住上下文稍短，适合中小项目
交互体验像和一个资深工程师对话像和一个高效执行者对话
价格Anthropic API按量付费OpenAI API按量付费

我的建议：两个都装上，看心情换着用。做架构设计、重构老代码，用Claude Code；写脚本、快速验证想法，用Codex。切换也很简单，把命令里的claude换成codex就行。

06你可能会遇到的问题

Q1：API Key花钱吗？

花。Claude Code和Codex都需要调用API，按token计费。我粗略算了下，重度使用一个月大概100-200块钱。但相比于省下来的时间，这个钱值。

Q2：公司代码能放上去吗？

不建议。虽然Anthropic和OpenAI都承诺不用你的数据训练模型，但小心驶得万年船。敏感代码还是在本地跑，或者用企业版。

Q3：和Cursor比，哪个更好？

这俩不是一个赛道。Cursor是IDE插件，适合日常写代码的时候用。Claude Code是独立的AI助手，适合做复杂任务、自动化工作流。我两个都用。
