一个基于 OpenClaw 搭建的交易机器人，7 天时间，执行了超过 47,000 笔交易，覆盖 31 个市场，扣掉手续费之后净赚 11.5 万美金。

一、先搞懂 Polymarket 的钱从哪来

在拆 Bot 之前，你得先理解 Polymarket 这个市场的运作逻辑，不然后面的策略全看不懂。

Polymarket 是一个预测市场。你可以对各种事件下注：BTC 接下来 5 分钟会涨还是跌？下个月美联储会不会降息？某某公司这季度财报会不会超预期？

核心机制很简单：每个事件有 YES 和 NO 两个代币。最终结果出来后，正确的那一方每个代币结算为 $1，错的那方归零。

比如一个"BTC 5 分钟内会涨"的市场：

  • YES 代币当前价格 $0.55（市场认为有 55% 概率会涨）
  • NO 代币当前价格 $0.47（市场认为有 47% 概率会跌）

注意：0.47 = 1.00。

这个"多出来的 2 分钱"，就是做市商和套利机器人的利润来源。

PANews 扒了 Polymarket 上两个大户的数据，数字很惊人：

  • 账户 k9Q2mX4L8A7ZP3R：累计盈利 $104.6 万，参与了 24,000 次交易
  • 账户 0x8dxd：累计盈利 $168 万，资金曲线几乎是一条完美的斜线往上走

这两个账户的共同点是：它们不预测结果，它们赚的是价差。

二、那个 OpenClaw Bot 到底在干什么

回到那个一周赚 $115K 的 Bot。它用的核心策略叫自动化做市 + 套利，拆开来讲：

2.1 做市：两边挂单，吃价差

做市的逻辑跟路边的黄牛差不多——低买高卖，两头赚差价。

Bot 同时在 YES 和 NO 两边挂买单。比如 YES 当前中间价 0.53 挂买单，在 0.04 差价。

一笔赚几美分，但它一天跑几千笔。 7 天跑了 47,000 笔，平均每笔赚大约 $2.4。积少成多。

2.2 套利：YES + NO < $1 的时候两边都买

这是更稳的赚法。

正常情况下，YES + NO 应该等于 0.47 + NO 0.95。

这时候 Bot 两边都买进去，花 1.00。

稳赚 $0.05，无关预测结果。

这就是为什么 cyberk.io 那篇分析文章说的：AI 代理不需要预测未来，只需要对非理性行为做出理性反应。

2.3 延迟套利：Binance 先动，Polymarket 后动

还有一种更高级的玩法。开发者 Chudi 写了篇详细的技术博客，专门拆这个策略。

逻辑是这样的：BTC 在 Binance 上暴涨了 0.3%，但 Polymarket 上"BTC 5 分钟涨跌"这个市场的做市商还没来得及调价。中间有 30-90 秒的时间差。

在这个窗口里，Bot 以旧价格买入正确方向的代币，等市场反应过来，价格调整后获利。

Chudi 自己的实盘数据（2025 年 2 月，小资金测试期）：

  • 23 笔交易，胜率 69.6%
  • 每笔投入 15
  • 净赚 $11.51

数字不大，因为他在用小钱验证策略。但胜率接近 70%，放大资金后的潜力很可观。

为什么推荐阿姆斯特丹？因为 Polymarket 的 CLOB（中央限价订单簿）服务器在伦敦，从阿姆斯特丹过去延迟只有 5-12ms。从美国东部打过去要 130-150ms。在延迟套利里，这个差距就是真金白银。

3.2 Polymarket 官方 SDK：py-clob-client

Polymarket 官方提供了 Python SDK，叫 py-clob-client。装好之后就能用代码下单。

pip install py-clob-client python-dotenv web3

初始化客户端（关键：signature_type 一定要设对）：

from py_clob_client.client import ClobClient

client = ClobClient(
    host="https://clob.polymarket.com",
    key=你的私钥,
    chain_id=137,         # Polygon 主网
    signature_type=1,     # MagicLink 账户用 1，EOA 钱包用 0
)

这里有个巨坑：SDK 里有两个函数 create_order 和 create_and_post_order。前者只在本地创建订单对象，不会提交到交易所。你必须用 create_and_post_order 才能真正下单。很多人第一次跑 Bot 发现"怎么一笔都没成交"，就是踩的这个坑。

3.3 信号层：从 Binance 抓实时价格

如果你做延迟套利策略，需要一个信号源。最常用的是 Binance WebSocket，实时拿 BTC 价格：

import asyncio, websockets, json
from collections import deque

THRESHOLD = 0.003   # 0.3% 波动触发
WINDOW = 60         # 60秒窗口

price_window = deque()

asyncdefmonitor():
    url = "wss://stream.binance.com:9443/ws/btcusdt@aggTrade"
    asyncwith websockets.connect(url) as ws:
        asyncfor msg in ws:
            data = json.loads(msg)
            price = float(data["p"])
            ts = data["T"] / 1000
            price_window.append((ts, price))

            # 清理60秒前的数据
            while price_window and price_window[0][0] < ts - WINDOW:
                price_window.popleft()

            iflen(price_window) < 2:
                continue

            pct = (price - price_window[0][1]) / price_window[0][1]

            ifabs(pct) >= THRESHOLD:
                direction = "UP"if pct > 0else"DOWN"
                # 触发交易信号
                await execute_trade(direction, price)

检测到 BTC 在 60 秒内波动超过 0.3%，就触发交易信号。

3.4 执行层：下单

拿到信号后，在 Polymarket 上买入对应方向的代币：

from py_clob_client.clob_types import OrderArgs

asyncdefexecute_trade(direction, price):
    # BTC 涨 → 买 YES 代币
    # BTC 跌 → 买 NO 代币
    token_id = YES_TOKEN_ID if direction == "UP"else NO_TOKEN_ID

    order = client.create_and_post_order(
        OrderArgs(
            token_id=token_id,
            price=0.55,    # 你愿意支付的最高价
            size=20,       # 下注金额
            side="BUY",
        )
    )
    print(f"下单成功: {order}")

又一个坑：side 永远填 BUY。在 Polymarket 里，YES 和 NO 是两个独立代币。你要做空某个方向，不是"卖出 YES"，而是"买入 NO"。

3.5 风控：别让 Bot 替你亏光

不管什么策略，风控是生死线。几条铁律：

  1. 单笔不超过总资金的 10%。Chudi 测试期每笔 15，本金大概 $200 左右
  2. 设止损线。亏损超过 30% 且市场剩余时间 > 3 分钟，直接平仓
  3. 不要调用 update_balance_allowance()。这个函数会读链上数据覆盖 CLOB 内部额度，如果刚成交完链上还没同步，会导致后续卖单全部失败
  4. 用 SQLite 记录每一笔交易。出了问题才能查是策略问题还是执行问题

  • 方向性押注：高风险高回报，跟赌博没本质区别

六、必须知道的风险

在你开搞之前，这几个风险不看完别动手。

6.1 Skill 安全问题

OpenClaw 的 Skill 是开源的，谁都能上传。安全公司 Snyk 在 BankrBot 生态里扫出了 386 个恶意 Skill——有些会偷你的私钥，有些会把你的交易重定向到别人的钱包。

装任何交易 Skill 之前，先跑一遍安全扫描：

# 检查有没有可疑的外部请求
grep -rn "curl\|wget\|fetch\|http://" scripts/

# 检查有没有读私钥
grep -rn "\.env\|private_key\|secret\|wallet" scripts/

# 检查有没有 base64 混淆
grep -rn "base64\|atob\|btoa" scripts/

或者直接用 Skill Guard 做自动扫描（npx clawhub@latest install benlee/skillguard）。

6.2 智能合约风险

Polymarket 跑在 Polygon 链上。智能合约有漏洞就可能被黑，资金直接清零。这种事在 DeFi 领域不是新闻。

6.3 监管风险

Polymarket 目前对美国用户有限制。其他地区的法律地位也在变化中。如果某天平台被要求关闭或冻结资金，你的钱可能取不出来。

6.4 竞争加剧

延迟套利的窗口从最初的 30-90 秒，正在被越来越多的 Bot 压缩。当足够多的人用同样的策略，利润空间会被挤到接近零。先到的人吃肉，后到的人喝汤，再后面的连汤都没有。

写在最后

那个一周赚 $115K 的 Bot，做的事情归结起来就是三件：盯着价差、两边挂单、比人快。

不神秘，但也不容易。赚钱的不是 OpenClaw 本身，是你基于它搭建的策略、你对市场微观结构的理解、以及你对风控的严格程度。

如果你有代码基础，$200 就能开始跑延迟套利的小规模测试。如果你不想碰代码，PredictEngine 的模拟模式零成本试错。

不管哪条路，先用小钱验证，再决定要不要加码。
