从每次重复到一次设置终身受益：Claude Skills详解

 

Claude Skills：让 AI 记住你的工作方式

"每次都要给 AI 重复同样的指令，真让人头疼……"
"AI 总是记不住公司的规则和格式……"
"团队里每个人用 AI 的方式都不一样，最后只有擅长用的人才能受益……"

Anthropic 刚刚升级了一项强大的功能，可以解决这些问题。它叫"Claude Skills"。

这不只是 AI 代理的一次更新。这是一个下一代功能，让你能把业务流程和专业知识教给 AI，让它成长为完全契合你公司需求的专家。

Claude Skills 是教会 Claude 执行特定任务和工作流的最强大功能。使用它最让我兴奋的是，你再也不用在每次对话中重复解释你的偏好和流程了。

一个 Skill 就是一套打包在简单文件夹里的指令，设置一次，终身受益。当你有固定的工作流时，它特别好用——比如根据规格生成前端设计，或者按照团队风格指南创建文档。

根据我的经验，Skills 不只是"宏"或"模板"，它更像一个"知识库"，能增强 Claude 的决策能力。它与代码执行、文档创建等内置功能配合得尤其好，让 Claude 能无缝处理复杂任务。

什么是 Skills？

Skills 是 Claude Code 可以参考的可复用知识片段和流程，用于执行特定任务。每个 Skill 主要由一个 Markdown 文件（SKILL.md）定义，可以根据需要包含相关脚本和资源。

Claude Code 会根据用户请求加载相应的 Skill，并按照指令执行任务。这让你能够一致地自动化复杂工作流和常规任务。

Claude Code 本身充当一个参考知识库，可以直接执行脚本和管理流程，所以你可以用规则系统定义并执行"在什么时候做什么"。

什么是 Skill-creator？

Skill-creator 是一个"元技能"，让你可以一次性创建、测试和改进技能。

简单来说，Skill-creator 做以下五件事：

  • • 询问"你想开发什么样的技能？"
  • • 自动生成 SKILL.md 草稿
  • • 用测试提示词实际运行测试
  • • 评估结果并提出改进建议
  • • 重复步骤 2 到 4，直到你满意为止

核心思路是让 Claude Code 自己经历手动编写 SKILL.md、尝试、修复的循环。

如何编写 SKILL.md？

基本上，你只需要在 .claude/skills 下创建一个文件夹，然后为技能创建一个 SKILL.md 文件。

SKILL.md 的内容如下，用 YAML 和 Markdown 编写：

---
name: Your Skill Name
description: Brief description of what this Skill does and when to use it
---

# Your Skill Name
## Instructions
Provide clear, step-by-step guidance for Claude.
## Examples
Show concrete examples of using this Skill.

首先解释 YAML 部分。这叫元数据（metadata），是创建技能时非常重要的一部分。

Claude 在启动时读取元数据，从而知道每个技能何时存在、何时可用，并将其纳入系统提示。这种方式让你可以拥有大量技能，而不会不必要地撑爆上下文。

当提示词或请求与技能的元数据匹配时，Claude 会从文件系统读取 SKILL.md。实际是否执行的准确性很大程度上取决于元数据的内容，所以这是一个非常重要的因素。

接下来解释内容部分。元数据在 Claude 启动时总是加载，但内容部分在运行时加载。然后，当代理技能被执行时，Claude 会处理内容部分。

官方最佳实践建议将 SKILL.md 控制在 500 行以内。如果超过，就把详细参考材料拆分到单独文件：

.claude/skills/
 my-skill/
 SKILL.md ← 主要指令（500 行以内）
 templates/ ← 模板文件
 reference.md ← 详细参考材料

用 SKILL.md 中的指令引导 Claude 只在需要时加载额外文件，按需展开而非一次性加载所有内容。这就是渐进式披露（Progressive Disclosure）：先提供核心指令，按需展开详情。

关键在于，即使 Agent Skills 的加载机制已经很高效，内容部分也最好保持简洁——当 Claude 加载内容部分时，它会与对话历史和其他上下文竞争。

因此，在内容部分应省略对系统提示、编程语言、库等的通用参考。创建高精度技能的技巧之一是确定省略哪些部分、从哪里开始。

Skills 的工作原理

Skill 就是一个包含命令的文件夹

Skill 的核心是一个包含 SKILL.md 文件的文件夹。这个 Markdown 文件用 YAML Front Matter 定义元数据（如名称和描述），主体部分包含清晰的分步任务指令和示例。

Claude 会自动发现和加载相关技能

不需要手动触发技能。会话开始时，Claude 会扫描所有已安装技能的元数据（名称和描述），并将这些简短信息加载到系统提示中。

当你的请求与某个技能的描述匹配时，Claude 会自动读取并加载该技能的完整指令。

"渐进式披露"机制让 Skills 极其高效

Skill 使用三层结构（YAML 前言、主体和文件引用）逐步按需将信息输入模型上下文，避免一次性过载，提高效率和 token 经济性。

Skills 的设计充分考虑了 token 效率。初始加载时，每个 Skill 只用几十个 token 存储元数据。技能的详细指令只有在被触发时才显示在上下文窗口中。

这种按需加载机制意味着你可以安装大量 Skills，而不会因为上下文窗口满载而影响模型性能。

对于更复杂的 Skills，不同指令可以拆分到多个文件，Claude 只读取当前任务需要的部分，进一步节省 token。

MCP 与 Skills

对于已经在使用 MCP（Model Context Protocol）的人来说，Skills 是另一个强大的层级。我觉得用厨房和食谱的类比来理解它们的关系最好。

MCP 提供专业厨房，让你能访问工具、食材和设备。而 Skills 是食谱，提供逐步指令来创造有价值的东西。

把两者结合起来，用户就能完成复杂任务，而不必自己搞清楚所有步骤。我最初构建 MCP 服务器时，以为只要提供工具访问就够了，但实际上缺少了如何使用工具的工作流指导，这让用户感到困惑。

引入 Skills 后，形成了清晰的分工：MCP 定义能做什么，Skills 教你怎么做，用户体验大幅提升。

安装这个 Skill

Skills 实际上是一种插件形式。我们通过市场中的插件来安装 Skills，Claude 会在需要时自动加载。

你可以输入 /plugin，然后输入官方 GitHub Skills 地址来添加插件市场：
https://github.com/anthropics/skills

你也可以用以下命令快速安装 Skills：

/plugin install document-skills@anthropic-agent-skills
/plugin install example-skills@anthropic-agent-skills

这两个技能插件的官方用途如下：

  • • document-skills：文档技能包，可处理 Excel、Word、PPT、PDF 等文档
  • • example-skills：示例技能集，可处理技能创建、MCP 构建、视觉设计、算法艺术、Web 测试、Slack GIF 创建、主题样式等

正文应该只包含 Claude 不知道的东西

Skill-Creator 指南这样说：

默认假设：Claude 已经很聪明了。只添加 Claude 不知道的上下文。

基本前提是 Claude 本身就很智能，所以在 SKILL.md 中写通用知识或编程基础只会浪费 token。

你应该专注于它不知道的信息（公司特定规则、特定库的特性、领域专属工作流等）。

建议避免冗长解释，使用命令式简洁写作风格。

根据任务的"自由度"匹配指令

没必要把每件事都规定得特别详细，关键是根据任务调整指令的粒度。

  • • 高自由度（文字指令）：当多种方法都有效时，比如写作
  • • 中等自由度（伪代码或带参数的脚本）：有推荐模式，但允许一些变化
  • • 低自由度（具体脚本，少参数）：当流程一致性至关重要，错误代价高昂时

如何判断任务是否适合做成 Skill？

当你发现自己频繁向 Claude 请求同一类任务，或有需要重复使用的模板和资源时，比如：

"用公司模板帮我写周报"：你需要每周写团队周报，每次都要告诉 Claude 按"本周成果、遇到的困难、下一步计划"三部分组织内容。这时，你可以创建一个"团队周报生成器"技能。

"按我们公司的风格创建演示文稿"：经常需要严格遵守品牌指南，包括 Logo 使用、品牌色、公司名称、公司业务内容、专业期望等。你可以把这些指南打包成"品牌演示风格"技能。

"按特定格式整理市场分析报告/竞品调研"：比如创建市场分析报告可能需要组合三组竞品数据、一组内部销售数据，并应用固定分析框架。这整个复杂流程可以封装成"市场分析报告"技能。

相反，如果只是偶尔的一次性请求，直接在聊天中说明即可，没必要创建 Skill。

结语

Claude Skills 是经常执行重复性日常工作的人的必备利器。它把你"模糊的工作经验"转化为 AI 能理解的"明确规则"，让 Anthropic 的工具完美适配你的需求。

无论你是产品经理、项目经理、文案撰稿人，还是任何在职场上使用 Claude 的人，都可以依赖它减少重复工作、确保输出一致——这就是 Skills 的核心价值。
