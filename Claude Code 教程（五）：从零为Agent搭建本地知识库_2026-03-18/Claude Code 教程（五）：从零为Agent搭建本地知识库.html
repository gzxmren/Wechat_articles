Claude Code 教程（五）：从零为Agent搭建本地知识库


先点关注不迷路

一定要记得，知识库不是为自己搭建的，是为你的Agent搭建的。

你的研报，AI 记得住吗？

上一篇教程，我讲了怎么用 Claude Code 追踪投资假设。有读者问了一个很实际的问题：

"你说把财报数据丢给 Claude Code 就能更新假设，但我手上有几百份研报和纪要，难道每次都手动告诉它？"

说实话，这个问题我自己也踩过坑。

一开始我就是这么干的——每次需要什么信息，就把 PDF 拖进 Claude Code，让它读。结果踩了不少坑。

慢。一份 50 页的研报，Claude Code 读完要好几分钟，有时候还卡住。

贵。每次读 PDF 都要消耗大量 token，一份研报可能烧掉好几块钱。你有 200 份研报，算算这个成本。

最要命的——它记不住。你今天让它读了 GE Vernova 的财报，明天再问它相关问题，它已经忘了。每次对话都是重新开始，上次读的内容不会自动带过来。

这就好比你雇了一个助理，每天早上来上班都失忆了，你得从头给他讲一遍公司的业务。

解决办法？搭一个本地的研究知识库——让它每次打开就能找到之前的资料。

先看看搭完是什么效果。

搭完之后长什么样？

我的工作空间目前长这样（简化展示）：

workspace/                         # 整个研究工作空间
├── CLAUDE.md                      # 总指挥：研究方向、模块说明、使用规则
│
├── hypothesis/                    # 假设追踪（教程四搭的）
│   ├── 全球电力大周期.md
│   └── AI算力周期.md
│
├── library/                       # 知识库（今天要搭的）
│   ├── 公司研究/
│   │   ├── GE-Vernova/
│   │   │   ├── 2025Q4-财报摘要.md
│   │   │   ├── 2025Q3-财报摘要.md
│   │   │   └── 管理层发言-关键引用.md
│   │   ├── 西门子能源/
│   │   └── 三菱重工/
│   ├── 行业数据/
│   │   ├── 美国电力需求月度.md
│   │   ├── 燃气轮机订单季度.md
│   │   └── AI数据中心签约容量.md
│   └── 研报存档/
│       ├── 2025-03-GE-Vernova-Q4.md
│       └── ...
│
└── output/                        # 输出（研究笔记、报告等）

注意：知识库是 workspace 下面的一个子目录，不是独立存在的。CLAUDE.md 在根目录，管的是整个工作空间——假设追踪、知识库、输出，都归它管。

每次打开 Claude Code，它会先读 CLAUDE.md，立刻知道：你在研究什么方向、资料在哪、假设追踪到什么程度了。然后你问它任何问题，它都能去对应的地方找信息。

比如你问"GE Vernova 最近三个季度的订单趋势怎么样？"——它会直接去 公司研究/GE-Vernova/ 下面翻三份财报摘要，给你拉出来对比。

不用你重新喂数据，不用重新解释背景。它"记得"。

CLAUDE.md：工作空间的总指挥

工作空间最核心的文件是根目录的 CLAUDE.md。

Claude Code 每次启动都会自动读它。你可以理解为——这是你写给 AI 的一封信，告诉它"你是谁、你在做什么、东西放在哪、各个模块怎么配合"。

我的 CLAUDE.md 大概包含这些内容：

你的身份和研究方向

我是一个股票研究员，主要覆盖海外电力设备和 AI 算力方向。当前最重要的投资假设是"全球进入 10 年电力基建大周期"。

工作空间的模块说明

  • 假设追踪在 hypothesis/ 下面，每个假设一个文件
  • 知识库在 library/ 下面，按公司研究 / 行业数据 / 研报存档分类
  • 公司研究按公司名分文件夹，每家公司有财报摘要、管理层发言、研究备忘

使用规则

回答问题时，优先从知识库中找证据。如果知识库里没有相关信息，告诉我"知识库中暂无相关数据"，不要自己编。更新假设追踪时，引用知识库中的原始数据作为证据。

关键点：CLAUDE.md 是全局的，知识库只是它管理的一个模块。教程四的假设追踪也在它的管辖范围内。这样一来，AI 能把知识库的数据和假设追踪联动起来——新数据进了知识库，假设追踪的确定性评分就该更新。

怎么写这个文件？打开 Claude Code，跟它说：

"帮我写一个 CLAUDE.md，我是股票研究员，工作空间里有假设追踪和知识库两个模块，帮我把结构和使用规则都写清楚。"

它会帮你起草，你改改就能用。

研报怎么进知识库？

知识库搭好了，接下来的问题是：那堆 PDF 研报怎么放进去？

这其实是整个流程里最容易卡住的一步。很多人搭了目录结构，写了 CLAUDE.md，然后面对几百份 PDF 不知道怎么处理，就放弃了。

直接把 PDF 丢给 Claude Code 读，前面说了，又慢又贵。更好的做法是先把 PDF 转成 markdown 文本，再让 Claude Code 处理文本。文本比 PDF 快几十倍，也便宜得多。

转换工具：marker

PDF 转 markdown 最好用的开源工具叫 marker。它专门处理学术论文和研报类 PDF，表格、图表、公式都能转，效果比一般的 PDF 解析库好很多。也有其他轻量化的工具，大家可以都试试

安装很简单，让 Claude Code 帮你装：

"帮我安装 marker-pdf，用 pip install marker-pdf"

装好之后，转换一份研报就是一行命令：

"帮我用 marker 把这份 PDF 转成 markdown"

如果你还有 PPT 和 Word 格式的资料，可以再装一个 markitdown——微软出的，专门把 Office 文件转成 markdown。

我的处理流程

我自己搭了一个半自动的流程，分享一下思路：

第一步：把研报丢进收件箱

在知识库下面建一个 inbox/ 文件夹，平时看到有价值的研报就往里扔。不用管格式，PDF、PPT、Word 都行。

第二步：批量转换 + 生成摘要

写一个 Python 脚本，自动做两件事：

  1. 用 marker 把 PDF 转成 markdown
  2. 调 LLM 对转换后的文本生成结构化摘要（核心观点、关键数据、管理层发言）

这一步完全不需要 Claude Code 参与，本地跑就行。一份研报的转换+摘要大概几分钟。

你可以让 Claude Code 帮你写这个脚本：

"帮我写一个批量处理研报的脚本。扫描 library/inbox/ 下面的 PDF 文件，用 marker 转成 markdown，然后调 API 生成摘要，输出到 library/inbox/ready/ 下面。"

第三步：确认归档

脚本跑完，你看一眼摘要，确认没问题，就让 Claude Code 把它们分类存到对应的公司文件夹下面。

"library/inbox/ready/ 下面有处理好的研报摘要，帮我按公司分类归档到 library/公司研究/ 对应的文件夹里。"

我自己处理了 168 份 PDF，转出来 278 个 markdown 文件。整个过程跑了大概半小时，Claude Code 的 token 消耗不到以前的十分之一。

不想折腾？从边用边建开始

如果你觉得写脚本太重，完全可以跳过。最轻的方式就是边用边建：

每次读到一份新研报，顺手丢给 Claude Code：

"这份研报的关键数据帮我提取一下，更新到 library/公司研究/GE-Vernova/ 下面。"

日积月累，知识库就自然长出来了。别一开始就想着把所有东西都整理好，那样你很可能整理到一半就放弃了。

公司研究怎么组织？

知识库里最有价值的部分，是按公司整理的研究笔记。

我的做法是每家公司一个文件夹，下面放几类文件：

财报摘要：每个季度一份，只提取和我的假设相关的数据。

比如 GE Vernova 的 Q4 财报摘要，我只保留：

  • 燃气轮机订单量（35GW）
  • 业务利润率（12%）
  • 在手订单年限（排到 2029）
  • 管理层对未来的指引

其他不相关的信息（比如风电业务、具体的资产负债表），我不存。知识库不是数据库，不需要什么都有，只需要有你关心的东西。

管理层发言：业绩会上 CEO/CFO 说的关键原话。

原话很重要。不是让 AI 总结的"管理层对前景持乐观态度"，而是原话——"燃气轮机进入了一个 10 年的大周期"。这种原话的分量和一句总结完全不一样。

研究备忘：你自己的思考和判断。

"这个季度的订单超了我的预期，但利润率低于预期，可能是因为新订单的交付周期还没到。下个季度重点看利润率能不能上来。"

这些备忘是你最宝贵的资产。三个月后你回头看，就知道当时是怎么想的。

行业数据怎么追踪？

除了公司研究，知识库还需要追踪一些行业层面的关键数据。

我的做法很简单：一个指标一个文件，按时间顺序更新。

比如 行业数据/燃气轮机订单季度.md：

# 全球燃气轮机订单（季度）

| 时间 | GE | 西门子 | 三菱 | 合计 |
|------|-----|--------|------|------|
| 2024 Q1 | 8GW | 5GW | 3GW | 16GW |
| 2024 Q2 | 10GW | 6GW | 3GW | 19GW |
| 2024 Q3 | 12GW | 7GW | 4GW | 23GW |
| 2024 Q4 | 15GW | 8GW | 5GW | 28GW |
| 2025 Q1 | 18GW | 10GW | 7GW | 35GW |

数据来源：各公司财报

趋势：持续增长，2025Q1 同比+119%

每个季度财报出来，更新一行就行。Claude Code 看这个文件就能画趋势、做对比。

你可以让 Claude Code 帮你维护这些文件：

"GE Vernova 刚发了 Q1 数据，燃气轮机订单 18GW。帮我更新行业数据表。"

怎么用 Claude Code 搭这个系统？

和教程四一样，不用写什么模板，直接聊就行。

第一步：建目录结构

"帮我在当前工作空间下建一个 library/ 目录，用来存研究知识库。我主要研究海外电力设备（GE Vernova、西门子能源、三菱重工）和 AI 算力。需要公司研究、行业数据、研报存档三个大类。"

Claude Code 会帮你把文件夹全建好。

第二步：写 CLAUDE.md

在 workspace 根目录创建一个 CLAUDE.md，告诉 Claude Code 整个工作空间的结构：

"帮我写一个 CLAUDE.md，我是股票研究员，工作空间里有假设追踪和知识库两个模块，帮我把结构和使用规则都写清楚。"

如果你之前跟着教程四已经有一些文件了，让它把知识库的部分也加进去就行。

第三步：导入存量资料

如果你手上已经有一些研报和笔记，告诉它在哪：

"我在 ~/Downloads/研报/ 下面有一批 PDF 研报，帮我批量转成 markdown，按公司和日期分类存到知识库里。"

这步可能需要跑一会儿，取决于你有多少份。

第四步：日常使用

知识库搭好之后，日常就是两件事：

有新信息就喂进去

"这是西门子能源的最新季报，帮我提取关键数据，更新公司研究和行业数据。"

有问题就直接问

"对比一下 GE 和西门子最近三个季度的订单增速，谁在加速？"

它会自己去知识库里找数据，给你分析。

自查清单：你的知识库搭对了吗？

结构层

  • [ ] 有 CLAUDE.md 文件，写清楚了你的研究方向和知识库结构？
  • [ ] 目录按 公司研究 / 行业数据 / 研报存档 分类？
  • [ ] 每家重点公司有独立的文件夹？

数据层

  • [ ] 存量研报转成了 markdown 格式？（至少重点公司的）
  • [ ] 财报摘要只保留和你假设相关的数据？（别存一堆用不上的）
  • [ ] 行业数据有独立的时间序列文件？

使用层

  • [ ] 新研报能顺手丢进去更新？
  • [ ] 问问题的时候 Claude Code 能从知识库找到答案？
  • [ ] CLAUDE.md 里有"找不到就说找不到"的规则？（防止 AI 瞎编）

和假设追踪的联动

  • [ ] 知识库的数据能直接更新假设追踪文件？
  • [ ] 周度复盘的时候能引用知识库中的原始数据？

最后说两句

教程四讲了怎么追踪假设，这一篇讲了怎么给假设喂数据。两者合在一起，就是一个完整的投资研究系统。

知识库存着你所有的证据和数据，假设追踪根据这些证据做判断。新的证据进来，知识库更新，假设的确定性跟着调整。

这套系统不会让你每次都赚钱。但它会让你每一次决策都基于充分的信息，而不是三个月前的模糊印象。

搭起来试试。
