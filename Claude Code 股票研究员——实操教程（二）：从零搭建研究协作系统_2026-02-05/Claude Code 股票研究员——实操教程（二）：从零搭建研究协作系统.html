Claude Code 股票研究员——实操教程（二）：从零搭建研究协作系统

前言

假设你正在研究一家 AI 硬件公司，打开了 20 篇研报、3 份专家纪要、5 个季度的财报电话会纪要。你花了 3 小时读完、做了 50 条笔记，最后写报告时发现——忘了哪条笔记来自哪份材料。

这就是我之前研究工作的常态。每次都觉得自己记性还行，每次都被打脸。

用上 Claude Code 研究协作系统后，同样的工作变成了：我说一句「帮我研究这家公司的竞争格局」，他自动读取我过去写的所有相关笔记、调取最新的行业数据、生成一份带来源标注的研究报告，全程 15 分钟。

这不是魔法，是协作系统的力量。（好吧其实有点像魔法）

01 什么是研究协作系统？

研究协作系统 = 你的「第二大脑」。他记住你读过的每一份材料、追踪你验证过的每一个假设、按照你习惯的方式输出报告。

说白了就是一个比你记性好一万倍的研究助理，而且不用请他吃饭。

核心组件只有三个：

  • 知识库：存放所有研报、笔记、数据文件的地方
  • 配置层：告诉 Claude 你的研究偏好、写作风格，工作流程
  • 命令系统：用 /research、/kb-research 等命令触发自动化流程

听起来复杂？其实从零开始只需要 4 步。我当时搭建的时候也觉得很复杂，但跟着做下来发现，真的就是和 Claude 聊聊天的事。

02 前置准备

在开始之前，确保你已做好以下准备：

1. Obsidian 或任意文件夹用来存放你的研究文件。Markdown 格式最佳（.md 后缀），没有格式要求，Claude 能自动读取。不用 Obsidian 也行，但我强烈推荐，因为真的很好用。

2. 学会用终端打开 Claude右键你的工作文件夹 → 用终端打开 → 输入：

claude --dangerously-skip-permissions

3. 切换到 Opus 4.5进入 Claude 后，输入 /model，把模型调到最强版本。研究工作需要深度推理能力，普通模型不够用。咱要用就用最聪明的模型给咱打工。

4. 准备好你的资料把过去写过的研究报告、研报摘要、Excel 数据都放进工作文件夹。Claude 会读取并学习你的研究习惯。你喂给他的东西越多，他就越了解你。

03 第一步：用自然语言搭建基础框架

这是最关键的一步。很多人在这一步直接让 Claude「帮我搭建研究系统」，然后得到一个泛泛而谈的框架。

正确做法是：用 Plan Mode + 详细描述你的工作流程。

如何进入 Plan Mode？按两次 Shift + Tab，左下角会显示「Plan Mode」。

Prompt 示例：

我是一个对冲基金研究员，主要研究大宗商品，电力设备、新能源方向。我每天的工作流程是：
1. 监控自选股异动，找到需要深入研究的公司
2. 收集资料：研报、专家纪要、财报电话会
3. 形成投资观点：核心假设、估值测算、风险监控
4. 写研究报告

我之前用 Claude 做研究最大的痛点是：
- 每次都要重新告诉 Claude 我的写作风格
- 找不到之前做过笔记的材料
- 研究结论和原始资料对不上

请帮我设计一个研究协作系统，包括：
1. 文件夹结构
2. 配置文件结构
3. 常用命令设计
4. 和 Claude 协作的核心原则

Claude 会输出一个详细的设计方案。你可以追问细节。

小技巧： 在 Plan Mode 里追问是零成本的。Claude 不会嫌你问题多，你问得越细，最后的系统越好用。我当时问了他大概二十几个问题，他一个都没抱怨（废话他也不会抱怨）。

04 第二步：建立你的「记忆层」

研究协作系统的核心优势是「记住你的一切」。但 Claude 的记忆是分层的，你需要理解并正确配置。

这个概念一开始我也没搞懂，后来被 Context 超限坑了好几次才明白。所以这里我尽量说人话。

记忆分层结构：

为什么分层这么重要？因为 Claude 的 Context 有限制（20 万 tokens）。如果你把所有东西都塞进 CLAUDE.md，每次对话光读配置就要花掉一半额度。我之前就犯过这个错误，写了一个超长的 CLAUDE.md，结果聊几句就超限了，血的教训。

实操建议：

1. CLAUDE.md 只写 3 条核心原则

# 全局协作原则
1. 直接行动，减少中间步骤
2. 用列表点反馈和总结
3. 先规划再执行

2. workspace-config.md 写你的研究偏好

# 研究偏好配置
## 投资风格
- 基本面驱动 + 供需框架 + 估值纪律

## 写作风格
- 数据驱动，用具体数字替代形容词
- 开头点明核心观点
- 大量表格对比

3. 具体研究内容存在动态文件里每次做新研究时，创建新的笔记文件。Claude 会根据配置自动读取风格要求。这样你的研究笔记可以写得很长，反正他只在需要的时候才读。

05 第三步：设计命令系统

用 Claude 研究，最爽的事情是：只需要说一句话，整个流程自动跑。

这就是 Skill（命令）的作用。你可以把它理解成给 Claude 写的「标准操作手册」，以后他就按这个手册干活。

创建第一个命令 /research：

帮我创建一个 /research 命令，放在 .claude/skills/research.md

功能：
1. 读取我的研究配置和最新研究笔记
2. 询问我要研究的主题
3. 自动搜索最新资料（WebSearch）
4. 生成研究报告初稿
5. 标注来源

输出格式：
- 开头：核心观点
- 供需分析（表格对比）
- 估值测算（标注时间戳）
- 风险监控
- 参考资料列表

Claude 会帮你创建一个完整的 Skill 文件。以后只需要输入：

/research AI 硬件竞争格局

它就会自动执行整个研究流程。是不是很爽？我第一次跑通的时候差点从椅子上跳起来。

06 第四步：用好「反思系统」

研究最大的陷阱是「自圆其说」——你找到了支持自己观点的资料，却忽略了反面证据。

这个毛病我也有，而且还挺严重的。所以我让 Claude 在研究结束后强制问我一些「灵魂拷问」。

核心问题清单（每次研究形成结论时使用）：

  1. 盈利能力与护城河：市场交易的是什么因素？
  2. 业务变化趋势：护城河是变宽还是变窄？
  3. 短期与长期一致性：交易逻辑和投资逻辑是否匹配？
  4. 估值与预期：当前股价隐含了什么预期？

Prompt 示例：

我已经完成了对 XX 公司的研究，现在需要做反思验证。请逐一问我以上 4 个问题，我的回答会帮助我完善投资观点。

Claude 会像苏格拉底一样追问，帮助你发现盲点。这比你自己闷头检查有效得多。有时候被他问得哑口无言，才发现自己的逻辑确实有漏洞。

07 第五步：建立你的知识库

研究协作系统的另一个核心能力是「记住你读过的一切」。

我以前的研报都是看完就扔，下次要用的时候根本找不到。现在全部喂给 Claude，他比我自己还清楚我看过什么。

知识库使用流程：

  1. 把 PDF/研报放到文件夹的 inbox 目录
  2. 说「/kb-add」
  3. Claude 自动读取、生成摘要，重命名归档

检索命令：

你说
Claude 做
参考知识库
读取知识库索引
研报怎么说
读取该公司的研报摘要
/kb-research AI 芯片
启动知识库驱动的研究流程

和普通搜索的区别：

普通 WebSearch 是从互联网上搜资料。知识库检索是：从你过去读过的所有材料中找答案。

这意味着：

  • 你之前做过的研究不会白做
  • 你过去的观点和结论会被持续引用
  • 你的「研究画像」会越来越清晰

说白了就是，你终于可以站在自己的肩膀上了，而不是每次都从零开始。

08 实战案例：从想法到报告的完整流程

假设你想研究「数据中心备用电源」这个主题。这是我前段时间实际做过的一个研究，拿来当例子。

Step 1：启动研究

/research 数据中心备用电源的竞争格局
要求：
- 结合我之前写的电力设备研究笔记
- 搜索最新的行业数据和公司信息
- 对比我关注的几家供应商（列出优劣势表格）
- 标注数据来源和时间戳

Step 2：阅读初稿Claude 生成初稿。你阅读后指出需要补充的地方：

补充一下天然气发电和柴油发电的成本对比，以及政策因素（碳排放要求）的影响。

Step 3：反思验证

/reflect

Claude 逐一追问 9 个核心问题，帮助你发现盲点。

Step 4：输出报告完成所有补充和反思后，说：

生成正式研究报告，格式按我的写作偏好。

整个过程从想法到报告，大约 30-45 分钟。如果你手动操作同等深度的研究，至少需要 3 小时。省下来的时间可以多喝两杯咖啡，或者多研究一个标的。

09 常见问题和避坑指南

这些都是我踩过的坑，希望能帮你省点时间。

Q1：Claude 读不到我文件夹里的文件？A：确保文件格式是 .md 或 .txt。PDF 需要先用工具转成 Markdown。我之前就是因为这个卡了半天，后来发现 Claude 根本读不了 PDF，得先转格式。

Q2：研究结论和原始资料对不上？A：要求 Claude 在输出时标注来源。知识库检索的结果标注 [KB]，WebSearch 的结果标注 [WS]。这样你就知道哪句话是从哪来的了。

Q3：上下文用完了怎么办？A：Claude Code 的 Context 上限是 20 万 tokens。如果经常超限：

  • 把大文件拆分成小文件
  • 用 Skill 封装流程
  • 让 Claude 少说废话（这个很重要，他有时候真的很啰嗦）

Q4：研究太发散，收不住？A：在 Prompt 里限定范围。比如：「只研究中国市场的竞争格局」「只分析电源设备，不涉及储能」。不然他会给你写一篇论文出来。

Q5：零代码能做吗？A：完全可以。我本人没有任何编程基础，曾经尝试两次学习 Python，但就停留在了 print("Hello world!!!") 的阶段。但这次我从零到搭建完成只用了不到一周。Claude 会帮你写所有代码，你只需要确认和调整。

10 结语：AI 时代的竞争优势

研究工作正在被 AI 重构。

以前，一个研究员一天最多深度研究 1-2 个标的。

现在，配合研究协作系统，一个研究员一天可以：

  • 完成 3-4 个标的的快速筛选
  • 对 1-2 个核心标的做深度研究
  • 产出 1 份带来源标注的研究报告

但这不是最重要的。

最重要的是：你的研究经验会被系统沉淀。

你过去读的每一份研报，做过的每一个笔记、形成的每一个观点，都不会白费。他们会被系统记住，在未来的研究中持续被引用和迭代。

这就是「第二大脑」的力量。

当然，AI 再厉害也只是工具。最后做决策的还是你，赚钱亏钱也是你的。所以请肩负起作为人的责任吧！（笑）
