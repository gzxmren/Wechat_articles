把 Claude Code 拆开看，核心代码只有 165 行

 

 

这是个能让你从 0 到 1 手搓一个 coding agent 的教学仓库，也可能是目前理解 AI 编程工具最清晰的路径。

前两天让openclaw帮我查找github上，关于claude code和openclaw相关的有趣项目时，发现了这个repo，出于好奇下载下来看了看，发现对理解claude code和openclaw是如何工作的挺有帮助，这两天也在学。这里分享给大家。

学习这个教学仓库你得有基本的python基础，它的教学都是基于python 语言的。

是什么

learn-claude-code 是一个教学仓库，把 Claude Code 这类 coding agent 的核心架构拆成了 20 个渐进章节，每章配有可运行的 Python 代码（共 8500+ 行），让你一步步自己写出一个类似系统。它不追求 1:1 抄源码，而是追求一件事：

让你理解"为什么这么设计"，然后自己能重新造出来。

核心理念：模型负责思考，代码负责给模型提供工作环境

这句话是理解整个仓库的关键。

Claude Code 本质上不是一个"智能模型"，而是一个 harness（操控框架） —— 给大模型提供一个能干活的环境：

  • • 能读文件、写文件、跑命令（工具）
  • • 能把大任务拆成小步骤（规划）
  • • 能控制上下文不爆炸（上下文管理）
  • • 能防止模型乱执行危险操作（权限控制）
  • • 能把经验记下来跨会话复用（记忆）
  • • 能把工作分给子 agent 并行处理（多智能体）

模型负责推理，harness 负责让这个推理过程变成真实的生产力。 这个仓库教的，就是这个 harness 的搭建方法。

四个阶段：能跑，跑稳，跑久，分工跑

这是整个仓库最用心的设计 —— 它没有按claude code的源码目录顺序教，而是按机制依赖顺序教。也就是说：后一章必须建立在前一章的理解之上。

第一阶段：让 agent 能跑（s01-s06）
  → 最小的可用闭环
  
第二阶段：让它不乱跑（s07-s11）
  → 安全、扩展、记忆、恢复
  
第三阶段：让它能长期跑（s12-s14）
  → 持久化任务、后台任务、定时调度
  
第四阶段：让它分工跑、隔离跑（s15-s19）
  → 多 agent 协作、工作树隔离、MCP 插件

阶段一：从 165 行代码开始

s01_agent_loop.py 只有 165 行，就是一个最朴素的循环：

用户输入 → 模型回复 → 有工具调用就执行 → 把结果喂回去 → 继续

跑通这个循环，你就已经拥有了一个真的能写代码的 agent。接下来所有复杂功能，都是在这个循环上加零件。

阶段二：加安全，加记忆，加恢复

有了能跑的 agent，下一步不是加更多功能，而是针对它会遇到的问题去解决问题：

  • • s07 权限系统：模型想跑 rm -rf /需要先过安全门
  • • s08 Hook 系统：不改核心循环就能扩展行为
  • • s09 记忆系统：跨会话记住有用的信息
  • • s10 系统提示词：把规则、工具、上下文组装好
  • • s11 错误恢复：出错了怎么继续，而不是直接崩掉

阶段三：从临时清单到持久化平台

s12-s14 把会话里的临时规划升级成真正的任务系统：

  • • s12 任务系统：持久的任务图
  • • s13 后台任务：非阻塞执行
  • • s14 定时调度：按时间触发的自动化

阶段四：从单兵到团队

最后阶段才是真正的高级能力：

  • • s15 Agent 团队：持久化的多 agent 协作
  • • s16 团队协议：统一的协调规则
  • • s17 自主 Agent：自我认领任务、自我恢复
  • • s18 工作树隔离：每个任务独立的工作空间
  • • s19 MCP 插件：接入外部工具平台

它不会教你什么

大多数技术教程的问题是什么都讲，最后读者淹没在细节里。而这个仓库明确说了不教什么：

  • • 打包、编译、发布流程
  • • 跨平台兼容
  • • 企业策略、遥测、账号体系
  • • 历史兼容分支
  • • 只对特定产品环境有意义的胶水代码

不是这些不重要，而是它们不该成为初学者的主线。 应该让读者先抓住核心脉络，这些细节以后再说。

特点

每章都能跑

不是只写文档，每个章节都有对应的 agents/sXX_xxx.py，可以直接运行：

python agents/s01_agent_loop.py    # 最小 agent 循环
python agents/s18_worktree_task_isolation.py  # 工作树隔离
python agents/s_full.py            # 完整版

共 20 个 Python 文件，8500+ 行代码，每个文件教一个概念，从 165 行到 700 行不等。

"桥接文档"解决学习中的卡壳

除了 20 个主章节，仓库还有大量 bridge docs（桥接文档），专门回答初学者最容易困惑的问题，比如：

  • • s00d：为什么章节是这个顺序？
  • • s00f：本地代码文件该按什么顺序看？
  • • s00a：查询控制面是怎么工作的？
  • • s00b：一条请求在系统里是怎么流转的？
  • • s02a：工具控制面 vs 工具执行时的区别？
  • • s10a：消息和提示词的组装流水线

这些文档的存在，让学习者不至于在章节之间迷路。

自带 Web 学习界面

仓库内置了一个可视化学习网站：

cd web
npm install
npm run dev

提供时间线视图、阶段边界图、章节对比等功能，比纯看文档直观得多。

适合谁

  • • 想理解 Claude Code 底层原理的开发者 —— 不只是会用，而是知道它为什么这样设计
  • • 想自己造类似工具的工程师 —— 学完可以真的写出来
  • • 对 AI Agent 架构感兴趣的人 —— 这套设计模式适用于各种 agent 系统
  • • AI 团队的 tech lead —— 评估和选择 agent 框架时，有判断力

最低要求：会一点 Python，知道函数、类、字典、列表就够了。

建议学习路径

  1. 1. 先看架构总览（s00），建立一个全局的概览，大概知道都有哪些功能要实现
  2. 2. 按顺序读 s01 → s06，每学完一章，停下来自己手写一个最小版本
  3. 3. 读 s07 → s11，补全安全与健壮性
  4. 4. 读 s12 → s14，升级成持久化平台
  5. 5. 读 s15 → s19，进入多 agent 世界
  6. 6. 最后看 s_full.py，把所有机制串起来
  7. 7. 遇到不懂的名词和概念，可以阅读桥接文档，还看不懂可以问AI哦

不足之处

可以明显的看出这个教学库的代码应该是多人或多AI编写的，前后代码包括但不限于不连贯、重复编写、丢失代码等情况。

当然我们应该学习的是它好的地方，比如如何一步一步构建一个coding agent的思路。

最后

不要等全部看完再动手。每学完一个阶段，自己写一个最小版本，比读十遍文档都有用。

如果你想知道 Claude Code 的 harness 到底是怎么工作的，而不是停留在"它能帮我写代码"的表层认知，这个仓库是目前最清晰的学习路径。
