GitHub爆火项目OpenCLI：给 AI Agent 装上万能遥控器

 


Karpathy 曾吐槽过一句话，我觉得特别在理。他说，让用户去网页上点按钮、填表单，这事儿其实挺“粗鲁”的。

为什么？因为在 AI 眼里，那些花里胡哨的图形界面（GUI）简直就是沟通的障碍。AI 不需要看图标，它只需要指令。

最近 GitHub 上有个叫 OpenCLI 的项目火了，短短几天就冲上了 2000 多颗星。它干的事儿就一件：把那些原本只能点、只能看的软件，统统变成一行行冰冷的命令。

不管是 B 站、知乎、小红书这种网页，还是微信、Cursor、Notion 这种桌面应用，在 OpenCLI 面前，都得乖乖听话。

给 AI 装上万能遥控器

我有个做开发的朋友跟我抱怨，现在的 AI Agent 看着聪明，其实很笨。为什么？因为它进不去你的软件啊！

你想让 AI 帮你把 B 站的热门视频总结一下发到飞书群里？太难了。以前你得去申请 API Key，求爷爷告奶奶地看文档，很多平台压根就不开放接口。

OpenCLI 的出现，直接把这道墙给拆了。

# 下载小红书笔记中的图片/视频
opencli xiaohongshu download abc123 --output ./xhs

opencli list -f yaml            # 用 YAML 列出命令注册表
opencli bilibili hot -f table   # 默认：富文本表格
opencli bilibili hot -f json    # JSON（适合传给 jq 或者各类 AI Agent）
opencli bilibili hot -f yaml    # YAML（更适合人类直接阅读）
opencli bilibili hot -f md      # Markdown
opencli bilibili hot -f csv     # CSV
opencli bilibili hot -v         # 详细模式：展示管线执行步骤调试信息

它的逻辑特别野：既然你在浏览器里已经登录了，那我就直接借你的账号用用。

这事儿最妙的地方在于零配置。

你不需要去搞什么 Cookie，也不需要配置密钥。OpenCLI 在你本地起一个守护进程，再装个浏览器插件，就像个传话筒一样。你在终端敲一行 opencli bilibili hot，它就通过 WebSocket 告诉浏览器：“嘿，去把 B 站热门抓回来。”

浏览器乖乖执行，把结果吐回终端。你的账号、密码，从头到尾都没离开过浏览器，安全得很。

它是怎么进入 Electron 应用的？

光搞定网页还不算完，OpenCLI 最近还干了一件大事：把 Electron 应用也给收编了。

你可能不知道，现在你电脑上那一堆桌面应用，什么飞书、VS Code、Discord、Figma，甚至微信开发者工具，有一大半其实都是披着外壳的浏览器。

既然是浏览器，就逃不过 Chrome DevTools Protocol（CDP）这双眼睛。OpenCLI 直接给每个应用分配个端口，Cursor 是 9222，ChatGPT 是 9224，通过 CDP 直接跟应用内核对话。

这就好比 Agent 绕过了前台，直接进了经理办公室发号施令。

有个技术细节我觉得挺有意思。它在处理 Notion 这种富文本编辑器时，没有傻乎乎地去改代码里的 .value（那样框架根本不认），而是模拟了真实的键盘输入 document.execCommand('insertText')。

这意味着什么？意味着 Agent 可以像真人一样，在 Cursor 里写代码，在 Notion 里写文档，在 Discord 里发消息。

OpenCLI 也可以作为现有命令行工具的统一入口，负责发现、自动安装和纯透传执行。

两条路，通向同一个未来

之前有个叫 CLI-Anything 的项目也很火，但它俩的路数完全不同。

CLI-Anything 像是个拿着手术刀的医生，它要把软件的源码剖开，分析底层 API，然后生成 CLI。这招对开源软件好使，但遇到闭源的网站或者 App，它就歇菜了。

OpenCLI 呢？它更像是个老练的用户。我不看源码，我就看你点哪个按钮会有反应，然后我照着做。

一个是从“源码”出发，一个是从“体验”出发。

这俩兄弟凑一块，基本上把咱们能见到的软件给全覆盖了。有源码的找 CLI-Anything，没源码的网站和 Electron 应用找 OpenCLI。

Agent 的母语

为什么 OpenCLI 要专门为 AI 设计 explore、synthesize、cascade 这三个命令？

我觉得这才是它最核心的价值。

当 AI 遇到一个陌生的网站，它可以用 explore 像人一样去点一点、滚一滚，摸清这网站有哪些能用的接口；然后用 synthesize 自动生成一个适配器；最后用 cascade 去试探权限。

这不就是人类探索新软件的过程吗？

以前我们说“万物皆可 API”，现在看来，可能是“万物皆可 CLI”。

对于 AI Agent 来说，GUI 是人类给自己画的妆容，好看但难卸。而 CLI 才是它们原本的素颜，是它们的母语。文本输入、结构化输出、没有歧义、可组合，这些特性简直是为 AI 量身定做的。

权力与风险并存

当然，这事儿也不是没有隐患。

OpenCLI 复用的是你的登录态，这意味着 Agent 拥有的权限和你一模一样。你能发弹幕，它就能发；你能删文档，它就能删。

这就是一把双刃剑。

好处是省事，拿来就能用；坏处是，万一 Agent “发疯”了，或者你给的指令出了偏差，它搞出来的乱子，可都是算在你头上的。

以前我们担心 AI 会不会毁灭世界，现在我们得先担心它会不会在你的微信群里乱发消息。

软件的边界正在消失

过去四十年，人类拼命给计算机穿上 GUI 这件漂亮的外衣，让鼠标和触摸屏统治了世界。现在，AI 来了，OpenCLI 和 CLI-Anything 正在帮计算机把这件外衣一件件脱掉。

软件不再是孤岛。

你可以写个脚本：从 B 站抓字幕 -> 让 AI 总结 -> 存进 Notion -> 推送到飞书。这种以前只有大厂系统工程师才能搞定的自动化流程，现在一个人加一个 Agent 就能跑通。

试着去用用这些工具吧。在 AI 时代，命令行可能才是那个最接近真理的入口。
