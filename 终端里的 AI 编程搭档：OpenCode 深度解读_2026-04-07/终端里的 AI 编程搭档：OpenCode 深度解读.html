终端里的 AI 编程搭档：OpenCode 深度解读

120K ⭐、800+ 贡献者、500 万月活开发者——这个开源 AI 编程工具正在重塑终端开发体验。

🔥 为什么开发者都在聊 OpenCode？

如果你还在用传统方式写代码，OpenCode 可能是一个值得认真了解的选项。

它不是又一个"AI 代码补全插件"，而是一个终端原生的 AI 编程 Agent——你可以直接用自然语言与它对话，让它读代码、写代码、执行命令、帮你 Debug，甚至调用 MCP 工具扩展能力。

GitHub 上超过 12 万颗星，每月有 500 万开发者在使用它，这不是一个玩具项目。

⚡ 安装：一条命令搞定

支持几乎所有主流平台：

# 最快方式
curl -fsSL https://opencode.ai/install | bash

# Homebrew (macOS/Linux)
brew install anomalyco/tap/opencode

# npm
npm install -g opencode-ai

# Arch Linux
sudo pacman -S opencode

# Docker
docker run -it --rm ghcr.io/anomalyco/opencode

Windows 用户可以用 scoop、choco 或直接下载二进制。安装完成后，一条 opencode 命令直接进 TUI。

🧠 核心特性：为什么它值得用

1. 多模型支持 — 不绑定任何厂商

OpenCode 基于 AI SDK + Models.dev，支持 75+ LLM 提供商，包括：

  • • Anthropic Claude 系列（Claude 3.5/3.7 Sonnet、Claude 4 Opus）
  • • OpenAI GPT-4o、O1、O3 系列
  • • Google Gemini 2.0/2.5 Flash/Pro
  • • AWS Bedrock（Claude）
  • • Azure OpenAI、Groq、OpenRouter
  • • 本地模型（Llama 4、Qwen 等）

更重要的是：可以随时切换模型，上下文完全保留。

2. LSP 原生集成 — AI 真的"看懂"了你的代码

OpenCode 自动加载对应语言的 LSP 服务：

  • • Go → gopls
  • • TypeScript → typescript-language-server
  • • Nix → nil
  • • ……以及你能想到的所有主流语言

AI 在做决策时，拥有和你一样的代码上下文，而不是靠"猜测"。

3. MCP 协议扩展 — 连接一切

通过 Model Context Protocol (MCP)，OpenCode 可以连接外部工具和服务，支持三种传输方式：

// stdio：命令行服务
// http：HTTP 接口
// sse：Server-Sent Events
{
  "mcp": {
    "github": {
      "type": "http",
      "url": "https://api.githubcopilot.com/mcp/",
      "headers": { "Authorization": "Bearer $GH_PAT" }
    }
  }
}

这意味着你可以让 AI 直接操作 GitHub Issues、数据库、API……只要有人写 MCP Server。

4. 多会话 & 上下文管理

  • • 多个并行会话，同一项目内多 Agent 协作
  • • /share 生成分享链接，团队协作无障碍
  • • Auto Compact：对话过长时自动压缩上下文，防止超出限制
  • • 持久化存储（SQLite），会话不丢

5. 隐私优先

OpenCode 不存储任何代码或上下文数据，可以在离线/内网环境运行。适合对代码安全有严格要求的企业团队。

🛠️ 进阶用法：从入门到精通

自定义命令（Custom Commands）

把常用 Prompt 模板存成文件，快速调用：

# 创建命令文件
~/.config/opencode/commands/git-commit.md

内容示例：

RUN git diff --staged
RUN git log --oneline -5
---
帮我根据 git diff 写一个规范的 commit message，参考最近的提交风格。

使用 Ctrl+K 唤起命令面板，输入 user:git-commit 即可执行。

支持命名参数：

# Fetch Context for Issue $ISSUE_NUMBER
RUN gh issue view $ISSUE_NUMBER --json title,body

配置管理：多层覆盖

配置优先级（从低到高）：

远程配置（组织级）→ 全局配置 → 自定义配置 → 项目配置 → 行内配置

示例 ~/.config/opencode/opencode.json：

{
  "model": "anthropic/claude-sonnet-4-5",
  "small_model": "anthropic/claude-haiku-4-5",
  "autoupdate": true,
  "mcp": {
    "filesystem": {
      "type": "stdio",
      "command": "node",
      "args": ["/path/to/mcp-server.js"]
    }
  },
  "lsp": {
    "go": { "command": "gopls", "disabled": false }
  },
  "permission": {
    "bash": { "rm -rf *": "deny" }
  }
}

企业级部署：MDM 托管配置

macOS 企业用户可通过 .mobileconfig（Jamf、FleetDM 等）强制推送配置，无法被用户覆盖：

<key>ai.opencode.managed</key>
<dict>
  <key>share</key>
  <string>disabled</string>
  <key>server</key>
  <dict>
    <key>hostname</key>
    <string>127.0.0.1</string>
  </dict>
</dict>

Plan 模式：安全变更

切换到 Plan 模式（Tab 键）后，AI 只能读代码、给建议，不能做任何修改。适合：

  • • 新人 Code Review
  • • 高风险重构前先讨论方案
  • • 给团队 leader 演示方案思路
