AI投研，自己搭还是等现成的？

点关注不迷路


写了七篇 Claude Code 教程，身边小伙伴的反应挺有意思——真正自己动手搭的人有一些，但更多人的反应是：

"能不能有个现成的？我不想找API，不想配环境变量，更不想对着黑乎乎的终端打字。"

理解。不是所有人都喜欢折腾。很多人就是想要一个成熟的打包产品——模块化的功能，设计好的交互界面，打开就能用。

最近做AI投研的创业公司越来越多，Alpha Engine、Gangtise、Alpha 派……各种产品层出不穷。有人开始焦虑了：是不是不用AI做投研，就要被淘汰了？

我用了几个月AI做投研，今天不推荐具体产品，聊聊我的真实感受。

核心观点先放在前面：不用焦虑，慢一点没关系。

现在市面上都有什么？

AI投研工具大体分两条路线。

一条是成品路线——注册就能用，开箱即有。像 Alpha Engine（以下简称AE），装好就自带研报库、会议纪要、行业点评，还有内置模型帮你分析和写报告。你不用操心数据从哪来、模板怎么写，产品经理已经帮你想好了。其他如gangtise，alpha派之类的估计也类似，这里只是以它为例子

另一条是自建路线——我自己在用的 Claude Code。什么都没有，但什么都可以搭。数据源你自己接，工作流你自己定义，输出格式你自己设计。像是招了一个什么都不会但学习能力极强的实习生，你得花时间教他，但教会了他就完全按你的方式干活。

这两条路没有好坏之分，但区别也不小。


三层对比

我把AI投研工具拆成三层来看：

层级
成品路线（AE 类）
自建路线（Claude Code）
数据层
自带研报、纪要、点评，国内覆盖全，开箱即用
模型自带
自己接API（FMP、Tushare等），灵活但要花时间配
自选模型
工作流层
预设模板 + 可导入Skill，上手快
完全自定义Skill和命令，从零搭建
输出/记忆层
有记忆能力，可以设定模板，但个性化有天花板
分层记忆架构，高度个性化，你说什么它都能记住

成品路线卖的是"你不用想"，自建路线卖的是"你想怎样都行"。

以AE为例，稍微展开说说（声明：非广告，纯个人体感，我自己用了两次）。

它的核心优势是门槛极低。不需要 API Key，不需要 Docker，不需要终端，安装完就能用。数据源是现成的——国内券商研报、会议纪要、行业点评，日更量很大。你想了解一个公司，直接问它，它会从自己的数据库里找资料回答你。

而且它写出来的东西是真的能用的。让它帮你梳理一个行业、写一份初步报告，出来的质量已经完全可以作为研究的起点了。对于快速了解一个不熟悉的领域，这种"问一句就给你一份像样的报告"的体验，确实比自己从零配置要高效得多。

它也支持导入 Skill，相当于你可以给它一个模板，告诉它"按这个格式帮我分析"。知识库、会议纪要整理、翻译这些基础功能也都有。从框架上来说，已经很完整了。

不足的地方是，DIY 空间有限。你没法像 Claude Code 那样深度定制每一个环节——从记忆架构到输出格式到工作流触发条件。它给你的是一个设计好的工作台，你可以调整桌上的东西，但不能改桌子本身。另外用户暂时没法自己选择底层模型。

但说实话，对大多数人来说，这些限制根本不是问题。因为大多数人要的不是一张可以随意改造的桌子，而是一张摆好了就能干活的桌子。

我为什么选了"自己搭"？

坦白说，不是因为自建路线更好，是因为我这个人喜欢折腾。

AE那种产品给我的感觉是——功能很全，但有些东西我压根不想看。点评公告？我不需要。预设的信息推送？太多了。我更喜欢做减法，只留我需要的。

我的理念是 MVP——最小可运行版本。先让系统跑起来，然后不断修改，不断砍。

举个例子，我自己搭了一个播客模块。我平时听很多高质量的AI相关英文播客，Claude Code 帮我自动抓取全文，翻译成中文，整理要点。这个功能很消耗token，但对我很有价值，没有任何成品工具会专门做这个，因为这是只属于我的需求。

还有知识库——我用 Obsidian 插件一键保存网页内容，再让 Claude Code 一键归档、生成摘要。整个链路是我的工作习惯长出来的，不是产品经理设计出来的。

而且我会定期砍功能。发现某个模块两周没用过？删掉。某个 Skill 输出的东西我从来不看？砍了。保持整个系统干净整洁。

大部分人用工具是不断加功能，我是不断减功能。

我选自建不只是因为效率，是因为搭这个东西本身让我开心。

做二级研究这么多年，产出永远是报告、观点、交易决策——都是无形的。搭 Claude Code 系统可能是我职业生涯里第一次真正在"做一个产品"——有架构设计、有版本迭代、有功能取舍，用户就是我自己。

而且这个过程给了我一些意外的收获。MVP、快速迭代、做减法、顶层架构设计——这些工程师思维，反过来影响了我做投研的方式。怎么快速建立对一个新行业的认知框架？先搭一个最小可用的理解，然后不断用新信息去修正，砍掉不重要的变量，保留核心逻辑。这不就是搭系统的思路吗？

成品工具给你效率，自建给你搭积木的快乐。

当然，这条路不适合所有人。它需要你愿意花时间折腾，愿意忍受前期的混乱，愿意在"搭系统"和"做研究"之间分配精力。

如果你搭不动，那不是你的问题，是这条路本来就不适合所有人。


几个月下来，我最大的认知变化

一开始我觉得 Claude Code 就是一个高级工具——帮我快速收集信息、获取数据、写简单的报告。一个效率提升器，仅此而已。

用了几个月之后发现不是这样的。它更像一个和我互补的搭档。

很多新行业的逻辑我并不了解，包括对一些宏观、资金流的理解也不到位，但它至少给我提供了一个70-80分的视角，让我能快速建立起"大致发生了什么"的认知框架。我想了解一下量化策略的逻辑，它可以帮我模拟、调整策略参数。

有一次让我印象特别深。

我把自己的模拟组合告诉了 Claude，本来只是做个记录。结果它突然反问我：

"你的假设里对AI硬件做了这么多研究，置信度也很高，但模拟组合里AI硬件的股票权重不到10%——是不是知行不合一？"

说实话，那一下就击中了。我自己盯着模拟组合看一百遍都没意识到这个矛盾。其实我的内心还是畏高，畏高都是苦命人啊。

这种事情，帮你查数据的工具做不到，帮你写报告的工具也做不到。它需要AI同时理解你的研究体系、你的假设判断、和你的实际行为，然后把三者放在一起，发现矛盾。

这是深度个性化带来的独特价值——它不是在执行你的指令，而是在"理解你"之后给你反馈。目前这种能力，成品工具还做不到，因为它需要你长期和AI共同搭建起的那套记忆和上下文，他是你独有的助理，不是别人共享的模板。

AI是你的放大器

说回那个最常收到的问题——"搭不动怎么办？"

我的回答是：那就别搭，等一等。

AI投研工具这个赛道，现在有大量的创业者在做。他们比你更懂技术，比你更了解投研需求，他们的全部精力就是把这些模块打磨好、打包好，端到你面前。

自有大儒为你辩经。

你要做的不是焦虑自己跟不上AI，而是把自己的投研能力练扎实。因为有一件事我用了几个月AI之后非常确定：

你强，它更强。你弱，它就蠢。

AI是加成，不是替代。它做的事情是让你的工作流以五倍、十倍的速度运行，把大量的垃圾工作时间省掉——收集数据、整理纪要、对比财务指标、写初稿——这些以前要花一整天的事情，现在可能半小时就搞定了。

但它放大的是你本来就有的东西——你的判断力、你的行业理解、你的投资框架。如果你自己对一个行业没有基本的认知，AI给你的70-80分视角你根本没法判断对错。

投资永远不是短跑，是马拉松。只要保证自己不出局，慢一点是没有关系的。

AI是放大器，不是指南针。

方向，得你自己找。
