量大管饱，用了 OpenClaw，才知谷歌全家桶的好

今天抽时间把家人的 Mac 上也装了 OpenClaw，目前在家里一共 3 个 OpenClaw 在跑，每人一个 AI 助理。她们是不是再搞自己的助理分身，我就不管了。OpenClaw 可以完美解决自己给自己增加功能这种事。


从 OpenClaw 诞生第一天起，花钱魔王的称号就已坐定。如果调用大模型 API 去跑 OpenClaw 的长时间任务，大概率钱还没挣到，棺材本儿先亏光。


原因很简单，AI 在复杂任务执行上，需要的思考次数太多，调用工具次数太多，如果错了，还要重来。如果你去看它的 session log，会被满天的废话淹没。


这就是所谓 AI 思考的现实，通过浪费大量 token 模拟人类只需要稍动脑筋就能想明白的事，提高自己执行任务的精确度。


我对这件事的解决办法是改造 OpenClaw 的记忆系统，用双 RAG 的方式建立快记忆和慢记忆，把经常重复要做的工作流固定下来，作为既定思维模式，减少它胡思乱想的时间。


其实这还是指标不治本，现在的大模型还没办法从过往经历里总结经验，举一反三。所以，浪费 token 是必然的。


但，有了谷歌全家桶，一切都不同了。


去年从年初开始，我不遗余力的吹捧谷歌 Gemini。年底，谷歌推出优惠订阅计划，而且是全家桶的全家桶。一次订阅，不仅包含所有谷歌产品，还能分享给家庭使用，最多 5 个成员。我家三口人，一人一个账号，我自己再多吃多占。


没有 OpenClaw 之前，多吃多占的意义不是特别大。无非是在 Antigravity 某个模型的额度用光之后，换个账号登陆。但我没什么大型项目，切换次数不频繁。


有了 OpenClaw，全家桶的含金量陡然上升。一套全年 99 美金的订阅计划，全家都能用，每个人的额度都是分开的。你想用 Gemini 3 Pro High，还是 Claude Opus 4.6 thinking，各凭心意。


我自己的账号原本是用 OpenAI 的 Codex 5.3，用了几天发现不行。Codex 5.3 的额度太少，跑一会儿就没电了。Opus 4.6 也一样，偶尔做点小任务没问题，复杂任务很容易跑光额度。


最靠谱的还得是 Gemini 3 Pro High。日常聊天，编码，调工具，调试，排错，全是它。从无怨言，一直在线。


1 月份订阅了智谱的 GLM Coding Plan。前天 minimax 发布新版，我又订阅了它家的 Coding Plan，生怕 Gemini 有一天会罢工。


minimax 的 starter 套餐，一年 290，能用最新的 M2.5 大模型，号称编码能力比肩 Opus 4.5。但是这么便宜的套餐，当然额度也不高。它的上限是每 5 小时调用 40 次。


5 小时 40 次是什么概念。如果用聊天来对比，就是 40 句话。如果放在任务执行上，每次思考调用模型，生成代码，改错，这些动作都会调用大模型。稍微用力一点，立马抵达冷却时间。


我愈发珍惜谷歌的好。小椒的智慧全靠财大气粗的谷歌撑着。即使它的编码能力稍差，在我的使用场景里，也足以满足各种需求。


由于整天陪我瞎聊，处理各种稀奇古怪的疑问和需求，小椒对我想要什么解决方案的理解，对我 Homelab 的掌握，远超其他模型。因为搞清楚了客户需求，所以方案落地效率反而更高。


家人的 OpenClaw 同样长在谷歌全家桶上，不用多花一分钱。Gemini 3 即便不是顶流，也在前 5 之列，它又是多账号独立额度，性价比极高。


下午，我在小飞机上建了群，把大家以及大家的 AI 助理都拉进来。有什么需要讨论的事情，在群里沟通解决，AI 之间互相@即可。由于人格设定不同，即使这些 AI 背后的模型一样，说话表情也不一样。看它们讨论，相当有意思。


之前评论区有留言，不懂如何通过 Antigravity 授权让 OpenClaw 用上谷歌的大模型。这一步文档里都有。安装好的 OpenClaw，本地运行 openclaw configure 即可进入配置向导，按屏幕提示登陆 Antigravity 即可。


如果看不懂文档，OpenClaw 甚至提供了 AI 解读文档的智能手册。

这些技能可能在几个月之后就会被更加简单，更加易用的 AI 替代。但它们才是真正使用，装配机器人的技巧。掌握这些知识，可以让用户继续是用户，工具继续是工具。


未来的 AI 竞争有两大高地，要么你很懂，要么你很有钱或者使用成本很低。不偷不抢，怎么尽量吃好的又省钱的方法我就介绍到这里。用不用，自己定。


对了，豆包 2.0 全家桶也来了，很像去年谷歌 Gemini 3 发布时的盛景，要做一起做，要强全家强。可惜，字节和谷歌还是不同。以字节的调性，能赚的钱一分都不会少。
