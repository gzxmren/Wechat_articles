SuperClaude：给 Claude Code 加一套工作流系统

Claude Code 现在有官方的 skills 系统。SuperClaude 是第三方做的另一套东西。两者容易混淆，先说清楚区别。

官方 skills vs SuperClaude

官方 skills 解决的是"能不能"的问题。比如生成 Word 文档、处理 Excel、创建 PDF。这些是能力扩展，让 Claude 能做更多类型的事。

SuperClaude 解决的是"怎么做"的问题。它不增加新能力，而是把现有能力组织成固定流程。你输入一个命令，它按预设的步骤执行。

打个比方：官方 skills 像是给工人发新工具，SuperClaude 像是给工人一本操作手册。

SuperClaude 具体做什么

30 个斜杠命令，覆盖开发流程：

  • /brainstorm 头脑风暴
  • /design 系统设计
  • /implement 写代码
  • /test 生成测试
  • /research 深度调研
  • /document 生成文档

举个例子。你想研究某个技术方案：

/research "向量数据库选型"

它会自动执行多跳搜索，从一个来源跳到相关来源，最后整合成报告。这个流程是预设好的，你不用每次都写 prompt 引导。

16 个 Agent

框架内置了不同角色：PM、安全工程师、前端架构师、研究员等。

系统根据任务自动切换。问安全问题用安全模式，问架构问题用架构模式。本质上是预设了不同场景的 prompt 模板。

性能

官方数据：配合 MCP 服务器，速度快 2-3 倍，token 消耗少 30-50%。

不装 MCP 也能用，装了更快。

安装

pipx install superclaudesuperclaude installsuperclaude doctor

重启 Claude Code，输入 /sc 看所有命令。

什么时候用哪个

需要新能力（生成 PPT、处理表格）→ 用官方 skills

需要固定工作流（每次调研都按同样步骤）→ 用 SuperClaude

两者不冲突，可以同时用。

我的判断

SuperClaude 的价值是把 prompt 工程变成可复用的命令。省去每次重复写引导语的时间。

缺点：30 个命令要学，框架本身占上下文空间。

适合每天重度使用 Claude Code、有固定工作流需求的人。偶尔用一下的话，学习成本不划算。
