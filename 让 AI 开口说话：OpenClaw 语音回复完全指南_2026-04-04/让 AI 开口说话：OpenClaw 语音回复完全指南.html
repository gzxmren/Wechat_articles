你有没有想过，让 AI 用声音回你？

大多数人用 AI 助手的方式，都是打字发消息，等待文字回复。

这没什么问题。但有些场景，文字回复并不是最好的选项。

比如你在开车，手机收到 AI 的分析结果，你没法低头看屏幕；比如你在厨房做饭，手上油腻腻的，不方便盯着屏幕；比如你只是想换个交互方式，让 AI 助手多一点「人」的感觉。

OpenClaw 有一个不太被人注意的功能：TTS，Text-to-Speech，语音回复。

AI 的文字回复，自动转成一条语音消息，直接发给你。Telegram 里是原生语音条，WhatsApp 里一样。

这篇文章就聊聊这个功能：怎么开，怎么配，什么时候用最合适。

免费版就能用，不需要 API Key

很多人以为语音合成一定要付费，要接什么 API。

OpenClaw 的 TTS 支持三个语音服务商：

  • Microsoft（免费，内置，无需 API Key）
  • OpenAI（需要 OpenAI API Key）
  • ElevenLabs（需要 ElevenLabs API Key，音质最好）

其中 Microsoft 的方案是开箱即用的——它背后用的是 Microsoft Edge 的在线神经网络 TTS 服务，不需要你注册、不需要你付费，只要开启 TTS，它就默认启用。

音质说实话并不差。对于日常的 AI 回复，完全够用。

当然，如果你对声音品质有要求，接上 OpenAI 的 gpt-4o-mini-tts 或者 ElevenLabs 的多语言模型，效果会明显提升一个档次。

四种触发模式，按需选择

开启 TTS 之前，先想清楚你想要什么模式。OpenClaw 提供四种：

always——所有回复都转语音

AI 每次回你，都会附带一条语音消息。适合你想完全用语音交互的场景。

inbound——你发语音，AI 才回语音

只有当你给 AI 发语音消息时，它的回复才会转成语音。你发文字，它就回文字。这个模式最符合直觉——你用什么方式跟它说话，它就用什么方式回你。

tagged——只有 AI 主动标记的回复才转语音

AI 可以在回复里加一个 [[tts]] 标记，只有带这个标记的回复才会转成语音。适合高度定制的场景。

off——关闭

默认状态就是关闭的。什么都不配置的话，TTS 不会启动。

最简单的配置：一行开启

打开 openclaw.json，在 messages 下加上这段：

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(line{  messages: {    tts: {      auto: "always",    },  },}

保存，重启 Gateway。完成。

这时候 OpenClaw 会自动检测你有没有 OpenAI 或者 ElevenLabs 的 API Key：

  • 有 OpenAI Key → 用 OpenAI
  • 有 ElevenLabs Key → 用 ElevenLabs
  • 都没有 → 自动用 Microsoft 免费方案

什么都不用额外操作，它自己找到能用的方案。

想用 OpenAI 语音，这样配

OpenAI 的 TTS 模型 gpt-4o-mini-tts 性价比很高，支持多种声音风格（alloy、echo、fable、onyx、nova、shimmer），中文发音也相当自然。

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(line{  messages: {    tts: {      auto: "always",      provider: "openai",      openai: {        apiKey: "你的 OpenAI API Key",        model: "gpt-4o-mini-tts",        voice: "nova",      },    },  },}

nova 这个声音相对中性、清晰，用来播报 AI 回复很合适。你可以根据自己偏好换。

想用 ElevenLabs，音质最好

如果你对声音有讲究，ElevenLabs 是最佳选择。它的 eleven_multilingual_v2 模型在中文发音上表现出色，情感也更自然。

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(line{  messages: {    tts: {      auto: "always",      provider: "elevenlabs",      elevenlabs: {        apiKey: "你的 ElevenLabs API Key",        voiceId: "你选择的 Voice ID",        modelId: "eleven_multilingual_v2",        languageCode: "zh",        voiceSettings: {          stability: 0.5,          similarityBoost: 0.75,          speed: 1.0,        },      },    },  },}

Voice ID 在 ElevenLabs 的网站上可以找到，每个声音都有对应的 ID。

长回复怎么办？自动摘要

AI 回复有时候很长，几千个字的分析报告，让它全念出来并不现实。

OpenClaw 的 TTS 有一个自动摘要机制：当回复超过一定长度（默认 1500 字符），它会先用 AI 把这段内容压缩成一个简短摘要，然后把摘要转成语音。

所以你听到的语音，是核心要点，不是完整全文。

这个阈值可以调：

ounter(line/tts limit 2000

或者直接在配置里设 maxTextLength。

如果你不想要自动摘要，想要它忠实地把所有文字都读出来（或者在文字过长时干脆不发语音），可以关掉：

ounter(line/tts summary off

Telegram 里是真正的语音条

这里有个细节值得提一下。

OpenClaw 的 TTS 输出格式，会根据你使用的渠道自动调整。

在 Telegram、WhatsApp、WhatsApp 上，AI 的语音回复会以原生语音消息的形式发送——也就是那个带波形的语音条，点一下就播放，跟朋友发给你的语音消息完全一样的形式。

在其他渠道，则会以 MP3 文件附件的形式附上。

这意味着如果你主要在 Telegram 上用 OpenClaw，开启 TTS 之后，AI 的回复体验会跟和真人朋友聊天很接近。

用斜杠命令临时控制

不想动配置文件？用命令更快。

在跟 AI 的对话里直接发：

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(line/tts always    # 开启，所有回复都发语音/tts inbound   # 只有你发语音时才回语音/tts off       # 关掉/tts status    # 看当前状态/tts provider openai   # 切换到 OpenAI/tts audio 测试一下语音   # 发一条测试语音（不改设置）

这些命令写入的是本地偏好文件，不影响全局配置，退出后可以随时改回来。

Discord 用户注意：Discord 有自己的 /tts 命令，OpenClaw 在 Discord 里把这个命令注册成了 /voice，用法一样。

主、备语音服务商

跟模型故障转移一样，TTS 也支持主备切换。

比如配置 OpenAI 为主，ElevenLabs 为备：

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(line{  messages: {    tts: {      auto: "always",      provider: "openai",      openai: {        apiKey: "openai_key",        model: "gpt-4o-mini-tts",        voice: "nova",      },      elevenlabs: {        apiKey: "elevenlabs_key",        voiceId: "voice_id",        modelId: "eleven_multilingual_v2",      },    },  },}

OpenAI 正常就用 OpenAI，OpenAI 请求失败了自动用 ElevenLabs 兜底。两个都没有，Microsoft 兜底。三层保障。

AI 可以自己控制语音风格

这是一个比较高级的功能：AI 在回复里可以嵌入 TTS 指令，控制这条回复的语音风格。

比如 AI 回复时可以加上：

ounter(line[[tts:voice=nova speed=1.1]]

这样这条回复就会用 nova 声音、1.1 倍速播放，不影响其他回复的设置。

这个功能默认是开启的（modelOverrides.enabled: true），但允许切换 provider 需要额外开启：

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(line{  messages: {    tts: {      modelOverrides: {        enabled: true,        allowProvider: true,      },    },  },}

实际使用中，这让 AI 助手可以针对不同内容选择合适的语音风格——比如播报天气用轻快的语气，读分析报告用稳重的语气。

我的推荐配置

如果你是第一次尝试，我建议从最简单的开始：

ounter(lineounter(lineounter(lineounter(lineounter(lineounter(lineounter(line{  messages: {    tts: {      auto: "inbound",    },  },}

inbound 模式最自然——你发语音，它回语音；你发文字，它回文字。不会突然在你发文字问题的时候冒出一条语音把你吓到。

用了一段时间，觉得体验不错，再考虑换成 always 或者接入更好的语音服务商。
