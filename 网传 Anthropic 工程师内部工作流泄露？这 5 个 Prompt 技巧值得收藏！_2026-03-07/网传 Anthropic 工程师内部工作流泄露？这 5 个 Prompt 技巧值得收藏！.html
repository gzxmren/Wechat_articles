网传 Anthropic 工程师内部工作流泄露？这 5 个 Prompt 技巧值得收藏！

 


最近，外网一篇关于 Anthropic 工程师工作流的文章疯传。

虽然无法证实这是否真为内部泄露，还是高手的经验总结，但文中提到的“性能提升 3 倍”、“幻觉减少 40%”等数据着实让人眼红。

抛开争议不谈，我针对文中提到的 5 项核心技术 在 Claude 上进行了逐一实测。结论是：无论出处如何，这些技巧确实能很好用。

如果你经常觉得 AI 生成的内容差点意思，或者总是需要反复纠正，那么这篇文章值得你收藏细读。

1. 记忆注入：教会模型懂你

很多时候，AI 给出的答案没问题，但“味儿”不对。比如你想要简洁的代码，它却写了一堆注释。

核心思路： 预先加载一个上下文块，直接设定你的偏好。

实测 Prompt：

“你是我的编程助手。请记住这些偏好：我使用 Python 3.11，偏好类型提示，倾向于函数式编程，并且总是包含错误处理。确认这些偏好，并在未来的所有回复中使用它们。”

效果评价：
这招对于长期项目简直是神器。一旦设定好基调，接下来的整个会话中，你再也不用反复纠正它的默认行为。

注意：此方法受限于会话长度，如果是新开对话，需要重新注入。

2. 反向提示：让模型先提问

大多数人使用 AI 的习惯是：扔需求 -> 等结果 -> 发现不对 -> 重来。这不仅浪费时间，还消耗耐心。

核心思路： 别急着让 AI 干活，先让它搞清楚状况。

实测 Prompt：

“我需要分析客户流失数据。在提供帮助之前，请针对我的数据集、业务背景和预期结果提出 5 个澄清性问题。在掌握所有信息之前不要开始。”

效果评价：
前期花费几秒钟澄清，能避免后面几小时的推倒重来。这不仅是准确性的提升，更是效率的质变。

3. 级联约束：别想一口吃成胖子

你是不是习惯写一个巨型 Prompt，把所有要求一次性扔给模型，然后祈祷它能全盘照收？

实际上，模型处理复杂指令时容易“顾此失彼”。

核心思路： 一次只给一个任务，确认理解后，再叠加下一层指令。

实测 Prompt：

“首先，用 3 句话总结这篇文章。[等待回复]
现在，找出 3 个最薄弱的论点。[等待]
最后，针对每个弱点撰写反驳论点。”

效果评价：
与其说是考验模型能力，不如说是优化交互结构。对于多阶段代码重构或复杂文档撰写，这种“步步为营”的方式能确保每一步输出都精准可控。

4. 角色叠加：三个臭皮匠，顶个诸葛亮

我们习惯了给 AI 设定单一角色（如“你是一个产品经理”）。但在复杂决策中，单一视角往往存在盲区。

核心思路： 在同一个 Prompt 中叠加多个专家视角，强制模型进行“左右互搏”。

实测 Prompt：

“同时从三个视角分析这项营销策略：关注病毒式传播的增长黑客、关注指标的数据分析师，以及关注用户动机的行为心理学家。展示所有三个观点。”

效果评价：
增长黑客可能只看传播速度，而数据分析师会立刻标记指标漏洞。单一输出包含多维视角，能直接暴露那些被忽略的潜在冲突。

5. 验证循环：让模型自我纠错

如果你正在处理支付逻辑、核心算法等高风险内容，AI 偶尔的“一本正经胡说八道”可能是致命的。

核心思路： 生成内容只是第一步，让模型自己“找茬”才是关键。

实测 Prompt：

“编写一个处理用户支付的 Python 函数。编写完成后，找出代码中的 3 个潜在错误或边缘情况。然后重写该函数以修复这些问题。”

效果评价：
这相当于强制模型执行了两轮操作：第一轮生成，第二轮批判性审查。

虽然耗时稍长，但对于正确性优于速度的场景（如支付处理、数据管道），多出的这一步自我纠错，能帮你省下巨大的 Debug 成本。

总结与建议

这 5 个技巧看似简单，但本质上都在解决一个问题：如何更有效地控制大模型的注意力机制。

与其期待 AI 突然“变聪明”，不如优化我们的交互逻辑。建议大家在实际工作中尝试以下组合：

  1. 1. 日常开发： 使用“记忆注入”，减少重复沟通成本。
  2. 2. 复杂任务： 采用“约束级联”或“反向提示”，确保方向正确。
  3. 3. 高风险输出： 必须开启“验证循环”，让 AI 自己给自己做 Code Review。

技术的红利，往往属于那些更会使用工具的人。
