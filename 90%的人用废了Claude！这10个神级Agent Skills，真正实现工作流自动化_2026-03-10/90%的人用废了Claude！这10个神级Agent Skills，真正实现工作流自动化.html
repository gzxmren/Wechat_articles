Skills 到底是什么？？？

💡 一句话解释：

把你的流程性知识，变成可复用的能力包，让 Agent 随叫随到，稳定发挥。

Skills 本质就是一个文件夹。文件夹里装的东西很清楚，如果你是职场人士：

📄SKILL.md相当于告诉你这个岗位是做什么的
🛠️scripts/是常用到的工具软件
📚references/是业务流程涉及的知识库
📁assets/是工作模板库

把这些东西打包放好后，让 Claude 干活的时候自己去查、去用、去执行。

你不用每次开新对话都从头交代一遍“我要什么格式”“我的风格”“按什么流程处理”等等。只需要调用这个 skills 就行。

✨ 容易被忽略的好处：省 Token ✨

以往，在大模型的交互中，对话越长，模型越笨。且越耗费 Token。Token 确实又寸土寸金（尤其像 Claude 这种）。

而 Skills 呢，把大量背景知识从对话中抽离出来。需要时按需调取，主对话窗口始终保持干净。这就间接省了不少 Token。

什么时候该用 Skills？

有一条判断标准特别好用：任何你不想一遍遍重新解释的东西，都值得写成 Skill。官方给了三类典型场景：

1. 组织级工作流 🏢品牌规范、法务合规流程、标准化文档模板。这些东西公司里每个人都要遵守，每次找 AI 帮忙都得重新说一遍。写成 Skill 就能一劳永逸。
2. 专业领域经验 🧠比如 Excel 公式套路、数据分析流程、PDF 处理方法、代码标准、安全审计清单等。类似这些是你在某个领域摸爬滚打攒下来的经验。
3. 个人偏好和习惯 🎨你的笔记结构、代码风格、研究方法论。每个人干活都有自己的一套讲究，Skill 能让 AI 记住你的偏好和风格。

怎么安装和创建 Skills？

📥 安装方式（2种）

  • 命令行直接装：
    直接告诉 Claude"帮我安装这个 skill，项目地址为 xxx"，它自己就能搞定。
  • 手动放置：
    把 Skills 文件夹拖到本地目录~/.claude/skills即可。

🛠️ 创建方法

  • 基础版（非程序员推荐）：
    直接跟 Claude 说"我要创建 skill，一步步引导我"，它会带你走完流程，最后生成 zip 包，上传安装。
  • 进阶版：
    先安装官方出的skill-creator，提出需求时让它调用这个 skill 帮你做设计，产出结构更规范稳定。

🔥 被低估的用法：把 GitHub 项目打包

（思路来自卡兹克，整个生态里最值得关注的玩法）

Skills 最正确的用法，是将整个 GitHub 项目压缩成你自己的超级技能库。原本需要读文档、装环境、跑命令的事情，变成了一句话的事。

操作步骤：

  1. 复制目标 GitHub 项目链接；
  2. 给 Claude Code 提需求：“帮我把这个开源工具打包成一个 Skill，实现xxx功能”；
  3. 让 Claude 开启计划模式做规划，再写 Skill，提升稳定性；
  4. 第四步是关键：
    每次踩坑、每次修复，都喂回给 Skill：“把这些经验更新到这个 skill 里”。

💡 它就会成为持续进化的 skills。

Skills 不只是文档，还能跑代码

很多人以为 Skills 只是一堆提示词文件，其实不是。里面还可以包含可执行脚本（比如 Python 脚本）。

解决 AI 生成代码的痛点（不稳定）：

  • 调用库不固定：今天用requests，明天可能换成axios；
  • 代码输出不统一：同一个任务每次生成的代码不同，调试成本高。

而 Skill 里的脚本是提前写好、验证过的，逻辑固定，结果可预期。大量参考资料可放在references/目录里，Claude 需要的时候自己去查，不占主对话的上下文。就像给员工配了一个资料柜。

五步框架：把工作流变成可进化的 Skill

（方法论来自宝玉AI：几乎所有能用 workflow 完成的 AI 任务，都可以用 Agent + Skills 实现。）

1拆分把工作流拆成单一职责的 skill 或 subagent。每个模块只做一件事，做好一件事。
2编排在主 skill 里用自然语言描述整个流程。不需要写代码，像给同事交代任务一样说清楚就行。
3存储所有中间结果都保存成本地文件，而不是留在内存或上下文里。
4分摊这条很重要Subagent 之间只传文件路径，不传内容。传路径的话，subagent 会自己去读文件，上下文保持得很干净。
5迭代这是 Agent + Skills 的最大优势——持续进化。它可以自己迭代 subagent 的 system prompt，实现自我进化。

🚀 核心思想：拆得开、存得住、传得轻、改得动

💡 一条最重要的认知

Skills 一定是你本身实践过或者沉淀好的工作流，只是把它自动化。

不是让 AI 帮你从零发明流程，而是把你已经验证过的流程固化下来。

好的组织，会把经验变成可复用的 skill。好的个人，也一样。
差距不在工具，在于你自己有没有值得固化的东西。

最后我安装了哪些 Skills 呢？

这是第一次尝试装 skills，探索后安装了以下 10 个，供参考：

  • 1. Skill-Vetter（安全守门人）

    • 核心功能
      ：扫描其他技能代码，检测越权行为、恶意代码和红旗标识，保护本地文件安全
    • 下载量
      ：42k + 安装，安全类第一
    • 必装理由
      ：所有技能的 "基础防线"，必须先装它再装其他技能，开源工具安全第一
  • 2. Tavily-Search（实时联网神器）
    • 核心功能
      ：赋予 AI 联网搜索实时信息的能力，支持时间过滤、深度检索
    • 下载量
      ：37k + 安装，AI Agent 标配
    • 必装理由
      ：解决大模型知识滞后问题，没有它 AI 就是 "井底之蛙"，查不了实时信息
  • 3. Find-Skills（技能导航员）
    • 核心功能
      ：自动识别缺失功能，提示安装对应 Skill，相当于 OpenClaw 的 "应用商店"
    • 下载量
      ：41k + 安装，新手必备
    • 必装理由
      ：不用记所有技能名，AI 会自己 "采购" 所需功能，极大降低学习门槛
  • 4. Office-Automation（办公全能王）
    • 核心功能
      ：一键处理 Word、Excel、PPT，数据汇总、格式转换、周报生成
    • 下载量
      ：39k + 安装，办公党最爱
    • 必装理由
      ：10 分钟完成人工 1 小时工作量，彻底告别重复繁琐的办公操作
  • 5.File-Organizer（本地文件管家）
    • 核心功能
      ：自动分类桌面 / 下载文件夹，批量重命名，清理重复文件，支持所有格式
    • 下载量
      ：36k + 安装，高频使用工具
    • 必装理由
      ：解决电脑文件混乱痛点，一句话指令让桌面 / 文件夹清爽如新
  • 6. Self-Improving-Agent（AI 自进化技能）
