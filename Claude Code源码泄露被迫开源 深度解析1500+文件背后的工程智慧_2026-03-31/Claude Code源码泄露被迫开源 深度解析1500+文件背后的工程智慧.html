🔑 关键技术点1：自研 Ink 渲染引擎

这是最让人惊喜的部分——**Claude Code 自研了一套终端渲染引擎！**

**路径：** `src/ink/`

**它是什么？**

- 类似 React DOM 的**终端渲染引擎**

- 用于替代传统的 ncurses

- 支持 **yoga-layout**（Facebook 的 Flexbox 布局库）

- 组件化开发 terminal UI

**这意味着什么？**

- 终端 UI 也能用 Flexbox 布局了！

- 像写 Web React 一样写终端界面

- 布局能力甩 ncurses 十条街

这技术，真的太超前了！

---

🔑 关键技术点2：工具系统，30+种工具统一管理

**路径：** `src/tools/`

**核心工具包括：**

- `BashTool` - Bash 命令执行

- `PowerShellTool` - PowerShell 支持

- `FileEditTool` - 文件编辑

- `FileReadTool` - 文件读取

- `AgentTool` - Agent 调用

- `MCPTool` - MCP 协议支持

- `WebFetchTool` - 网页抓取

- `WebSearchTool` - 网络搜索

**每个工具的结构都超级统一：**

- `UI.tsx` - 工具的 UI 组件

- `prompt.ts` - 工具的提示词

- `constants.ts` - 工具的常量

这种设计，太优雅了！

---

🔑 关键技术点3：Agent 系统，内置5个智能体

**路径：** `src/tools/AgentTool/`

**内置 Agent 包括：**

1. `claudeCodeGuideAgent` - Claude Code 向导

2. `exploreAgent` - 探索代理

3. `generalPurposeAgent` - 通用代理

4. `planAgent` - 计划代理

5. `verificationAgent` - 验证代理

**Coordinator 模式：** `src/coordinator/coordinatorMode.ts`

- 多 Agent 协作调度

- 智能分配任务

- 结果汇总验证

这才是真正的**多 Agent 系统**！

---

🔑 关键技术点4-8：更多黑科技

4. 权限系统

**路径：** `src/components/permissions/`

- 路径验证：`src/utils/permissions/filesystem.ts`

- 支持规则引擎和 shell 命令分类

- 文件/Shell 操作需要用户授权

5. 记忆系统

- `src/memdir/` - 记忆目录管理

- `src/services/SessionMemory/` - 会话记忆

- `src/services/teamMemorySync/` - 团队记忆同步

- `src/services/compact/` - 记忆压缩（autoCompact）

6. MCP（Model Context Protocol）

**路径：** `src/services/mcp/`

- 支持 stdio 和远程 MCP 服务器

- 内置 MCP 服务：`claudeai.ts`

7. 桥接系统（Bridge）

**路径：** `src/bridge/`

- Remote Session 管理

- JWT 认证、WebSocket 通信

- Teleport：远程会话恢复功能

8. 特性开关（Feature Flags）

- 通过 `feature('NAME')` 从 `bun:bundle` 读取

- 已知特性：`KAIROS`, `NATIVE_CLIENT_ATTESTATION`, `tengu_scratch`

- 支持 GrowthBook A/B 测试集成

🔐 安全特性：做得相当到位

Claude Code 在安全方面也下了不少功夫：

1. Attribution Header

请求携带 `x-anthropic-billing-header`，含版本指纹和入口点

2. 原生证明

`NATIVE_CLIENT_ATTESTATION` 特性使用 Bun 原生 HTTP stack 注入 attestation token

3. 权限系统

文件/Shell 操作需要用户授权

4. 加密存储

`src/utils/secureStorage/`
