Claude Code 模型扩容：薅Antigravity的模型
前几天讨论群里有人说可以去海鲜市场买个Google AI Pro账号，然后使用Antigravity Tools进行api反代，就可以在claude code 中使用Antigravity里提供的claude-opus4.5模型。
了解了一下，感觉很有性价比，于是花了几十块钱买了个学生版pro账号，验证了一下。今天来分享一下这个过程。

准备工作

1. 请提前安装好 claude code,可参考：GLM-4.7出现以后，终于开始使用claude code。
2. 用买来的学生pro账号在Antigravity进行登入，可参考：不用TUN模式也能方便使用Antigravity的方法，这一步主要是验证一下账号能否正常使用。

安装Antigravity Tools

1. 下载地址：
https://github.com/lbjlaq/Antigravity-Manager/releases
如果是 Windows 电脑：下载以 .exe 结尾的文件。
如果是 Mac 电脑：下载 .dmg 结尾的文件。
2. 安装完Tools后，打开软件添加pro账号
3. 点击【开始OAuth授权】后，这时候会自动打开浏览器，让你登录谷歌账号，选择你的pro谷歌账号进行授权。
4. 浏览器授权成功后，返回Tools软件界面，可显示已经添加的账号

注：如果有多个账号可以继续添加，Tools支持管理多个账号。

2. 往下滑，找到下面这个Claude Code配置，点击【立即同步配置】。
点击同步后，claude code的settings.json内容变成如下：

3. 回到终端窗口，输入claude,然后回车。
如果出现下面这个界面，看到 > 指着Yes，直接回车。
4. 显示如下界面，说明claude-sonnet-4.5模型接入成功
5. 在Antigravity Tools中查看一下请求日志
点开一条日志，查看用的是什么模型：
6. 切换opus 模型，输入/model，选择opus
切换后

配置gemini-3-pro模型

使用Antigravity Tools进行claude code配置文件同步，只能使用使用claude模型，要想在claude code中使用gemini-3模型，可以自己配置，这里我直接在CC-Switch中进行配置，步骤如下：
1. 在Antigravity Tools --> API反代，下滑到自定义映射，添加一个gemini-3的映射。
2. 打开CC-Switch，新添加一个服务商

api key在Antigravity Tools --> API反代 中的API密钥；

模型填写上一步新添加的自定义映射名称；

3. 在cc-switch 切换到gemini-3 的配置
4. 重新打开claude code

ok，到现在为止，已经成功的将Antigravity中的模型接入到claude code中，Claude Code 模型再扩容，哈哈。
使用过程中偶尔会出现503错误，提示"No available accounts: All accounts limited. Wait 1248s."，有时候重启Antigravity Tools能解决，或者多买几个pro账号进行负载均衡。
