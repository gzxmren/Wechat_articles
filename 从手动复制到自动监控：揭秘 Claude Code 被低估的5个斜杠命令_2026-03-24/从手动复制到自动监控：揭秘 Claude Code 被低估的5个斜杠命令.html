从手动复制到自动监控：揭秘 Claude Code 被低估的5个斜杠命令

 


如果你还没有测试过 Claude Code 最近的几次更新，我完全理解。Anthropic 在 Claude Code 上推出新功能的速度让人难以跟上。

在过去几周，Claude Code 就迎来了新的斜杠命令，它们改变了投入程度、自动化任务、复制代码，甚至为会话着色的方式。

在本文中，我将带你了解最新 Claude Code 版本中的每一个新斜杠命令 —— 每个命令的作用、使用时机以及如何融入你的工作流。

以下是我们将涵盖的内容：

  • • /effort ：控制 Claude 对每项任务的思考深度
  • • /loop ：按固定间隔运行提示词
  • • /copy ： 从回复中挑选并复制特定代码块
  • • /color ： 为每个会话的提示栏着色
  • • /simplify 和 /batch ： 针对常见工作流的打包命令
  • • /reload-plugins ： 无需重启即可热重载插件更改

让我们开始吧。

/effort — 控制 Claude 的思考力度

这个命令非常适合管理速度和成本。

在 /effort 出现之前，你必须打开 /model 菜单并使用方向键调整投入程度滑块。

这虽然可行，但它隐藏在一个同样处理模型切换的菜单中。

现在你只需输入：

/effort

你会看到三个等级 —— 低、中、高。每个等级由简单的符号表示：

  • • ○ 低 —— 跳过深度推理。快速且廉价。
  • • ◐ 中 —— 平衡。适合大多数编码任务。
  • • ● 高 —— 完全推理能力。最适合复杂问题。

如果你想重置为默认值，运行：

/effort auto

这让 Claude 根据任务复杂度决定投入程度。

Opus 4.6 默认为中等投入程度。这是大多数工作的最佳平衡点。

但是，当你调试棘手的多文件问题时，你需要高投入程度。而当你重命名变量或格式化代码时，低投入程度可以节省 Token 和时间。

/effort 现在可以在 Claude 回复时工作。因此，如果你在回复中途意识到 Claude 需要在下一轮思考得更深入，你可以无需等待直接切换。

以前，你也可以在提示词中输入“ultrathink”来触发一轮高投入程度。

这仍然有效，但 /effort 让你在整个会话中拥有持久的控制权。

使用场景

  • • 低： 简单编辑、快速提问、格式化、重命名
  • • 中： 日常编码、标准功能、常规任务
  • • 高： 架构决策、复杂调试、多文件重构、代码审查

/loop — 自动化循环提示词

/loop 允许你按固定间隔运行提示词或斜杠命令。你设定时间，给它一个任务，Claude 就会自动重复执行。

/loop 5m check the deploy

这告诉 Claude 每 5 分钟检查一次部署状态。

你可以将其与任何提示词或斜杠命令搭配使用：

/loop 10m /stats

这会每 10 分钟运行一次 /stats，以便你在长时间会话中跟踪 Token 使用情况。

使用场景

这对于以下任务很有用：

  • • 在长时间编码会话中监控正在进行的部署
  • • 更改后检查测试结果
  • • 监控 CI 流水线中的构建失败
  • • 对项目运行定期健康检查

在幕后，/loop 利用了同一版本中添加的 cron 调度工具。

Claude Code 现在可以在会话内处理循环提示词，无需外部 cron 作业或脚本。

如果你需要停止循环，可以设置环境变量：

CLAUDE_CODE_DISABLE_CRON=1

这会终止会话中所有预定的 cron 作业。

快速提示：

/loop 最适合轻量级的只读检查。在循环中运行繁重的代码生成会迅速消耗你的 Token。将其用于监控和状态检查；而不是用于重复构建或重构。

/copy — 挑选你想复制的内容

在 /copy 出现之前，从 Claude Code 回复中复制代码并不容易。

你要么向上滚动并手动选择所需的块，要么复制整个回复并自行修剪。/copy 通过交互式选择器解决了这个问题。

当 Claude 的回复包含代码块时，输入 /copy 会弹出一个选择菜单。

你会看到每个代码块单独列出，然后选择你需要的那个。

/copy

你会看到如下选项：

  • • 代码块 1（React 组件）
  • • 代码块 2（API 路由）
  • • 代码块 3（测试文件）
  • • 完整回复

选择一个，它就会被复制到剪贴板。

通过 SSH 写入文件

如果你通过 SSH 工作，剪贴板访问受限，/copy 非常方便。

在选中的项目上按 w，它会直接写入文件，绕过剪贴板。

这是一个小细节，但如果你曾在 tmux 或远程会话中遇到过剪贴板问题，你会感激这个功能。

“始终复制完整回复”模式

如果你发现自己总是复制完整回复而不是单个块，这里有一个快捷方式。

从选择器中选择“始终复制完整回复”，未来的 /copy 命令将完全跳过代码块菜单。它每次都会复制完整回复。

你可以随时通过选择器重置此行为。

使用场景

  • • 从多文件回复中获取特定函数
  • • 复制单个配置块而无需周围解释
  • • 通过 SSH 工作，手动选择不可靠
  • • 快速提取代码片段以粘贴到另一个文件或工具中

/color — 为你的会话着色

这个命令很简单，但在运行多个 Claude Code 会话时非常有用。

/color

它为当前会话的提示栏设置颜色。

如果你运行三个终端:一个用于前端，一个用于后端，一个用于基础设施, 可以为每个终端分配不同的颜色。

一眼就能知道你正在哪个会话中输入。

重置颜色

如果你想恢复默认值，以下任一命令均可：

/color default
/color gray
/color reset
/color none

它们都能恢复原始提示栏。

与 /rename 配合使用

/color 与 /rename 配合使用效果很好。

会话名称现在显示在提示栏上，因此将颜色与描述性名称（如“auth-service”或“frontend-v2”）结合使用，使会话管理变得毫不费力。

/simplify 运行并行代理审查代码

/simplify 仍然是 Claude Code 中最被低估的命令之一。它是一个实现后清理工具。你构建功能，让它运行，然后运行：

/simplify

它检查 git diff 以识别更改，然后并行启动三个专门的代理：

  • • 代码复用代理 — 查找重复逻辑并合并
  • • 代码质量代理 — 发现代码异味并强制执行 CLAUDE.md 规则
  • • 效率代理 — 标记性能问题和结构性改进

这三个代理同时运行。

当它们完成时，/simplify 会汇总发现并在一次通过中应用修复。

你也可以将其集中在特定领域：

/simplify focus on error handling

/simplify 在 90 秒内将 69 行混乱的代码减少到 33 行。它捕获了原始 SQL 字符串拼接，将 CommonJS 迁移到 ES 模块，用数组方法替换了嵌套循环，并删除了死代码；所有这些都基于 CLAUDE.md 中的定义。

/batch — 大规模并行迁移

如果 /simplify 是你的代码质量检查关，那么 /batch 就是你的迁移引擎。

它处理的工作通常意味着要在数十个文件中进行漫长的查找和替换，希望没有遗漏任何东西。

/batch migrate src/ from Jest to Vitest
/batch replace all uses of lodash with native equivalents
/batch add type annotations to all untyped function parameters
  • • 第一阶段 —— 研究与计划。 /batch 首先进入计划模式。它探索整个目录，扫描每个文件，并生成一个迁移表，显示哪些文件需要更改，哪些已经合规。然后它展示完整计划，在编写一行代码之前等待你的批准。
  • • 第二阶段 —— 并行执行。 一旦批准，它将工作分解为独立单元，并在隔离的 git worktree 中生成代理。每个代理在自己的代码库副本上工作。
  • • 第三阶段 —— 验证与 PR。 每个工作程序在进行更改后运行验证检查。它知道真正的失败与预期错误（如演示环境中缺少包）之间的区别。工作程序在创建 PR 之前也会运行 /simplify，因此每个拉取请求都是经过预审查的。

batch和simplify的区别

/simplify 是对你最近更改的质量检查 —— 在构建功能后运行它。/batch 用于大规模迁移 —— 当你需要在数十个文件中应用某种模式时运行它。

/batch 先思考，向你展示计划，然后等待。最后一部分很重要。在它接触任何东西之前，你要审查分解方案。

/reload-plugins 无需重启即可热重载插件更改

如果你使用 Claude Code 插件，你就知道其中的痛苦。

你更新了插件，但在重启会话之前什么也不会发生。这意味着丢失上下文、重新建立工作状态以及等待所有内容重新加载。

/reload-plugins 解决了这个问题。

/reload-plugins

它激活挂起的插件更改而无需重启会话。编辑你的插件代码，运行 /reload-plugins，更改立即生效。

总结

Anthropic 发布 Claude Code 更新的速度令人难以匹敌。

这些斜杠命令不是作为一个大版本发布的，而是在几周内分散在多个版本中。

我最常用的是 /effort 和 /copy。动态切换投入程度已成为第二天性，快速编辑用低投入，需要 Claude 推理复杂问题时用高投入。

我不常使用 /loop，但在需要时非常强大。在长时间会话中监控部署或跟踪 Token 使用情况而无需切换终端，是一个很好的工作流改进。

/color 和 /rename 一起使用清理了我的多会话工作流。
