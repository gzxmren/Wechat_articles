💀 为啥这事这么严重？（真实风险了解一下）

Clawdbot 确实香。它是咱们盼了好久的个人 AI 助手——自己托管、24小时在线、连着你的消息、文件和各种工具。

但大部分人没意识到：网关本来是为本地使用设计的，从来没打算让它暴露到互联网上！

当你在服务器或 VPS 上跑 Clawdbot 却没配好设置时，相当于大敞着你家前门，还挂个牌子写着"API 密钥在里面，欢迎光临"。

实际会暴露啥？

安全研究员扫描易受攻击的 Clawdbot 实例时，发现了这些：

  • • 📱 完整的聊天记录——Telegram、WhatsApp、Signal、iMessage 全都有
  • • 🔑 Claude、OpenAI 等 AI 服务商的 API 密钥
  • • 🔐 OAuth 令牌和机器人凭证
  • • 💻 宿主机的完整 shell 访问权限

一位安全研究员现场演示：用简单的提示词注入攻击，不到 5 分钟就从被入侵的系统里提取出了私钥！

🔍 5 分钟快速自查

在动手修复之前，先看看你是不是已经暴露了。

检查 1：你的网关在监听哪个地址？

打开终端（Mac 上 Spotlight 搜 "Terminal"；Windows 搜 "Command Prompt" 或 "PowerShell"），运行：

clawdbot gateway status

找 "bind" 设置：

状态
含义
✅ bind=loopback (127.0.0.1)
安全
——只绑定本地
❌ bind=lan 或 bind=0.0.0.0
危险
——监听所有网络，赶紧修！

用大白话解释

  • • 127.0.0.1 (loopback) = 只有你自己的电脑能连。就像在隔音房间里说话，外面听不见。
  • • 0.0.0.0 (所有接口) = 你网络上的任何人（如果端口开放，还包括整个互联网）都能连。就像拿着扩音器在大街上喊话。

检查 2：能从外面访问到你的 Clawdbot 吗？

如果你在服务器上跑 Clawdbot，试试用手机（别连 WiFi）访问：

http://你的服务器IP:18789
https://你的服务器IP:18789

专业技巧：我经常用 netcat (nc) 命令检查连通性：

nc -zv {你的服务器IP} {端口}

如果不用输密码就能看到 Clawdbot 界面，那你就暴露了！

🛡️ 15 分钟完全锁定（手把手教程）

下面是具体步骤，按顺序来。每一步都会先讲清楚它在干啥，再让你动手。

步骤 1：只绑定到本地主机（2 分钟）

这步干啥：告诉 Clawdbot 只接受来自同一台电脑的连接。这是最重要的设置！

打开配置文件：

Mac/Linux：

nano ~/.clawdbot/clawdbot.json

Windows：用记事本打开 %USERPROFILE%\.clawdbot\clawdbot.json

找到或添加这个设置：

{
  "gateway": {
    "bind": "loopback",
    "port": 18789
  }
}

保存文件（nano 里：按 Ctrl+X，然后 Y，再按 Enter）

重启网关：

clawdbot gateway restart

验证生效了没：

clawdbot gateway status
# 你应该看到 bind=loopback (127.0.0.1)

步骤 2：锁定文件权限（2 分钟）

这步干啥：确保只有你能读 Clawdbot 的配置文件。配置文件里有敏感信息，比如 API 密钥和聊天记录。

运行这些命令：

Mac/Linux：

chmod 700 ~/.clawdbot
chmod 600 ~/.clawdbot/clawdbot.json
chmod 700 ~/.clawdbot/credentials

这些数字啥意思：

  • • 700 = 只有你能访问这个文件夹
  • • 600 = 只有你能读这个文件

或者让 Clawdbot 自动帮你修：

clawdbot security audit --fix

这条命令会自动收紧敏感文件的权限。

步骤 3：禁用网络广播（1 分钟）

这步干啥：阻止 Clawdbot 向网络上的其他设备广播自己。默认情况下，Clawdbot 用 mDNS（也叫 Bonjour）帮你找设备——但这也会暴露信息。

加到你的环境变量里：

Mac/Linux，把这行加到你的 shell 配置（~/.bashrc 或 ~/.zshrc）：

export CLAWDBOT_DISABLE_BONJOUR=1

然后重新加载 shell：

source ~/.bashrc  # 或 source ~/.zshrc

Windows，设置环境变量：

  1. 1. 开始菜单搜 "Environment Variables"
  2. 2. 点 "Environment Variables"
  3. 3. 在 "User variables" 下点 "New"
  4. 4. 变量名：CLAWDBOT_DISABLE_BONJOUR
  5. 5. 变量值：1
  6. 6. 点 OK

步骤 4：运行安全审计（2 分钟）

这步干啥：扫描你的 Clawdbot 安装，找常见安全问题，告诉你该修啥。

clawdbot security audit --deep

这会检查：

  • • 文件权限
  • • 网络暴露
  • • 配置问题
  • • 已知漏洞

自动修复发现的问题：

clawdbot security audit --deep --fix

看输出结果。它会明确告诉你发现了啥、改了啥。

步骤 5：更新 Node.js（3 分钟）

这步干啥：Clawdbot 跑在 Node.js 上，旧版本有安全漏洞，攻击者能利用。

检查你的版本：

node --version

你需要 22.12.0 或更高。

如果你的版本更旧：

Mac（用 Homebrew）：

brew update && brew upgrade node

（我自己更喜欢从 nodejs.org 完整安装，不用 homebrew）

Ubuntu/Debian Linux：

curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

Windows：从 nodejs.org 下载最新安装包运行。

步骤 6：设置身份验证（3 分钟）

这步干啥：加个密码保护，这样就算有人找到你的网关，没密码也进不去。

选项 A：令牌认证

设置带安全令牌的环境变量：

export CLAWDBOT_GATEWAY_TOKEN="你的安全随机令牌"

生成安全令牌：

openssl rand -hex 32

选项 B：密码认证

在 clawdbot.json 里加：

{
  "gateway": {
    "auth": {
      "mode": "password"
    }
  }
}

然后设置密码：

export CLAWDBOT_GATEWAY_PASSWORD="你的安全密码"

改完这些记得重启网关。

🚀 更进一步：设置 Tailscale

如果你在云服务器（AWS、DigitalOcean、Hetzner 等）上跑 Clawdbot，需要额外保护。

设置 Tailscale（安全远程访问的简单方法）

Tailscale 干啥：在你的设备之间建私有加密隧道。你可以从任何地方访问 Clawdbot，不用把它暴露给公共互联网。

步骤 1：安装 Tailscale

在你的服务器上：

curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

在你的个人设备上，从 tailscale.com 下载 Tailscale，用同一个账户登录。

步骤 2：配置 Clawdbot 用 Tailscale

在 clawdbot.json 里：

{
  "gateway": {
    "bind": "loopback",
    "tailscale": {
      "mode": "serve"
    }
  }
}

现在你可以从 Tailscale 网络上的任何设备安全访问 Clawdbot 了。

🔐 SSH 加固基础

如果你通过 SSH 访问服务器：

禁用密码认证（改用 SSH 密钥）：

sudo nano /etc/ssh/sshd_config

找到并改成：

PasswordAuthentication no
PermitRootLogin no

重启 SSH：

sudo systemctl restart sshd

🧱 防火墙设置（UFW）

这步干啥：UFW（简单防火墙）是 Ubuntu 的内置防火墙。它阻止所有入站流量，除了你明确允许的——就像你服务器的门卫，只让名单上的客人进。

为啥对 Clawdbot 重要：就算你意外把 Clawdbot 配成监听所有接口，正确配置的防火墙也会阻止外部访问 18789 端口。

步骤 1：检查 UFW 装没装

sudo ufw status

如果你看到 "Status: inactive" 或命令能跑，那就 OK。如果不行，安装它：

sudo apt update && sudo apt install ufw -y

步骤 2：设置默认策略

这是基础——拒绝所有入站，允许所有出站：

sudo ufw default deny incoming
sudo ufw default allow outgoing

啥意思：

  • • deny incoming = 没人能连你的服务器，除非你允许
  • • allow outgoing = 你的服务器还能上网（更新、API 调用等）

步骤 3：允许 SSH（别跳过！）

关键：如果你正通过 SSH 连着，没允许 SSH 就启用 UFW，你会把自己锁在外面！

sudo ufw allow ssh

如果你用非标准 SSH 端口（出于安全考虑推荐）：

sudo ufw allow 2222/tcp  # 把 2222 换成你的端口

步骤 4：允许 Tailscale（如果用的话）

如果你装了 Tailscale 做安全远程访问：

sudo ufw allow in on tailscale0

这允许通过 Tailscale 隧道的所有流量，同时阻止公共互联网访问。

步骤 5：启用防火墙

sudo ufw enable

提示时输入 y。你的防火墙现在激活了。

步骤 6：验证你的规则

sudo ufw status verbose

你应该看到类似：

Status: active
Logging: on (low)
Default: deny (incoming), allow (outgoing), disabled (routed)
New profiles: skip

To                         Action      From
--                         ------      ----
22/tcp                     ALLOW IN    Anywhere
Anywhere on tailscale0     ALLOW IN    Anywhere

千万别做!!!!!!!!!!!!!

别公开允许 18789 端口：

# 千万别这么干——这会暴露你的 Clawdbot 网关
sudo ufw allow 18789
