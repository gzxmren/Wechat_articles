解决 Claude Code 官方 Web Search 搜索不到结果的问题

Claude Code 自带的 Web Search 功能有时不稳定(个人感觉主要是因为网络问题，命令行搜索的话，必须开飞机的TUN模式，否则的话，每次搜索结果都会为空，白白浪费好长时间)，可以考虑用开源的 Open Search MCP 服务器来替代。
Open Search MCP 会使用多个开源搜索引擎（百度、Bing、DuckDuckGo、CSDN、Linux.do、掘金等）进行搜索，对肉身在国内的同学来说更友好。

配置方法

因此配置思路一共分成两步：

  1. 1. 禁用官方的 Web Search 功能
  2. 2. 添加 Open Web Search的mcp

禁用官方的 Web Search 功能

修改 ~/.claude/settings.local.json，添加如下内容：

{
"xxxx":"其它配置不用动",
"permissions":{
"deny":[
"WebSearch"
]
},
}

添加 Open Web Search的mcp

  1. 1. 根据自己环境选择命令来安装 OpenWebSearch
# With environment variables (Linux/macOS)
DEFAULT_SEARCH_ENGINE=duckduckgo ENABLE_CORS=true npx open-websearch@latest

# Windows PowerShell
$env:DEFAULT_SEARCH_ENGINE="duckduckgo"; $env:ENABLE_CORS="true"; npx open-websearch@latest

# Windows CMD
set MODE=stdio && set DEFAULT_SEARCH_ENGINE=duckduckgo && npx open-websearch@latest
  1. 2. 添加 MCP 配置

如果你不会添加 MCP 服务的话，可以留言。

mcp配置如下，默认使用百度进行搜索，解决网络问题。你可以根据需要进行修改。

{
"args":[
"open-websearch@latest"
],
"command":"npx",
"env":{
"DEFAULT_SEARCH_ENGINE":"baidu",
"USE_PROXY":"true"
},
"type":"stdio"
}

更多配置参考官方文档：https://github.com/Aas-ee/open-webSearch
