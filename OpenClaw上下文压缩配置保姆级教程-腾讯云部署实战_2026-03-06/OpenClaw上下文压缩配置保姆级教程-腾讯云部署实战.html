4.2 Compaction（智能摘要）配置

配置路径：agents.defaults.compaction

这是最核心的省钱机制——当对话太长时，让 AI 自动总结之前的对话。

{  "agents": {    "defaults": {      "compaction": {        // 模式选择        // "safeguard": 只在接近上限时才压缩（保守）        // "default": 在合理阈值就主动压缩（推荐）        "mode": "default",
        // 为 AI 回复预留的 token 空间        "reserveTokens": 20000,
        // reserveTokens 的最小值（兜底）        "reserveTokensFloor": 20000,
        // 保留最近多少 token 的原始对话（不压缩）        "keepRecentTokens": 40000      }    }  }}

参数详解：

参数
含义
推荐值
调整建议
mode
压缩策略
"default"
追求省钱用 default，追求细节用 safeguard
reserveTokens
给 AI 回复留多少空间
20000
对话经常很长的可以设大一点
reserveTokensFloor
预留空间的最小值
20000
一般不需要改
keepRecentTokens
保留最近多少原始对话
40000
需要记住更多细节就设大一点

{  "agents": {    "defaults": {      "contextPruning": {        // 模式选择        // "cache-ttl": 基于缓存过期时间裁剪（推荐）        // "off": 关闭裁剪        "mode": "cache-ttl",
        // 缓存过期时间        // "5m": 5分钟（激进，最省钱）        // "1h": 1小时（平衡）        // "4h": 4小时（保守）        "ttl": "5m",
        // 保护最近 N 条 AI 回复的工具结果        "keepLastAssistants": 3,
        // 软裁剪触发阈值（上下文使用超过 30% 时）        "softTrimRatio": 0.3,
        // 硬清理触发阈值（上下文使用超过 50% 时）        "hardClearRatio": 0.5,
        // 工具白名单（这些工具的结果不会被裁剪）        "tools": {          "allow": ["bash", "read"],          "deny": ["web_search"]        }      }    }  }}

两级裁剪策略：

阶段
触发条件
会发生什么
软裁剪
上下文 > 30%
超过 4000 字的旧工具结果，只保留开头和结尾各 1500 字
硬清理
上下文 > 50%
整个旧工具结果替换为 "[旧工具结果已清理]"

⭐⭐⭐

4h
⭐⭐
⭐⭐⭐⭐
