25 万「小龙虾」公网裸奔，你的 OpenClaw 中招了吗？！

GitHub 26 万星，超越 React，史上增长最快的开源项目。

OpenClaw，中文名「小龙虾」。

这几天，一个叫「OpenClaw Exposure Watchboard」的网站在 AI 圈疯传。

打开 openclaw.allegro.earth，你会看到一个实时更新的表格，列着 250586 个暴露在公网上的 OpenClaw 实例。

IP 地址、云服务商、所在国家、是否泄露了凭证、关联了哪些已知漏洞和 APT 黑客组织。

2506 页。每页 100 条。每条都是一个活生生的「小龙虾」。

这个数字在一个月前还是 1000。


1 月 25 日，安全公司 Censys 扫描全网，发现大约 1000 个公网可达的 OpenClaw 实例。

一周后，1 月 31 日，变成了 21000。

2 月中旬，SecurityScorecard 的 STRIKE 团队出了份报告，上午写的时候还是 40000，下午发出来就变成了 135000。

到今天，Allegro 团队的扫描结果已经到了 250586。

分布在全球 82 个国家，跑在阿里云、腾讯云、百度智能云、DigitalOcean、Hetzner 这些大大小小的云服务商上。

Signal Cage 的报道说，「这个数字增长的速度，本身就是新闻。」


为什么这么多「小龙虾」在公网裸奔？

因为旧版 OpenClaw 默认绑定 0.0.0.0:18789，监听所有网络接口，包括公网。

你装上它，什么都不改，它就已经对整个互联网敞开了大门。

打开 Watchboard 翻几页你会发现，国内 IP 占比相当大。

阿里云、腾讯云、百度云，三大云厂商的实例密密麻麻。端口基本都是 18789。

大部分「龙虾」名字都是默认的「Assistant」。少数改了名的有「小七」「海小宝」「小隽」「狐尼克」「大白」「小牛马」「小财」，甚至还有一个叫「雷宗助理」。

这些名字背后，可能是中小企业或者个人开发者。他们把 OpenClaw 部署到云服务器上，开了公网端口，没加认证，然后就这么挂着了。

很多实例的 has_leaked_creds 一栏赫然写着 Leaked。

泄露了什么？API Key。

OpenClaw 的配置文件里存着你的各种 API Key 和 OAuth Token，全部明文保存。只要有人能访问到你的实例，这些东西就是现成的。

有个心大的网友直接把 OpenClaw 的配置文件提交到了 GitHub，第二天发现他的 API Key 被狂刷，账单直接爆了。

但 API Key 泄露还只是前菜。

SecurityScorecard 扫了一圈，63% 的公网暴露实例存在可利用漏洞，其中 12812 个可以被远程代码执行直接接管。

也就是说，别人一行命令就能控制你的整台机器。

53000 多个暴露实例的 IP，曾经出现在已知的数据泄露事件中。


OpenClaw 的安全漏洞多到可以单独写一篇论文。但最经典的是两个。

第一个，CVE-2026-25253，CVSS 评分 8.8。

攻击方式相当离谱。OpenClaw 的控制面板会接受 URL 里的 gatewayUrl 参数，然后自动建立 WebSocket 连接，把你的认证 Token 发过去。不需要你确认。

攻击者构造一个链接，你点一下，Token 就被偷走了。然后攻击者连上你的 OpenClaw，关掉安全策略，执行任意命令。

一个链接，全盘接管。

官方 1 月 29 日就发了补丁。但根据 STRIKE 的数据，至今还有超过 50000 个实例没安装。

第二个，ClawJacked，Oasis Security 披露，2 月 26 日修复。

你在本地跑 OpenClaw，绑定了 localhost，觉得很安全吧。

但，浏览器允许 WebSocket 连接到 localhost，而 OpenClaw 对 localhost 连接不做速率限制。

攻击者在网页里嵌一段 JS，每秒几百次暴力猜你的密码。猜到了就注册为受信任设备，完全接管。

你只需要访问一个恶意网页。绑定 localhost 也救不回来。


微软 Defender 安全研究团队的通报这样写道，OpenClaw 应该被当作「不受信任的代码执行」来对待，不适合在个人或企业工作站上运行。

翻译成人话，别用。

如果非要用，建议只在完全隔离的环境中部署，用专用的低权限凭证，只访问非敏感数据。

Meta 直接禁止员工在公司设备上使用 OpenClaw。

OpenClaw 的项目维护者也说，「如果你连命令行都搞不明白，这个项目对你来说太危险了。」


如果你正好也在养「龙虾」，现在就打开「OpenClaw Exposure Watchboard」搜一下你的 IP。

如果在列表里，立刻把 Gateway 绑定从 0.0.0.0 改成 127.0.0.1，不要监听公网。

更新到最新版本，至少 v2026.2.25 以上。

启用认证，不要裸奔。

用 Tailscale 或者 Cloudflare Tunnel 做网络隔离，不要直连公网。

运行 openclaw doctor，检查配置有没有安全隐患。

审查你装了哪些技能，卸载来路不明的。

划重点，以上这些操作，都可以让 OpenClaw 自己做。
