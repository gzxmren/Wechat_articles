NotebookLM 进化了：接入 AntiGravity 后，它能帮你干这 5 件以前根本不可能的事

你有没有这种体验：把一堆资料喂给 NotebookLM，问什么都答得头头是道，但让它"帮你做点什么"——发邮件、写代码、更新文件——它就傻了？这个缺口，最近被堵上了。

一、先把底层逻辑说清楚

三个工具，各有各的边界。搞清楚这个，后面 5 个用例才看得明白。

NotebookLM 是什么

本质是个本地 RAG 系统。你塞进去多少资料，它就知道多少事——文件之外的世界，它一概不知。更关键的是，它没有"手"：无法代你行动，只能回答。

Gemini 是什么

NotebookLM 是只读自己书架的助手，Gemini 是能说"你这本书里有这个，别处我还知道更好资料"的图书馆员。Gemini 现在加了个性化智能，可以读取你的日历、Gmail、YouTube 搜索记录，以此为上下文优化答案。另一个关键区别：在 Gemini 里你能同时对话多个 NotebookLM 笔记本，而 NotebookLM 本身一次只能开一个。

AntiGravity 补上了什么

连接一次，Token 永久自动刷新。连上之后，AntiGravity 可以调用 NotebookLM 的 29 个功能：创建笔记本、写入内容、跨本查询、导出幻灯片……这些以前都要你手动操作，现在 AI 可以代劳。

三句话总结分工：NotebookLM 是记忆，Gemini 是推理，AntiGravity 是行动。

附带说一个新功能：NotebookLM 生成的幻灯片现在可以直接编辑了。截一张你喜欢的网站设计图扔给 Gemini，提取出品牌色和排版风格，再贴进 NotebookLM 生成演示文稿——样式按你指定的来，改颜色、调布局都是一句话的事。

二、先连上：一条提示词搞定安装

打开 AntiGravity，新建一个文件夹，把下面这段提示词直接粘进去发送，它会自动完成安装、配置、浏览器授权、验证的完整流程：

Install and set up the NotebookLM MCP server for me:• Install the notebooklm-mcp-server package using uv (or pip if uv isn't available)• Add it to my MCP configuration file (usually at ~/.config/opencode/opencode.json or similar)• Run the authentication command (notebooklm-mcp-auth) and open a browser to authorize it• Verify the installation is working by listing my NotebookLM notebooksMake sure to:• Use the correct installation method for my system• Configure the MCP server properly in my config file• Guide me through the browser authentication process• Confirm everything is working at the end• You must add this here once completed: @mcp_config.json• You may need to make sure the user is signed out of their browser before doing this.

同样道理，也可以安装飞书的MCP，将 AntiGravity 生成的内容直接上传至飞书文档。

安装完成后，AntiGravity 就解锁了对 NotebookLM 的 29 项操作能力，分四类：

笔记本管理：创建、重命名、删除笔记本；获取详情和 AI 摘要；设置公开链接或邀请协作者。

内容导入：支持网页 URL、YouTube 链接、本地 PDF/文本/音频、Google Drive 文件四种方式直接添加资料源；支持 Drive 源同步保持内容最新。

深度研究：一条指令触发网络或 Drive 深度调研（快速模式约 30 秒，深度模式约 5 分钟），自动发现并批量导入相关来源。

内容生产：直接生成音频播客、视频、幻灯片、信息图、报告、思维导图、闪卡、测验、数据表，生成后可下载到本地，也可导出为 Google Docs 或 Google Sheets。

二、用例 1+2：把知识存好，让 AI 随时取用

用例 1：跨笔记本知识调取，保住上下文窗口

产品经理最熟悉这种场景：写需求文档时，要同时翻 PRD 历史版本、竞品分析报告、用户访谈记录、技术评审纪要——四个文件夹来回切，还要把关键段落手动复制进 AI 对话，上下文很快就撑满了。

换个做法：把这些文档存进各自的 NotebookLM 笔记本，AntiGravity 写需求时随时跨本调取，调取的内容不占 AntiGravity 自身的上下文窗口。竞品情报一本、用户反馈一本、技术债记录一本，精准抓取，互不干扰。

更进一步：让 AntiGravity 说"我们下季度要做用户留存，帮我在 NotebookLM 里建三个笔记本，分别收集行业最佳实践、竞品策略和我们自己的历史数据"——它自动规划方向、建好笔记本、填充深度调研内容，一个专题笔记本可以自动入库 55 个来源。前期调研这道工序，现在也外包给 AI 了。

用例 2：用 NotebookLM 当项目记忆库

做过长期项目的人都懂：项目到后期，AI 根本记不住早期的决策。你说"按之前定好的方案来"，它说"请问是哪个方案"；上周刚改过的接口格式，今天又给你按旧的写——因为上下文早就滚出去了。

解法：把项目的关键决策、需求变更记录、接口文档、已知 bug 清单全写进 NotebookLM 笔记本，当做项目的持久化外部记忆。每次新开对话，AntiGravity 去笔记本取背景，不需要你重新交代。就算项目跨越几个月，它依然记得第一天定下的架构原则。

运营和市场团队同样适用：把每次活动的投放数据、用户反馈、复盘结论存进去，下次策划新活动时，AntiGravity 自动参考历史经验，不会重蹈覆辙。

三、用例 3+4：从"知道"到"做出来"

用例 3：通用知识 × 你的具体处境 = 真正用得上的策略

笔记本里收集的是行业通用知识，但你面对的是具体的团队、具体的预算、具体的 KPI 压力。这个缝隙，是用例 3 要填的。

比如你负责一个 SaaS 产品的用户增长，笔记本里存了增长黑客、转化率优化、用户生命周期管理等研究资料。现在把这些知识交给 AntiGravity，同时告诉它你的实际处境："我们月活 3 万，主要用户是中小企业财务人员，近三个月流失率上升到 18%，预算只有 5 万/季度"——让它基于笔记本知识加上你的真实数据，给出一份90 天留存改善方案。

AntiGravity 不会只给通用建议，它会主动查补行业留存率基准数据，把"教科书方案"和你公司的具体约束条件拟合在一起。这是单独问 NotebookLM 得不到的，单独问 AI 又缺乏行业知识沉淀的支撑。


用例 4：方案变工具，一条指令

拿到留存方案之后，真正的执行经常卡在"没有顺手的工具"上——方案是 Word 文档，每周对照着看一遍、手动更新进度，坚持不了两周就荒废了。

给 AntiGravity 一条指令："把这份留存方案做成一个可以每周填写的追踪看板，要能记录各项指标的实际完成值，对比目标自动算进度。"

结果：一个可运行的 Web 看板，周活跃用户数、功能使用率、NPS 调研完成情况各自有输入框，填入实际数据后自动生成进度图，哪项落后一眼看出来。团队周会打开浏览器直接用，不需要维护 Excel。关键是看板的指标体系完全来自你的笔记本知识和你自己的业务数据，不是从模板生搬硬套的。

四、用例 5：笔记本直接变 PPT

NotebookLM 能生成幻灯片，但格式是封闭的，没法直接拿走用。用例 5 解决的是"最后一公里"。

职场里最常见的场景：周五下午老板说"下周一要做一个竞品分析的汇报"——你花了一下午把竞品资料整进 NotebookLM，NotebookLM 生成了结构清晰的演示内容，但格式是它自己的，没法直接发给同事，也没法在会议室投屏。

在 AntiGravity 里装一个自定义 Skill，说一句"把这个竞品分析笔记本转成 Google Slides"，它自动拉取内容、完成排版转换，给你一个可以直接共享的 Slides 链接，或者下载成 PPT 文件。调研到演示文稿，中间没有手动排版这道工序。

临时汇报、客户提案、周会总结——只要资料进了 NotebookLM，出 PPT 就是一句话的事。

值得记住的判断标准：

用这三个工具之前，先问自己需要的是哪一层：想检索已有资料，用 NotebookLM；想跨本推理、叠加个人上下文，用 Gemini；想让 AI 代替你行动、构建软件、调用工具，用 AntiGravity。

搞清楚分工，才不会用错工具，也才能真正把这套组合跑通。
