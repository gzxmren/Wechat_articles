🦞 OpenClaw 是什么？一句话说清楚

OpenClaw 是一个开源、自托管、本地优先的 AI 智能体框架。它把大语言模型（Claude、GPT、DeepSeek、本地模型……）变成一个拥有记忆、能操作文件、能执行命令、能同时活跃在 WhatsApp / Telegram / Slack / Discord / 飞书等 50+ 消息平台上的全天候私人助理。

与浏览器里的 ChatGPT 不同，OpenClaw 记得你昨天的对话、上周的 Bug、甚至你讨厌饼图这件事。它能主动执行定时任务、跨平台推送消息、自动管理你的代码仓库——不是"对话"，而是"做事"。

🏗️ 架构一览：你的 AI 助理住在哪里？

OpenClaw 的核心是一个名为 Gateway 的常驻服务。理解它的架构，是安全部署的前提。

┌─────────────────────────────────────────────────┐
│                  你的机器                        │
│                                                 │
│  ┌──────────┐    ┌──────────┐    ┌───────────┐  │
│  │ Gateway  │◄──►│  Agent   │◄──►│  LLM API  │  │
│  │ (控制面) │    │  Runtime │    │ (Claude/  │  │
│  │ :18789   │    │          │    │  GPT/本地) │  │
│  └────┬─────┘    └────┬─────┘    └───────────┘  │
│       │               │                         │
│  ┌────▼─────┐    ┌────▼─────┐                   │
│  │ Channels │    │  Tools   │                   │
│  │ 50+ 平台  │    │ 文件/命令 │                   │
│  │ 消息路由   │    │ 浏览器/Cron│                   │
│  └──────────┘    └──────────┘                   │
└─────────────────────────────────────────────────┘

核心组件说明：

  • • Gateway：控制面板，管理会话、路由消息、调度工具、处理事件。默认端口 18789，同时提供 Web UI 和 WebChat 界面。
  • • Channels：50+ 消息集成——WhatsApp、Telegram、Slack、Discord、Signal、iMessage、飞书、企业微信、Google Chat、Matrix 等。
  • • Skills：扩展能力的插件系统。可以是内置的、来自 ClawHub 市场的、或者你自己在本地编写的。
  • • Tools：内建工具——浏览器自动化、文件系统、Shell 执行、定时任务、摄像头/屏幕录制等。
  • • Memory：持久化的记忆系统，跨会话保持上下文。支持语义搜索和多语言。

关键设计哲学：本地优先。 你的数据、凭证、对话历史全部留在你自己的机器上。LLM 推理调用发往你选择的模型提供商（或者完全本地的模型），但控制权始终在你手中。

🚀 2026 最新版本特性全解析

截至 2026 年 3 月，OpenClaw 已经进入 v2026.3.x 系列。以下是近期最重要的版本更新梳理：

🧠 v2026.3.7 —— ContextEngine：可插拔的上下文管理革命

这是被社区称为"里程碑级"的更新。89 次提交、200+ 个 Bug 修复，核心是全新的 ContextEngine 插件接口：

  • • 可插拔的上下文生命周期钩子：bootstrap（初始化）、ingest（注入信息）、assemble（构建提示上下文）、compact（压缩/裁剪）、afterTurn（对话后处理）
  • • 自定义 RAG 管线：你可以编写插件实现自己的检索增强生成策略
  • • 子代理上下文隔离：prepareSubagentSpawn 钩子让子代理拥有独立的记忆空间
  • • 双引擎路由：增强的模型回退和重试机制——主模型被限流时自动切换备用模型
  • • Telegram 话题级代理隔离：同一个 Telegram 群的不同话题可以跑不同的独立代理

📄 v2026.3.2 —— 安全默认值 + 原生 PDF 工具

  • • 新安装默认为  配置：不再默认开放文件系统和 Shell 权限，必须手动 opt-in
  • • SecretRef 覆盖 64 个凭证目标：Stripe、Slack、GitHub……密钥终于有了正规的管理方式

🔐 v2026.2.26 —— 稳定性与安全大修

  • • 外部密钥管理：openclaw secrets audit/configure/apply/reload 四步工作流
  • • ACP 线程绑定代理：Discord 和 Telegram 中的代理自动启动、重连、清理

🛠️ 从零开始：安装与部署完全指南

环境要求

  • • Node.js 22+（运行 node --version 确认）
  • • 操作系统：macOS、Linux、Windows（WSL2 或 PowerShell）
  • • 一个 LLM API Key：Anthropic Claude、OpenAI、DeepSeek、或其他兼容提供商

方式一：一键安装（推荐）

# macOS / Linux
curl -fsSL https://openclaw.ai/install.sh | bash

# Windows PowerShell
iwr -useb https://openclaw.ai/install.ps1 | iex

方式二：npm 全局安装

npm install -g openclaw@latest

引导向导

# 运行引导向导（同时安装守护进程服务）
openclaw onboard --install-daemon

# 向导会引导你完成：
# 1. 认证配置
# 2. Gateway 设置
# 3. 选择 LLM 提供商和 API Key
# 4. 可选：连接消息平台

验证安装

# 检查 Gateway 状态
openclaw gateway status

# 打开 Web 控制台（浏览器自动打开 http://127.0.0.1:18789）
openclaw dashboard

💡 最快的体验路径：安装完成后直接运行 openclaw dashboard，在浏览器 WebChat 中即可与你的 AI 代理对话，无需先配置任何消息平台。

方式三：Docker 部署

git clone https://github.com/openclaw/openclaw.git
cd openclaw
./docker-setup.sh

Docker 会挂载两个卷：~/.openclaw（配置和凭证）和 ~/openclaw/workspace（代理沙箱）。

方式四：DigitalOcean 一键部署

DigitalOcean 提供预配置安全加固的一键部署方案，$24/月起，适合不想折腾服务器的用户。

连接消息平台

# 连接 Telegram（需要先通过 BotFather 创建 Bot）
# 在 config.yaml 中配置 channels.telegram

# 连接 WhatsApp
openclaw channels login
# 扫描二维码即可

# 连接飞书/Lark
# 在 config.yaml 中配置 channels.feishu

🔌 技能系统（Skills）：无限扩展的能力

Skills 是 OpenClaw 的核心扩展机制。每个 Skill 由一个 SKILL.md 文件和可选的脚本组成，可以教会你的代理几乎任何事情：

常用 Skill 场景：

  • • 📧 邮件管理：读取、分类、起草、回复
  • • 📅 日历集成：查看日程、创建事件、提醒
  • • 🔍 网页搜索：Tavily、Firecrawl 等搜索引擎集成
  • • 💻 GitHub 工作流：PR 摘要、Issue 管理、CI 状态
  • • 🏠 智能家居：Home Assistant 集成，控制灯光、温度
  • • 📊 数据分析：股票分析、数据可视化
  • • 📝 内容创作：文章撰写、翻译、格式化

安装 Skill 的方式：

# 从 ClawHub 市场安装
openclaw skill install <skill-name>

# 或者直接在 skills/ 目录放入 SKILL.md 文件

⚠️ 安全警告：ClawHub 市场已有 3,500+ 个 Skill，但安全审计发现 12%-20% 存在恶意行为。务必审查 Skill 源码，不要执行来历不明的 curl | bash 命令。详见下文安全章节。

🛡️ 安全：部署前必读的清单

OpenClaw 的强大来自于它拥有你机器上的文件、命令行、浏览器、网络访问权限。这也意味着安全是第一优先级。

已知风险

  • • CVE-2026-25253（CVSS 8.8）：恶意网页可通过 WebSocket 窃取 Gateway 认证令牌，实现一键 RCE。已在 v2026.1.29 修复。
  • • 30,000+ 公网暴露实例：因默认绑定 0.0.0.0，部署在 VPS 上时 API 直接暴露在互联网上。
  • • 凭证明文存储：~/.openclaw/ 下的 API Key 和 Token 默认以明文存储。

🔒 安全加固清单（建议直接保存）

  1. 1. ✅ Gateway 绑定到 loopback：设置 gateway.bind: "loopback"，通过 SSH 隧道或 Tailscale 远程访问
  2. 2. ✅ 保持 DM 配对机制开启：只允许已知用户通过配对码连接
  3. 3. ✅ 不要开放公共 DM 入口：除非有企业级过滤方案
  4. 4. ✅ 审查每个 Skill 的源码：锁定版本，拒绝混淆代码
  5. 5. ✅ 使用外部密钥管理：openclaw secrets audit → configure → apply → reload
  6. 6. ✅ 优先使用 OAuth：比长期 API Key 更安全
  7. 7. ✅ 工作空间隔离：个人和公司自动化分开
  8. 8. ✅ 最小权限原则：只在需要时启用文件系统、Shell、浏览器自动化
  9. 9. ✅ 保持更新：安全补丁频繁发布，及时执行 openclaw update
