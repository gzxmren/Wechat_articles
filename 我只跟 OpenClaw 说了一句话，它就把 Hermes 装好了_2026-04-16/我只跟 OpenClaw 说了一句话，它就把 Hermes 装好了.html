我只跟 OpenClaw 说了一句话，它就把 Hermes 装好了
最近很火的 Hermes 被曝核心代码抄袭了中国团队的开源项目，大概情况我在昨晚的文章里说了下。

后台很多人问我什么时候出 Hermes 的安装教程，这不就来了么。

不过，这次没有纳米级操作手册，因为我觉得简单的事情不要复杂化。

所以，给你们一个全网最简单的 Hermes 安装方法。

但我也要先说明，想实现一句话安装 Hermes 的前提，是你已经安装好了 OpenClaw。

当然，不限于 OpenClaw 这种 Agent，Claude Code 或者 Codex 其实都可以。

说白了，就是让你已经有的 Agent 去帮你安装另一个 Agent。

好，那直接上方法。

第一步，打开 OpenClaw 的对话框，直接跟它说这么一句话：

仔细阅读 Hermes 官方安装文档和配置说明，严格按照里面的步骤帮我安装：https://github.com/NousResearch/hermes-agent


注意，后面跟的这个链接是 Hermes 在 Github 上的地址，里面就包含了完整安装说明。



既然人可以阅读并按照这个说明的提示完成 Hermes 安装，那 Agent 能不能做到呢？


显然可以。


先说下环境和模型，我是在自己家里的电脑上调用本地 OpenClaw 在本地安装 Hermes，模型用的是 GPT 5.4。


而且，我完成这个操作时人在北京，而我的电脑在长沙家里。


也就是说，我全程无接触并通过远程指挥 OpenClaw 的方式让它操作我的电脑完成了 Hermes 的安装。



一开始，它以为我就在电脑边，所以设计的安装方法里有一步是需要我打开电脑终端去操作的。


但是，我很快告诉他我是在远程，并且要求它自己想办法直接完成安装。



接下来，它就会自己想办法安装各种环境并完成配置。过程中，你只需要让它继续或者做一些确认性回复即可。


大概经过了三四轮回复确认和继续，它就完成了 Hermes 的基础安装。



这个过程中，它还自己完成了从 OpenClaw 迁移配置和数据的工作。


比如整个 OpenClaw 的 Skill 体系和记忆文件，以及我已经在 OpenClaw 上配好的模型设置。


第二步，为 Hermes 设置模型。


这一步同样用对话的方式完成，一开始，OpenClaw 给 Hermes 配的是阿里 Coding Plan 的模型，但我想换成中转站 Ofox 的，于是就跟它说了这句话。



因为我的 OpenClaw 模型配置文件里有完整可用的信息，Hermes 也可以直接使用，只需要让 OpenClaw 代劳处理即可。


第三步，配置新的飞书聊天机器人。


Hermes 装好了，模型也配置好了，接下来就是给 Hermes 一个独立的通讯渠道了，这里我还是用飞书。


注意，如果你和我一样是在同一台电脑上既安装 OpenClaw 也安装 Hermes，那飞书渠道要隔离开，否则他们可能会抢同一个渠道。


我的做法是这样的，直接把明确的需求告诉 OpenClaw，让它理解我是要开一个新的飞书机器人专门给 Hermes 用。



然后，你就可以去飞书开放平台注册一个新机器人了。


如果你已经注册过多个飞书机器人了，对这个流程应该很熟悉了，我现在就属于靠直觉都能配好的那种。


首先，去飞书开放平台（https://open.feishu.cn/app）新建企业自建应用。



输入机器人名称并设置头像后，创建就成功了。

然后，选择左侧菜单里的「添加应用能力」，点击「机器人」模块下的「添加」按钮。


继续选择左侧菜单的「事件与回调」，再选择「事件配置」，把「使用长链接接收事件」勾上，并点击保存。


保存后，还是在这个页面，在下方事件那里点击「添加事件」。


在弹出的对话框里搜索「接收消息」并勾选接收消息的事件。


这一步操作的目的，是让我们给机器人发消息时，它能自动跟 OpenClaw 同步消息。


选择之后，系统会让你确认权限，直接点击「确认开通权限」即可。


除了这个权限之外，还需要添加几个权限。点击右侧菜单里的「权限管理」并点击「批量导入」按钮。


然后，先复制以下这段代码。

{"scopes":{    "tenant":[
      "im:message",      "im:message.p2p_msg:readonly",      "im:message.group_at_msg:readonly",      "im:message:send_as_bot",      "im:resource",
      "contact:user.base:readonly",      "im:message.group_msg",      "im:message:readonly",      "im:message:update",      "im:message:recall",      "im:message.reactions:read",
      "docx:document:readonly",      "drive:drive:readonly",      "wiki:wiki:readonly",      "bitable:app:readonly",      "task:task:read",      "contact:contact.base:readonly",      "docx:document",      "docx:document.block:convert",      "drive:drive",      "wiki:wiki",      "bitable:app",      "task:task:write"    ],    "user":[]  }}


在弹出的窗口里，用上面这段已复制好的代码替换掉窗口里的内容，点击下一步确认新增权限。


看到这些新增权限后，点击申请开通。


最后，选择左侧菜单的「版本管理与发布」，然后点击右侧的「创建版本」按钮。


在详情信息里填写必要内容，然后完成创建。


到这一步，飞书机器人就搭建好了。


接下来，直接把飞书的 AppID 和 Secret 复制一下发给 OpenClaw，让它去配置 Hermes 的飞书通道。


配置成功后，再去给 Hermes 配置的飞书机器人那里说句话，此时会收到一条这样的回复。

这里再说一下，唐指导是我的 OpenClaw，而唐小马是我的 Hermes。


按照过去配置 OpenClaw 的方法，我们需要复制下面那句命令去终端里执行。

但因为我的电脑不在手边，所以我依旧让 OpenClaw 帮我去远程操作。

方式也很简单，把现象告诉他即可。


如果配置完成后你继续和 Hermes 对话时还没有正常回复你，可能它会说这么一段话。


别着急，直接在 Hermes 机器人这里回复 /sethome 就行。

当它开始和你正常对话时，恭喜你，说明你的 Hermes 已经成功上线了。


总结来说，通过 OpenClaw 安装 Hermes 就三步。

第一步，安装 Hermes 源代码。

第二步，设置 AI 大模型。

第三步，配置飞书聊天机器人。

就这样，我在外面边带娃边拿着手机指挥，人在北京，却指挥长沙家里的龙虾完成了 Hermes 的安装。

这个过程，相比起我按照官方教程自己去安装折腾，至少省下来一两个小时。

通过这个案例或许你就知道了，下次要是再出来一个什么 Agent，你就可以用老 Agent 去装新 Agent 了，再也不用折腾各种复杂的技术配置和操作了。
