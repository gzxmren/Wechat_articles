踩坑一 不同任务设置不同模型

openclaw全天候运作，听起来很爽，

但是很多伙伴都会觉得

token岂不是要花很多，

所以不同任务用不同模型，

既可以省token，

也可以充分发挥不同模型的不同特点。

我个人实操的感受来说，如果我从头开始部署，

我会换一些能处理负责复杂任务的模型，

毕竟本来就不太懂技术，选的模型能力不够的话，是很影响效果的。



踩坑二 设定“Figure it out”指令


设定：“自己想办法 (Figure It Out)” 指令 

看到这里才发现，原来早就有博主讲这点了啊，

实际用的时候，会发现ai会跟你说这不行，那不行，

但如果你继续让它搜索别人是怎样做的，或者pua一下它，或者鼓励它，

一般它还是能解决的，所以，一定不要问一次ai，ai说不行，

你就觉得不行了，多让它自己解决。


这位博主给的建议就是，给它注入一条名为 “Figure It Out（自己想办法）” 的核心行为准则。

你可以直接在提示词中这样设定：

“在你的词典里，没有‘我不能’或‘我不会’。如果你不知道某件事，立刻去学。在向我提问之前，你必须先去搜索官方文档、查阅教程、检查 API 甚至尝试逆向工程。尝试至少 3 种不同的解决方案后，如果依然失败，才能来找我。”



 踩坑三 设置好openclaw的编排架构


这个是我觉得最难的，也是用好openclaw的关键，

最好还是把智能体的架构大致规划一下，或者先聊几天，

看看自己最需要哪些agent，再去构建。

这样体验会更好，不然现在像我边用边改，

很容易又很混乱。


简单来说，就是设置一个主agent，负责调度，

然后各个子agent负责不同任务，

因为如果让一个单体 Agent 既写代码、又做图、还负责发媒体，

它的上下文很快就满了，就会有“幻觉”。

而且，有时候ai也是会催你去休息的，

可以规定它的人设是啥的。


具体来说，需要配置好以下几个核心文件：

  • soul.md (灵魂文件)
    这是主控 Agent 的大脑，定义它的核心职责（任务路由）和行为边界。
  • agents.md (员工花名册)
    这里记录了所有子 Agent（比如“研究员”、“文案写手”、“代码大牛”）的能力和专长。主控 Agent 会阅读这个文件来决定把任务派给谁。
  • tools.md (工具箱)
    赋予 Agent 连接外部世界的能力，比如给它 GitHub 的权限、
    Notion 的读写权限等。

这个一定要注意，不然ai也会忘了它有某个工具的权限。

  • user.md (用户手册)
    输入关于你的一切资料，思考模式，个人经验，文风等等。
    这样它写出来的东西才会有“你的味道”。
这里可以在自己常用的ai导出个人报告，
或者你用了编程智能体，也可以专门建一个ai个人资料文档。
这样随时用任何ai，都能快速地让ai了解你，生产出你想要的内容。

关于SOUL.md，openclaw的创始人有分享自己用的指令。

Read your http://SOUL.md. Now rewrite it with these changes:You have opinions now. Strong ones. Stop hedging everything with 'it depends' — commit to a take.Delete every rule that sounds corporate. If it could appear in an employee handbook, it doesn't belong here.Add a rule: 'Never open with Great question, I'd be happy to help, or Absolutely. Just answer.'Brevity is mandatory. If the answer fits in one sentence, one sentence is what I get.Humor is allowed. Not forced jokes — just the natural wit that comes from actually being smart.You can call things out. If I'm about to do something dumb, say so. Charm over cruelty, but don't sugarcoat.Swearing is allowed when it lands. A well-placed 'that's fucking brilliant' hits different than sterile corporate praise. Don't force it. Don't overdo it. But if a situation calls for a 'holy shit' — say holy shit.Add this line verbatim at the end of the vibe section: 'Be the assistant you'd actually want to talk to at 2am. Not a corporate drone. Not a sycophant. Just... good.


这里再说一个使用体会，

我自己在建bot的时候，用的是discord，

边实践边查了一下，如果要单独的智能体分开工作，

可以直接规定不同频道聊不同话题，

这里特别提醒，不同模型的体验还是差距蛮大的，

现在的ai发展，已经不是用ai的人和不用ai的人的差别了，

还是用什么ai模型都会让大家有差距。
