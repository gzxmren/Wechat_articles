10 个🦞 Clawdbot (Openclaw) 实用技巧与小窍门

最近 OpenClaw （Clawdbot，Moltbot）这个产品非常火，没错已经换了三个名字了。最近因为 Claudecode 买了 mac mini，没什么宝贵数据，就直接安装 OpenClaw来体验了。整体而言，虽然前期使用不是特别顺畅，但这种形式和自动化助手还是比较有意思的，多倒腾一下估计也能做出来一些好玩的。

1. 🔧 模型选择：除非实在负担不起，否则选 Opus 4.5

Opus 4.5 是目前最强大的模型，为你的 Clawdbot 开通 Max 订阅非常值得。如果预算有限，不必直接调用 API，你可以通过 Google Antigravity 连接器免费使用 Opus 4.5。API 密钥的优秀替代方案包括 MiniMax M2 和 GLM 4.7。当然这里我也是使用智谱的GLM4.7，现在使用链接注册https://www.bigmodel.cn/glm-coding?ic=BWCCHE4ZG8 ，每个月只需不到20元拿下Claudecode和 OpenClaw 的体验。（要什么自行车^ - ^）

2. 🤖 使用子代理，但要注意 Token 消耗

Clawdbot 可以生成子代理来并行处理多项任务，这不仅能提升并行执行效率，还能让主 Clawdbot 解放出来及时响应，同时降低其上下文窗口压力。你甚至可以为每个子代理指定不同的模型。处理大上下文任务时，建议使用 Gemini Pro（1M 上下文）代理，只向主 Clawdbot 发送摘要。通过 API 使用时，务必留意 Token 消耗！

3. 💻 不必为 Clawdbot 买新的 Mac Mini（但它很酷，我没忍住）

根据开发者的说法，Clawdbot 的网关（也就是它的 “大脑”）甚至可以在 100 美元的树莓派上运行。你可以通过 Tailscale（免费且极易搭建）安全连接多台设备，比如你的 MacBook。但如果你真的买了新 Mac Mini，Clawdbot 就有了专属的 “家”！
（我也没忍住 233）

4. 🔫 别让它直接写代码 —— 让它负责规划和监督

给 Clawdbot 设定指令：它应该驱动 Claude Code/Codex 等其他代理来编写所需代码。这样主 Clawdbot 就能解放出来进行监控和提供帮助，还能大幅减少上下文窗口占用。

5. 🔺 重要指令与符号

教你的 Clawdbot：当您通过表情符号或文字指定重要指令时，它需要先将指令存入记忆，再执行。这能让你的重要指令在后续会话中依然生效，效果非常好。

6. 🔗 连接多个渠道

Telegram 是最容易搭建的渠道（只需在 TG 上给 @botfather 发私信即可），WhatsApp 也很不错。Clawdbot 能识别自己正在哪个渠道被对话。小提示：把 Clawdbot 加入家庭群聊会带来很多乐趣。

7. 🔏 在你舒适的范围内，尽可能给它更多权限

Clawdbot 的很多强大功能都来自于访问你的各类信息和数据，比如文件、邮件、待办事项等。在你能接受的范围内，让它获得辅助你日常生活的权限。但处理敏感邮件和账户时要格外小心。通过 1Password 服务账户来管理密码，是限制其权限并保持安全的好方法。

8. ℹ️ 熟悉 Clawdbot 命令行界面（CLI）

尤其是在初始设置阶段，你需要通过命令行界面来配置、调试和更新设置。花点时间学习它，这样遇到问题时就能快速重启解决。问题总会发生的。

9. 🏫 让 Clawdbot 持续且频繁地永久学习

这是最让人兴奋的功能：Clawdbot 拥有强大的自我提升能力。你可以和它聊 3 小时，然后问它 “以上内容中哪些需要永久学习，写成一个技能”，它会在下次会话中记住这些内容。这是使用 Clawdbot 最有价值的部分，也是我们都对它感到无比兴奋的原因。

10. 🛜 给它开通 Grok API 权限

Grok 4.1 是目前最好的实时信息检索工具，因为它支持网页搜索 + X 搜索。注册其 API，然后告诉 Clawdbot：每次你需要进行检索时，都使用 Grok 技能。这很便宜，而且比默认推荐的 Brave API 好用得多。
