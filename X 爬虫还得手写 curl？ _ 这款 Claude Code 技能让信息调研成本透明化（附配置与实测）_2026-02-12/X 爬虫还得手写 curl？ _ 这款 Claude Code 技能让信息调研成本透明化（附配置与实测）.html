X 爬虫还得手写 curl？ | 这款 Claude Code 技能让信息调研成本透明化（附配置与实测）


 

痛点与破局

在开发 AI Agent 时，你是否遇到过这样的场景：为了让 Claude 读懂最新的市场动态或科技趋势，不得不反复编写繁琐的 curl 命令去调用 X API？更令人头大的是，面对 X API 复杂的计费规则，往往不知道下一次搜索会扣掉多少预算，月底账单直呼肉疼。

x-research 就是为了解决这些痛点而生。它是一款专为 Claude Code 和 OpenClaw 打造的 X/Twitter 研究代理技能。它将复杂的 X API 封装为一组极速的 CLI 命令，让你（或你的 Agent）可以在终端中直接搜索推文、拉取完整讨论串、监控关键账户，并且每一笔 API 调用的成本都清晰可见。

核心功能深挖

x-research 不仅仅是一个 API 包装器，它在细节上做了大量优化，专门针对“研究”这一场景。

1. 极度精细的搜索过滤

普通的搜索往往充斥着垃圾信息。x-research 提供了多维度的排序与过滤功能，让你能瞬间找到高价值内容。

核心参数一览：

--sort likes|impressions|retweets|recent   (default: likes)
--since 1h|3h|12h|1d|7d     Time filter (default: last 7 days)
--min-likes N              Filter minimum likes
--min-impressions N        Filter minimum impressions
--pages N                  Pages to fetch, 1-5 (default: 1, 100 tweets/page)
--limit N                  Results to display (default: 15)
--quality                  Pre-filter low-engagement tweets (min_faves:10)

特别是 --quality 标志，它会在获取数据后自动过滤掉点赞数低于 10 的低质量推文，节省你的阅读时间。

2. 极具性价比的 Quick Mode

当你只需要快速了解某个话题的“脉搏”时，标准搜索可能略显浪费。x-research 引入了 --quick 模式，专门为低成本、高效率的查找设计。

Quick Mode 的机制：

  • 强制单页获取（最多 10 条结果），大幅减少 API 读取量
  • 自动追加 -is:retweet -is:reply 过滤器（除非你显式使用了这些操作符）
  • 使用 1 小时缓存 TTL（默认为 15 分钟），重复查询直接命中缓存
  • 搜索结束后显示详细成本摘要

3. 成本透明化

API 费用不再是个黑盒。x-research 内置了成本计算逻辑，并在每次操作后反馈消耗。

X API 资源单价参考（截至 2026 年 2 月）：

ResourceCost
Post read$0.005
User lookup$0.010
Post create$0.010

典型操作成本估算：

OperationEst. cost
Quick search (1 page, ≤100 posts)~$0.50
Standard search (1 page)~$0.50
Deep research (3 pages)~$1.50
Profile check (user + posts)~$0.51
Watchlist check (5 accounts)~$2.55
Cached repeat (any)free

省钱技巧：X API 拥有 24 小时去重机制。如果你在一天内请求相同的推文，只会被收费一次。配合 x-research 的缓存功能，重复运行相同搜索的实际成本往往远低于预估。

4. 项目架构

为了保证轻量与高效，项目结构清晰明了，核心逻辑全部由 TypeScript 编写，基于 Bun 运行时。

x-research/
├── SKILL.md              # Agent instructions (Claude reads this)
├── x-search.ts           # CLI entry point
├── lib/
│   ├── api.ts            # X API wrapper
│   ├── cache.ts          # File-based cache
│   └── format.ts         # Telegram + markdown formatters
└── data/
├── watchlist.json    # Accounts to monitor
    └── cache/            # Auto-managed

实战演示

1. 安装

无论你是使用 Claude Code 还是 OpenClaw，安装过程都只需几秒钟。

Claude Code 用户：

# From your project
mkdir -p .claude/skills
cd .claude/skills
git clone https://github.com/rohunvora/x-research-skill.git x-research

OpenClaw 用户：

# From your workspace
mkdir -p skills
cd skills
git clone https://github.com/rohunvora/x-research-skill.git x-research

2. 配置

你需要一个 X API Bearer Token（从 X Developer Portal 获取）。

export X_BEARER_TOKEN="your-token-here"

同时，请确保安装了 Bun 运行时以支持 CLI 工具。

3. 运行

配置完成后，你就可以通过自然语言与 Claude 对话，或者直接在终端使用 CLI 命令。

CLI 命令示例：

cd skills/x-research

# Search (sorted by likes, auto-filters retweets)
bun run x-search.ts search "your query" --sort likes --limit 10

# Profile — recent tweets from a user
bun run x-search.ts profile username

# Thread — full conversation
bun run x-search.ts thread TWEET_ID

# Watchlist
bun run x-search.ts watchlist add username "optional note"
bun run x-search.ts watchlist check

# Save research to file
bun run x-search.ts search "query" --save --markdown

Quick Mode 演示：

# Quick pulse check on a topic
bun run x-search.ts search "BNKR" --quick

# Quick check what someone is saying
bun run x-search.ts search "BNKR" --from voidcider --quick

# Quick quality-only results
bun run x-search.ts search "AI agents" --quality --quick

避坑指南与总结

避坑指南

  1. 安全警告：x-research 从环境变量读取 Token。请注意，AI 编码代理（如 Claude Code）可能会在会话日志中记录包含 HTTP 头部的工具调用。如果你在 Agent 会话中运行此工具，Bearer Token 可能会泄露。
    • 建议：将 X_BEARER_TOKEN 设置为系统级环境变量，避免在命令行内联；使用只读权限最小的 Token；怀疑泄露时立即轮换。
  2. 搜索范围限制：目前默认仅覆盖最近 7 天的推文（使用 /2/tweets/search/recent 端点）。全量归档搜索虽然 API 支持，但本技能暂未实现，预计会在后续版本更新。
  3. 操作符限制：由于 X API 不支持 min_likes 等搜索操作符，x-research 是在数据拉取后在本地进行过滤的，这意味着即使是过滤后的结果，API 侧依然会计费。

总结

如果你是深度 X 用户，或者正在开发需要实时社媒数据源的 AI Agent，x-research 是一个不可多得的效率工具。它不仅帮你省去了编写复杂 API 请求的时间，更通过 Quick Mode 和成本透明化设计，让你的每一次调研都物有所值。
