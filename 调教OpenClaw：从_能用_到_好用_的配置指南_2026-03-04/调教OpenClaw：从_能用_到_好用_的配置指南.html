调教OpenClaw：从"能用"到"好用"的配置指南

前面几篇文章讲了OpenClaw是什么、怎么部署、如何实现邮件自动化。

如果你照着做了，现在应该已经有一个能跑的AI Agent了。

但说实话，很多人装完之后的感觉是：好像...也就那样？

我第一次装完也是这种感觉。让AI写周报，它给我整出个"本周完成了工作，下周继续工作"——废话文学大师。让它处理邮件，回复得倒是快，但一看就是模板，客户直接问"你是真人吗"。

后来我跟几个用OpenClaw的朋友聊，发现大家的困境差不多：

  • • AI回复太泛泛，像复读机
  • • 复杂任务直接罢工，只能做简单问答
  • • 每次都要重复交代背景，它从来不记
  • • 技能之间各干各的，完全没配合

我当时差点就想卸载了。

但转机在第三周。我发现OpenClaw有个配置文件夹，里面一堆yaml文件。抱着"死马当活马医"的心态改了改，结果...效果出奇的好。

今天就把这些折腾出来的配置技巧分享给你。

配置一：提示词模板定制

这个改动最简单，但效果最明显。

OpenClaw默认用的是通用提示词，相当于给每个任务都派了个"临时工"。啥都懂一点，但啥都不精。

想让AI干好活，得给它"岗位说明书"。

我第一次改配置是改周报模板。之前AI给我写的周报是这种：

本周完成了产品功能开发，修复了若干bug，下周计划继续开发新功能。

看完我领导都笑了："这也是周报？"

改了提示词模板之后，同一周的周报变成了这样：

本周核心产出

  1. 完成用户中心模块开发，覆盖登录、注册、个人信息修改
  2. 修复线上bug 3个，其中P0级别1个（支付超时问题）
  3. 参与产品评审会2次，提出技术可行性建议

下周工作计划

  1. 推进订单模块开发，预计周三完成核心接口
  2. 优化数据库查询性能，当前慢查询问题已定位
  3. 跟进支付通道联调测试

领导看完说："不错，继续保持。"

区别在哪？后者有结构、有细节、有数字支撑。看着就专业。

如何配置

在OpenClaw的配置文件中（config/prompts.yaml），可以自定义任务类型的模板。

配置要点：

  • • 设定角色定位（如"资深产品经理"）
  • • 定义输出格式（结构化模板）
  • • 明确执行规则（5-8条核心规则）

实战案例：智能客服

之前给一个朋友的公司配置智能客服，他们卖的是SaaS产品。最开始AI经常承诺"今天就能解决"，结果技术团队背了不少锅。

后来我们加了一个规则模板，比如：

  • • 涉及bug修复时，回复"已记录，预计1-2个工作日处理"
  • • 涉及功能需求时，回复"已反馈给产品团队评估"
  • • 遇到紧急问题（无法登录、数据错误），立即引导转人工

配置之后，客服准确率从60%提升到85%，技术团队的抱怨也少了很多。

配置二：技能链编排

单个技能只能做单一任务，但实际工作往往需要多个步骤。技能链编排就是把多个技能串联起来，形成自动化工作流。

什么是技能链

举个例子，处理客户邮件不是简单回复就完事，而是一系列动作：

  1. 接收邮件
  2. 识别邮件类型（咨询/投诉/售后）
  3. 提取关键信息（订单号、问题描述）
  4. 查询系统状态
  5. 生成回复草稿
  6. 人工审核（可选）

用技能链可以把这些步骤串起来。

技能链的条件分支

最强大的是支持条件判断。比如上面的例子中，只有投诉类邮件才需要人工审核，其他类型可以直接发送。

实战案例：订单处理自动化

我之前给一个电商公司做过订单处理自动化。他们的流程是这样的：

  1. 客户下单
  2. 检查库存
  3. 库存充足 → 锁定库存 + 生成发货单
  4. 库存不足 → 通知供应商 + 标记缺货

配置之后，订单处理时间从平均2小时缩短到5分钟。

配置三：记忆与上下文管理

这个功能我差点错过了，一开始觉得没啥用。

后来有次我让AI帮我处理一个老客户的邮件，结果它完全不知道这人是谁，问了我一堆基本信息。我才发现：AI每次都是"失忆状态"。

OpenClaw的记忆管理就是解决这个问题的。

短期记忆 vs 长期记忆

短期记忆就像对话缓存，记住当前对话在说什么。但下次对话就忘了。

长期记忆才是真正有用的东西，可以跨对话保存信息。

避坑指南

配置过程中我踩过几个坑，说给你听听。

坑一：提示词写太多

刚开始我以为规则写得越全越好，结果整了20多条提示词。AI不但没变聪明，反而"懵"了——回复变得支支吾吾，像个被吓住的新人。

后来才明白：提示词是聚焦核心规则的，不是写用户手册。5-8条就够了，太多反而干扰判断。

坑二：技能链设计太长

有次我设计了一个12步的自动化流程，觉得自己是个天才。结果第一次运行，第3步就卡住了，整个流程直接报废。

教训：技能链别超过5步。太长的话，拆成多个子链，分步执行。

坑三：什么都想记住

记忆功能刚发现的时候，我让AI记住所有对话。一个月后数据库膨胀到好几个G，而且很多无用信息反而干扰了AI的判断。

教训：只记录"有意义"的信息。记得加个定期清理机制。

持续优化

配置不是一劳永逸的，需要根据实际效果持续调整。

优化指标：

  • • 任务完成率
  • • 准确率（特别是分类、判断类任务）
  • • 用户满意度
  • • 处理时间

优化周期：

  • • 第1周：密集观察，每天调整
  • • 第1月：每周优化
  • • 之后：每月复盘

写在最后

OpenClaw的强大之处不在于开箱即用，而在于可定制性。

从"能用"到"好用"，中间隔着的就是这些配置细节。花点时间调教，它真的能变成你的"得力助手"。
