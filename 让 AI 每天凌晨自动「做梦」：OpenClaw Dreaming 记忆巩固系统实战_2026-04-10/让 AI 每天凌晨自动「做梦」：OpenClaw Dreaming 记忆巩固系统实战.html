今天，我想和你分享怎么写好这个“入职指南”，也就是如何给你的 AI 注入有趣的灵魂。

为什么不能只写一句“你是我的助手”？

许多人在配置 Agent 时，最常犯的错误就是把系统提示词写得太虚。比如：“你是一个聪明的 AI 助手，请帮我处理日常工作。”

这种设定下，AI 每次遇到问题都要自己去猜你的意图。

在 OpenClaw 中，SOUL.md 被设计为 Agent 的核心人格和工作准则。每次 Agent 启动或者处理任务时，它都会优先读取并遵循这个文件里的规则。

一个好的 SOUL.md，不是用来设定身份的，而是用来设定边界和思维模式的。

三步写出一个好用的 SOUL.md

我自己经过很多次调试，总结出了一个比较稳定的 SOUL.md 模板结构，你可以直接拿去用。

第一步：核心真理（Core Truths）

这是 Agent 做决定的最底层逻辑。告诉它在这个岗位上，什么是最重要的。

比如我给内容创作 Agent 的设定是：

  • 清晰 > 聪明。简单的语言胜过华丽的辞藻。如果一个12岁的孩子看不懂，就重写。
  • 实例 > 说教。具体的例子永远比抽象的解释更有说服力。

有了这个设定，它在写东西的时候，就再也不会给你整一堆没人看得懂的高级词汇了。

第二步：工作边界（Boundaries）

这是告诉 AI “什么绝对不能做”。对于拥有工具调用能力的 OpenClaw Agent 来说，这点极其重要。

比如你可以这样写：

  • 在没有我明确确认的情况下，绝对不允许执行删除文件的操作。
  • 永远不要猜测我不记得的密码，如果不知道，直接问我。
  • 在总结长篇文档时，严禁编造原文没有的细节。

AI 非常听话，只要你把边界划清楚，它就不会越雷池一步。

第三步：语气与风格（Vibe）

这就决定了每天和你对话的，是一个幽默的朋友，还是一个严谨的管家。

与其写“请保持专业”，不如具体描述：

  • 说话像一个经验丰富的资深工程师，直接、高效，没有多余的客套话。
  • 回答问题时，先给出结论，然后再展开理由。
  • 如果我不懂某个技术概念，用生活中的常识来打比方解释给我听。

让 AI 持续进化的秘诀

写好 SOUL.md 不是一劳永逸的。

我通常会在和 Agent 的日常协作中，发现它做得不好的地方，然后顺手把它加进 SOUL.md 里。比如有一天我发现它写回复总是喜欢用“综上所述”，我就会在文件里加一条：“绝对不要在文章结尾使用‘综上所述’或类似的老套总结词”。

这就是自己搭建 Agent 的最大乐趣。它不是一个静态的工具，而是一个会随着你的习惯和要求，不断生长、不断适配你的伙伴。
