特别好用的Claude code插件playground
ClaudeCode有个官方插件叫playground。

这个插件做的事情很简单：让ClaudeCode生成一个独立的HTML文件，你在浏览器里打开它，可视化地交互操作，操作完了它给你一段prompt，粘贴回ClaudeCode继续干活。

为什么需要这个？因为有些问题用纯文字交互太痛苦了。你让Claude描述一个代码库的架构，它给你写三段话，你看完还是脑子里画不出来。但如果它画一张交互式架构图，你点击每个模块就能看细节、写评论、提修改意见，这感觉完全不一样。

安装就两行命令。在ClaudeCode里输入/pluginmarketplaceupdateclaude-plugins-official，然后/plugininstallplayground@claude-plugins-official。装好直接用。

触发方式是在prompt里加一句"Usetheplaygroundskillto..."，后面跟你想做的事。关键是描述你想要的交互方式，不是描述你想要的结果。

作者分享了五个实际案例。第一个是组件设计调整，生成一个实时预览界面，拖拽调整AskUserQuestion组件的布局。第二个是文案批改，把你的SKILL.MD文件逐行展示，每行旁边有接受、拒绝、评论三个按钮，审阅完一键生成修改指令。第三个是Remotion视频intro微调，动画参数用滑块实时调整，不用改代码再渲染再预览。第四个是架构图可视化，把整个emailagent代码库画成交互式节点图，点击任意节点可以评论提问或要求修改。第五个是游戏数值平衡，用滑块调整英雄各项属性数值，AI根据你的调整给出平衡建议。

核心工作流是一个闭环。ClaudeCode生成HTML，你在浏览器里交互操作，playground输出结果prompt，粘贴回ClaudeCode。文字世界和可视化世界之间无缝切换，各自做各自擅长的事。

作者给的建议是：想一种独特的交互方式，然后让模型去实现它。不要局限于已有的案例，任何需要"看到"或"操作"才能做判断的场景都适合用playground。
