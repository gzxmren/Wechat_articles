OpenClaw 的「中国套件」来了：飞书钉钉企微QQ一锅端

如果你还不知道 OpenClaw，先简单介绍一下。

OpenClaw 是目前 GitHub 上增长最快的开源 AI Agent 项目之一，Star 数已经突破 140k。它的定位是 多渠道 AI 助手网关 —— 一个进程，就能把 AI 能力接入 WhatsApp、Telegram、Slack、Discord 等几十个聊天平台。

但问题来了：OpenClaw 原生支持的都是海外平台，国内的飞书、钉钉、企微、QQ 一个都没有。

今天介绍的 OpenClaw-Docker-CN-IM，就是专门解决这个问题的。

给 OpenClaw 补上「中国拼图」

这个项目在 OpenClaw 的基础上，把 飞书、钉钉、QQ 机器人、企业微信 四大国内主流 IM 插件全部预装到了一个 Docker 镜像里。

你不用自己编译 OpenClaw，不用手动安装插件，不用一个平台一个平台地对接。

你只需要：

填几个环境变量 → docker-compose up -d → 完事。

不用自己去找插件，不用自己写对接代码，不用一个一个平台单独部署。

OpenClaw 的能力 + 国内 IM 生态

因为底层是 OpenClaw，所以它天然继承了 OpenClaw 的全部能力：

多 Agent 路由 —— OpenClaw 支持把不同的聊天渠道、不同的账号路由到不同的 AI Agent，每个 Agent 有独立的工作空间和会话。

AI 后端随便换 —— 支持 OpenAI 协议和 Anthropic 协议。你可以接 GPT-4，也可以接 Claude，甚至任何兼容的第三方 API 都行。

上下文窗口够大 —— 默认 200,000 tokens，长对话、复杂任务都扛得住。

内置代码助手 —— 集成了 OpenCode AI，可以直接在聊天里问代码问题。

浏览器自动化 —— 内置 Playwright，可以跑自动化任务。

中文语音合成 —— 内置中文 TTS，机器人还能"说话"。

数据持久化 —— 配置和工作空间通过 Docker Volume 持久化，容器重启数据还在。

而这个中国 IM 整合版在此基础上，额外把飞书、钉钉、QQ、企微四个插件预装好了，开箱即用。

部署只需三步

第一步，下载配置文件：

wget https://raw.githubusercontent.com/justlovemaki/OpenClaw-Docker-CN-IM/main/docker-compose.yml
wget https://raw.githubusercontent.com/justlovemaki/OpenClaw-Docker-CN-IM/main/.env.example
cp .env.example .env

第二步，编辑 .env 文件，填入你的 AI 模型配置：

最少只需要填 三个参数：

MODEL_ID=gpt-4
BASE_URL=https://api.openai.com/v1
API_KEY=sk-your-api-key

用 Claude 的话，改成这样：

MODEL_ID=claude-sonnet-4-20250514
BASE_URL=https://api.anthropic.com
API_KEY=sk-ant-xxx
API_PROTOCOL=anthropic-messages

小提醒：OpenAI 的 Base URL 要带 /v1，Claude 的不用带。

第三步，启动：

docker-compose up -d

就这么简单。启动后，Gateway 在 18789 端口，Bridge 在 18790 端口。

各平台怎么接？

IM 平台的配置都是 可选的，你用哪个就配哪个。

飞书： 填 FEISHU_APP_ID 和 FEISHU_APP_SECRET

钉钉： 填 DINGTALK_CLIENT_ID、DINGTALK_CLIENT_SECRET、DINGTALK_ROBOT_CODE 等

QQ 机器人： 填 QQBOT_APP_ID 和 QQBOT_CLIENT_SECRET

企业微信： 填 WECOM_TOKEN 和 WECOM_ENCODING_AES_KEY

不需要的平台留空就行，不影响其他平台正常运行。

飞书用户请注意：这个坑 99% 的人都踩过

接飞书的时候，光配置 APP_ID 和 APP_SECRET 是不够的。

你还必须去飞书开放平台，手动完成事件订阅：

  1. 1. 进入 应用管理 → 事件订阅
  2. 2. 添加事件 im.message.receive_v1
  3. 3. 接收方式选择 「使用长连接接收事件」

如果你的机器人能发消息、但收不到消息，大概率就是这一步没做。

几个常见的坑

改了环境变量没生效？

配置文件只在首次启动时生成。如果你改了 .env，需要先删掉 ~/.openclaw/openclaw.json，再重启容器。

报 401 错误？

检查 API_KEY 是不是填错了，以及 Base URL 的格式对不对。

端口被占用？

在 .env 里把 OPENCLAW_GATEWAY_PORT 和 OPENCLAW_BRIDGE_PORT 改成其他端口就行。

哪些场景适合用？

技术团队 —— 在飞书或钉钉群里部署 AI 编程助手，随时提问、Review 代码。

客服场景 —— 通过企业微信接入 AI，7×24 智能应答。

个人玩家 —— QQ 群里挂个 AI 机器人，聊天、翻译、写文案。

多平台统一管理 —— 一套后端同时服务四个平台，统一管控对话和配置。
