🎯 agent-manager 是什么？

agent-manager 是我编写的一个 OpenClaw 自定义 Skill，专门用于快速创建和配置多 Agent 及其飞书机器人绑定。

传统的 Agent 创建方式需要手动：

  • ❌ 编辑复杂的 openclaw.json
  • ❌ 创建工作区目录和配置文件
  • ❌ 在飞书开放平台手动创建应用
  • ❌ 配置飞书 Bot 绑定
  • ❌ 设置 Agent 间通信权限

agent-manager 让你只需描述需求，剩下的全部自动完成！

✨ 核心能力

1. 智能 ID 推断

你只需要说「创建一个销售助手」，技能会自动：

  • 根据中文名推断英文 ID：sales-assistant
  • 无需用户手动记忆命名规范

2. 自动生成完整工作区

创建 Agent 时，技能会自动生成完整的工作区文件：

文件
作用

SOUL.md

USER.md

3. 飞书机器人快捷创建

支持快捷授权流程：

  • 自动生成飞书授权链接
  • 用户扫码即可快速创建机器人
  • 自动获取 AppID 和 AppSecret
  • 自动完成账号绑定

4. 全自动初始化

创建完成后，技能自动：

  • ✅ 初始化工作区状态
  • ✅ 启用 Agent 间通信权限
  • ✅ 清理嵌套的 .git 目录（避免冲突）
  • ✅ 完成飞书账号绑定

🔧 核心脚本详解

1. manage_agent.py — Agent 创建脚本

功能：创建 Agent 并可选绑定飞书机器人

命令格式：

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/manage_agent.py create \
  --name "销售助手" \
  --id "sales-assistant" \
  --description "负责处理销售数据和客户跟进" \
  --app-id "cli_xxx" \
  --app-secret "yyy" \
  --require-pairing

参数说明：

参数
必填
说明
--name
✅
Agent 显示名称（中文）
--id
✅
Agent 唯一标识（英文，推荐用 - 连接）
--description
❌
Agent 功能描述
--app-id
❌
飞书 App ID（绑定时必填）
--app-secret
❌
飞书 App Secret（绑定时必填）
--require-pairing
❌
是否要求配对认证（默认 open）

2. feishu_auth.py — 飞书授权脚本

功能：生成飞书快捷授权链接，并轮询获取 AppID 和 AppSecret

两种模式：

模式 A：生成授权链接

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/feishu_auth.py generate

输出示例：

URL=https://open.feishu.cn/page/openclaw?user_code=XXXXXX

模式 B：轮询获取结果

用户完成授权后，运行：

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/feishu_auth.py poll

输出示例：

✅ Authorization successful!
App ID: cli_xxx
App Secret: yyy

3. verify_bindings.py — 绑定验证脚本

功能：查看当前飞书 Bot 与 Agent 的绑定关系

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/verify_bindings.py

输出示例：

🔍 Current Feishu Bot Bindings:
--------------------------------------------------
Account: sales_assistant_bot
  └─ App ID: cli_xxx
  └─ Agent:  sales-assistant

📖 完整使用流程

场景：创建「销售助手」并绑定飞书机器人

Step 1：用户提出需求

帮我创建一个「销售助手」，负责处理销售数据和客户跟进

Step 2：AI 收集信息并确认

AI 根据需求推断：

  • Agent ID：sales-assistant
  • 工作区：~/.openclaw/workspace-sales-assistant
  • 功能描述：负责处理销售数据和客户跟进
  • 飞书绑定：需要新建机器人

发送确认卡片：

📋 创建计划确认

请确认以下信息：
- Agent 中文名：销售助手
- Agent ID：sales-assistant
- 工作区路径：~/.openclaw/workspace-sales-assistant
- 功能描述：负责处理销售数据和客户跟进
- 飞书机器人：新建
- 机器人权限：全部放开（默认）

确认请回复"确认"或"是"，如有修改请说明。

Step 3：用户确认后，获取飞书授权

AI 运行授权脚本生成链接：

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/feishu_auth.py generate

输出：

URL=https://open.feishu.cn/page/openclaw?user_code=XXXXXX

AI 发送卡片引导用户授权：

🔗 👉 点击这里快捷创建飞书机器人 👈

(授权完成后，请回复"好了"或"已完成")

Step 4：用户完成授权，AI 轮询获取凭证

用户回复「好了」后，AI 运行：

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/feishu_auth.py poll

获取到 cli_xxx 和 yyy。

Step 5：执行创建

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/manage_agent.py create \
  --name "销售助手" \
  --id "sales-assistant" \
  --description "负责处理销售数据和客户跟进" \
  --app-id "cli_xxx" \
  --app-secret "yyy"

Step 6：收尾与重启

  1. 汇报创建结果
  2. 询问是否重启网关
  3. 用户确认后执行重启
✅ Agent 销售助手及其相关配置已经创建完毕！

⚙️ 进阶配置

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/manage_agent.py create \

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/manage_agent.py create \

  --app-secret "existing_yyy"

3. 需要配对认证（pairing 模式）

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/manage_agent.py create \

  --require-pairing

说明：--require-pairing 会将 dmPolicy 设置为 pairing，用户需要先配对才能使用。

🔬 工作原理

🚀 与多 Agent 架构的结合

agent-manager + 多 Bot 架构 = 完美的 Agent 生态系统

你可以通过 agent-manager 快速创建多个专属 Agent，再配合飞书多 Bot 配置，实现真正的「一个实例，多个专属助手」！

快速搭建流程

Step 1：用 agent-manager 批量创建 Agent

帮我创建：
- 新媒体专家（绑定 media_bot）
- 数据分析师（绑定 data_bot）
- 周报助手（绑定 report_bot）

Step 2：验证绑定

python3 ~/.openclaw/workspace/skills/agent-manager/scripts/verify_bindings.py

Step 3：重启网关

openclaw gateway restart

⚠️ 注意事项

路径冲突：创建前请确保 ~/.openclaw/workspace-<id> 不存在，避免覆盖已有数据。

飞书授权：授权链接有时效，请在生成后尽快完成授权。

权限变更：创建后需要重启 Gateway 才能加载新 Agent。
