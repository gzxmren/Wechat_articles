如何让Claude code和Antigravity共享大脑
打破 Agent 的“生殖隔离”：
让 Claude code 和 Antigravity 共享大脑
ADAPTIVE AGENT MCP
我们正在经历一场 AI Agent 的寒武纪大爆发。

[ UNIFIED_MIND ]
在这个技术迭代如原本奔涌的时代，我们每个人都像是在沙滩上捡贝壳的孩子。React 19 发布了，LangChain 又重构了，DeepSeek 提出了新的架构……
信息的碎片散落在公众号、GitHub Issue 和技术文档的角落里。我们焦虑，我们收藏，然后我们遗忘。
更糟糕的是，我们赖以为豪的 AI 助手，它的灵魂被困在了训练数据的截止日期里。
当你满怀期待地让它用 React 19 的新特性 useActionState 写一段代码，它却自信地甩给你一段 React 16 的 useEffect 甚至更古老的 Class Component，那一刻的无力感，大概就是“版本隔离”的厚壁障。
Adaptive Agent MCP 的诞生，不仅仅是为了给 Agent 装一个硬盘，更是为了在飞速流逝的时间里，为你和你的 AI，留住那些真正重要的“认知增量”。
你的 VS Code 里住着 GitHub Copilot，终端里跑着 Claude Code，白嫖着Antigravity、Qwen-Cli、Trae等等。它们都很强大，但它们之间存在着一道看不见的 生殖隔离。
你在 Claude Code 里精心调教的“代码洁癖”，Antigravity 对此一无所知；你在 Cursor 里纠正过的错误，换个 IDE 里的 Agent 依然会重犯。
如果你拥有哪怕三个以上的常用 Agent，你就会痛切地感受到这种 割裂 带来的认知损耗。
这也是 Adaptive Agent MCP 试图解决的核心问题。

# 场景演示：从收集到调用
  
在 Claude Code 中: "记住，我喜欢使用 TypeScript"
          ↓
     Agent 自动调用:
     • append_daily_log() → 记录到日志
     • update_preference() → 更新偏好
     • extract_knowledge() → 抽取知识图谱
          ↓
在 Antigravity 中: "我的编程偏好是什么？"
          ↓
     AI: "你喜欢使用 TypeScript，并且倾向于使用 Zod 进行校验……"
你可以把那些完全免费的 Agent 当作“采集员”，负责收集碎片化的想法；而让那些昂贵的、聪明的 Agent 当作“建筑师”，负责调取这些素材构建大厦。
当然，除了知识图谱等功能，Adaptive Agent MCP 也具备以年-月-周-日为单位的记忆写入/读取系统，通过 Markdown 文件的 YAML 头实现渐进式加载记忆，Agent能够按需精准找到某一天的记忆。同时也具备作用域 Scope 功能，防止 Agent 因为你不同项目的记忆导致思维混乱。

▲ 记忆存储结构

默认存储路径: ~/.adaptive-agent/memory（你也可以手动指定）。所有的 Agent，无论身处哪个终端窗口，它们都将共享这一份“灵魂”。

二、 举重若轻：Markdown 的外壳，数据库的芯
Adaptive Agent MCP 采用了一种 混合存储架构 (Hybrid Architecture)，试图在“可读性”与“高性能”之间寻找完美的平衡。
[ 人看的懂的 (Front) ]
所有数据以 Markdown 和 JSON 格式存储。你可以随时用文本编辑器打开，像写日记一样修改你的偏好。这种掌控感是任何黑盒数据库都无法给予的。
[ 机器跑得快的 (Back) ]
底层利用 SQLite (FTS5 + sqlite-vec) 构建毫秒级高性能索引。引入嵌入式模型 Embedding 和重排序模型 Rerank，大幅提高上下文准确率。
无论是搜“网络库”找到“Axios”，还是二次精排，系统都已为你封装好一切。
