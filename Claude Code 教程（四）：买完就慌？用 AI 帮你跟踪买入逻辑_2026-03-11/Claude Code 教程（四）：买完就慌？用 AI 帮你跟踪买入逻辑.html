Claude Code 教程（四）：买完就慌？用 AI 帮你跟踪买入逻辑


先点关注不迷路

买完之后，才是真正痛苦的开始

上一篇文章，我写了海外电力设备出口订单暴增的研究过程，从电气设备的异常信号一路挖到燃气轮机。

研究的时候很兴奋。逻辑清晰，数据支撑，周期位置完美。买入的那一刻，你觉得自己看透了一切。

买完之后呢？

你的逻辑没忘。你记得很清楚。

但每天都有新信息涌过来。一条行业新闻、一份券商研报、一个同行的观点、一条产业链的边际变化。你不自觉地就开始追踪这些"最新动态"，花大量时间去消化。

追着追着，注意力就被最新的信息占满了。

某家公司发了一条看起来负面的公告，你有点慌。一个同行说"这个周期可能没那么强"，你心里开始打鼓。股价回调了一波，你把这些碎片信息串在一起，觉得"逻辑是不是变了"。

冷静下来想想——这些信息，真的重要吗？

大多数时候，它们就是噪音。你最初买入时的核心逻辑——供给侧 40 年只有三家、服务粘性极强、15 年下行创伤导致扩产谨慎——这些真正关键的变量，根本没变。

但人就是这样。心理学管这个叫"近因效应"，说白了就是我们天然会高估最新的信息，低估那些早已验证但不那么"新鲜"的事实。

反映在投资上就是随波逐流。涨了觉得新信息在验证逻辑，跌了觉得新信息在推翻逻辑。逻辑一直没变，你被噪音牵着走了。

假设追踪干的事情，就是给不同的信息赋予不同的权重。哪些真正影响核心逻辑？哪些只是日常的边际波动？拆清楚了，你就不会被每天的信息流冲得东倒西歪。

今天这篇教程，我讲讲怎么用 AI 搭一个"假设追踪系统"。你的逻辑你记得住，但在每天的信息轰炸里守住那些真正重要的东西，光靠脑子不够。

先给你看看成品。

搭完之后长什么样？

别急着看怎么搭，先看看这个系统跑起来是什么效果。

一个文件，一眼就知道：假设验证到哪了、哪些指标超预期、哪些要盯着、每次操作为什么。

下面讲怎么搭。

什么样的假设，才值得追踪？

不是所有想法都叫假设。

"AI 是未来"、"新能源会爆发"——这些话正确但没用，你没法拿它们做决策。

我说的假设，是那种如果对了就能赚大钱的判断。大胆假设，小心求证。

拿海外电力周期举例。2024 年初我看到电气设备出口暴增的时候，脑子里冒出的假设是：

"全球正在进入一轮 10 年以上的电力基建大周期，燃气轮机是最大的受益方向。"

这个判断很大胆。对了，整个电力设备板块的估值中枢要系统性抬升，相关公司利润翻几倍。错了，可能只是短期出口扰动，你被套在高位。

这种级别的假设，才值得你花精力去追踪。那些"这个季度业绩可能超预期"之类的小假设，不值得——就算对了也就赚个波段。

大假设要拆成小指标

光有一个大判断不够，你得把它拆开，拆到可以验证的颗粒度。

我当时把"全球电力大周期"这个假设拆成了三层：

需求真的在爆发吗？

  • 美国电力需求增速能不能从 -1% 回到 +3%？
  • AI 数据中心的电力需求是 PPT 上的数字，还是真的在落地？

供给侧真的跟不上吗？

  • 三大燃气轮机厂（GE、西门子、三菱）会不会快速扩产？
  • 它们的在手订单到底排到哪一年了？

利润真的能兑现吗？

  • GE 燃气轮机业务利润率能不能从 -5% 回到 +10%？
  • 服务收入占比在提升还是在下降？

每一条都有明确的数字目标。每个季度的财报、每次业绩会的管理层发言、每一条行业数据——拿出来一对，马上知道假设是在被验证还是被证伪。

这才叫追踪。

怎么用 Claude Code 搭这个系统？

不需要你写什么 prompt 模板，直接和 Claude Code 聊就行。

第一步：把你的假设说清楚

打开 Claude Code，像跟同事聊天一样说：

"我最近在研究海外电力周期，核心假设是全球进入了一轮 10 年以上的电力基建大周期，燃气轮机是最大受益方向。帮我把这个假设拆成可追踪的指标体系，建一个追踪文件。"

Claude Code 会问你几个问题——需求侧你关注哪些指标？供给侧呢？盈利怎么看？你回答就行，它会帮你生成一个结构化的 markdown 文件，每个指标都有当前值、目标值、确定性评分。

你觉得哪里不对就说，它会改。来回聊两轮，假设树就建好了。

整个过程五分钟，比你自己在 Excel 里搭表格快得多。

第二步：喂数据给它

假设树建好之后，你平时看到相关的信息——财报数据、业绩会纪要、行业新闻——直接丢给 Claude Code：

"GE Vernova 刚发了 Q4 财报，燃气轮机订单 35GW，利润率 12%。帮我更新追踪文件。"

如果是文字信息、数据点，直接说就行。

但如果是 PDF 研报、财报原文，这里有个坑：Claude Code 直接读 PDF 其实很费劲。 几十页的研报丢进去，它可能读不全、漏信息、甚至卡住。

更好的做法是建一个本地知识库——把研报、财报、纪要统一处理成文本格式存下来，Claude Code 读起来又快又准。关于知识库怎么搭，这个话题够单独写一篇了，下期再聊。

这期先用最简单的方式——把关键数据手动告诉它。反正每次财报季你也会自己读，顺手把核心数字丢给它更新一下，几秒钟的事。

它拿到数据之后，会更新对应的指标，标注是 🟢 支持还是 🔴 反对，然后告诉你确定性评分该怎么调。

关键的地方：它会告诉你为什么调。 比如"GE 订单 35GW，超过你设的 30GW 目标，供给侧也没有大规模扩产的迹象，确定性建议从 70% 调到 78%。"

你同意就让它更新，不同意就告诉它理由，它会根据你的判断修正。

第三步：周度复盘

我一般周末花十分钟，问一句：

"帮我复盘一下这周的假设追踪，有什么变化？"

它会把过去一周所有的证据变化汇总，告诉你：

  • 哪些指标有新数据？
  • 确定性的趋势是向上还是向下？
  • 有没有需要你特别注意的信号？

你做的就是看一眼，想一想，该不该调仓位。

第四步：记录你的决策

每次你做了加减仓的动作，跟 Claude Code 说一声：

"今天加仓了 10%，主要是因为 GE 业绩会上 CEO 确认订单排到 2029 年，供给侧逻辑进一步验证了。"

它会把这条决策记录到追踪文件里，标注日期、仓位变化、触发原因、当时的确定性评分。

半年后你回头看，每一次操作的逻辑链都清清楚楚。交割单只有买卖记录，这个文件有你当时的完整思考过程。

自查清单：你的系统搭对了吗？

搭完之后，对照这个清单检查一下：

假设层

  • [ ] 你的核心假设够大胆吗？（"如果对了赚大钱"的级别）
  • [ ] 假设拆成了可验证的指标？（有明确的数字目标）
  • [ ] 指标控制在 10 个以内？（太多追不过来）

数据层

  • [ ] 关键数据能及时喂给 Claude Code？（手动输入或知识库，下期详聊）

追踪层

  • [ ] 有一个结构化的追踪文件？（参考前面的成品展示）
  • [ ] 每个指标都有当前值、目标值、状态标记？
  • [ ] 证据按时间线记录，标注了 🟢🔴🟡？

决策层

  • [ ] 每次加减仓都记录了原因和当时的确定性？
  • [ ] 有固定的周度复盘节奏？
  • [ ] 复盘时看的是证据变化，不是股价变化？

全部打勾了，你的假设追踪系统就算跑起来了。

最后说两句

上一篇文章我聊过"投资的审美"——好的投资应该像选择题，选对了方向，后面的路就顺了。

但选对了只是第一步。选完之后怎么拿住？什么时候该加？什么信号出来该跑？这些问题，光靠当初的研究结论回答不了，因为世界每天都在变。

假设追踪就是帮你把"选完之后"这段路走踏实。你的审美帮你选对了方向，假设追踪帮你在路上不翻车。

AI 帮你整理证据、标注权重、提醒变化。判断还是你做。

但至少你做判断的时候，手里拿的不是"我觉得"，而是一份清清楚楚的追踪文件。
