🦞 OpenClaw 二月狂飙：从能用变成“真·好用”的7版连更


 

写在前面：各位打工人、码农、AI 折腾党们，如果你最近没关注 OpenClaw，那可能错过了一个亿！这短短两周，OpenClaw 团队（Peter Steinberger和contributors）像天天喝红牛一样，疯狂输出了 7 个版本更新。今天咱们一起来看看这个月 OpenClaw 都搞了哪些大事情！


一、先来个"省流版"：这个月到底更新了啥？

如果把 OpenClaw 比作一辆车，那二月的更新大概是：

  • • 换了个更炫酷的仪表盘（Web UI 大升级）
  • • 引擎换成了 V8（支持了更多 AI 模型：Opus 4.6、Codex、Grok、GLM-5...）
  • • 新增了手机遥控功能（iOS/Android 设备控制）
  • • 装了顶级防盗系统（安全性全面加固）
  • • 雇了个更聪明的副驾驶（Agent 能力大幅提升）
  • • 修好了所有漏油的地方（Bug 修复大集合）

二、重磅功能：你的 AI 助理会"玩手机"了！

2026.2.9 版本搞了个大新闻——iOS Alpha 节点应用来了！

啥意思？简单说就是：你现在可以用 OpenClaw 控制你的 iPhone 了！想象一下：

你："Hey OpenClaw，帮我发条微信给老板，说我今天 sick leave。"

OpenClaw："好的，已经发送。顺便帮你把闹钟关了，再点了个外卖。"

这还不是全部！设备配对 + 手机控制插件让这个能力更上一层楼。现在你可以：

  • • 通过 Telegram /pair 命令配对设备
  • • 远程控制 iOS/Android 设备
  • • 让你的 AI 真的"动手"操作手机

网友辣评："以前 AI 是嘴强王者，现在终于开始动手了！"

六、消息系统大升级：再也不怕丢消息了！

写前交付队列（2.13 版本）

这是 2.13 的重磅修复！新增了一个写前交付队列，带崩溃恢复重试机制。

解决了什么问题？

  • • 以前：网关重启后，正在发送的消息可能会丢失
  • • 现在：消息会先写入队列，即使重启也能恢复发送

运维同学："终于不用担心重启丢消息了！"

自动线程回复（2.13 版本）

线程上下文保持（2.13 版本）

  • • message send 工具动作的 replyTo 和 threadId 会传递到渠道适配器
  • • 线程/回复路由正确保持
  • • 媒体运行保留线程上下文

七、Cron 任务：定时任务终于"定"了

配置安全大升级（2.13 版本）

重要改进：配置文件写入时保留 ${VAR} 环境变量引用！

这意味着什么？

  • • 以前：openclaw config set 会把密钥明文写入文件
  • • 现在：保留 ${SECRET} 形式，不会泄露密钥到磁盘
  • • 额外：修复了并发配置写入的竞态条件

安全圈评价："这波安全加固，我给满分！"

breaking change 提醒（2.12 版本）

Hooks 的 sessionKey 默认不再接受请求覆盖，这是为了防止被恶意利用。需要的话可以在配置里开启：


hooks:

allowRequestSessionKey: true # 需要旧行为就加这个
