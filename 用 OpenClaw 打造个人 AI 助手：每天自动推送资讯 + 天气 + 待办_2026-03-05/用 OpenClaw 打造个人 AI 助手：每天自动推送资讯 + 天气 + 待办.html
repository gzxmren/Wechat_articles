用 OpenClaw 打造个人 AI 助手：每天自动推送资讯 + 天气 + 待办

 

每天早上醒来，你是否也想象过有一个专属 AI 助手，主动帮你整理好今天的天气、新闻和待办事项？这篇文章手把手教你用 OpenClaw 实现。

为什么需要自动推送？

现代人每天被信息淹没：微信群、邮件、新闻APP、天气提醒……光是早上刷手机就要耗掉半小时。

如果有一个 AI 助手，每天定时把「你关心的信息」整理好推送给你——不用翻 Apps，点开即用——是不是会从容很多？

这正是 OpenClaw 的强项：定时执行任务 + 多渠道消息推送 + AI 能力集成。

我要做什么？

今天我们要搭建一个「早报助手」，每天早上 7:30 自动推送：

  1. 1. 今日天气 - 温度、空气质量、穿衣建议
  2. 2. 热门资讯 - 技术/行业热点摘要（2-3条）
  3. 3. 今日待办 - 当天重要的 3 件事提醒

推送渠道你可以选：微信/Telegram/飞书/Slack（任选其一，本文以飞书为例）

准备工作

1. 环境要求

  • • 一台长期运行的电脑/服务器（Mac/Linux/树莓派都可以）
  • • 已部署 OpenClaw（参考之前的部署文章）
  • • 飞书机器人 Webhook 或开通飞书应用

2. 需要的技能

  • • 会用简单的文本编辑器
  • • 能看懂基础的 JSON 配置
  • • 初中级水平即可

第一步：获取天气和资讯

天气怎么获取？

OpenClaw 内置了天气查询能力（基于 wttr.in 或 Open-Meteo），不需要额外 API Key。

在 /Users/admin/.openclaw/workspace 目录下创建配置文件：

mkdir -p tasks/morning-report

创建 tasks/morning-report/weather.yaml：

name: get_weather
description: 获取今日天气

triggers:
  - type: cron
    expression: "0 7 * * *"  # 每天早上7点

actions:
  - plugin: weather
    method: get
    params:
      location: 北京  # 改成你的城市
      format: compact

💡 小技巧：如果你的机器人部署在北京，但想查上海的天气，location 写成「上海」即可。

资讯怎么获取？

创建 tasks/morning-report/news.yaml：

name: get_news
description: 获取技术热点资讯

triggers:
  - type: cron
    expression: "0 7 * * *"

actions:
  - plugin: web_fetch
    params:
      url: https://www.36kr.com/information/tech/
      maxChars: 3000

⚠️ 注意：web_fetch 会抓取目标网页内容，建议选择有 RSS 或开放 API 的资讯源，效果更稳定。如果你有特定网站的 API Key，用起来会更顺手。

第二步：组装早报内容

现在我们需要一个「汇总任务」，把天气和资讯组合成一条完整的早报。

创建 tasks/morning-report/daily.yaml：

name: morning_report
description: 每日早报推送

triggers:
  - type: cron
    expression: "30 7 * * *"  # 7:30执行

# 定义任务流
jobs:
  - name: fetch_weather
    task: ./weather.yaml
    alias: weather

  - name: fetch_news
    task: ./news.yaml
    alias: news

# 组合推送
actions:
  - plugin: feishu
    method: send
    params:
      webhook: https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx
      message:
        msg_type: post
        content:
          zh_cn:
            title: 📰 每日早报
            elements:
              - tag: div
                text:
                  content: "🌤️ 今日天气"
              - tag: div
                text:
                  content: "${weather.output}"
              - tag: hr
              - tag: div
                text:
                  content: "📰 热门资讯"
              - tag: div
                  text:
                    content: "${news.output}"
              - tag: hr
              - tag: div
                text:
                  content: "✅ 今日待办提醒\n1. 查看项目进度\n2. 回复重要邮件\n3. 准备周会材料"

🔑 关键点：

  • • ${weather.output} 和 ${news.output} 是变量占位符，会自动替换为前序任务的输出
  • • feishu 插件替换成你实际使用的渠道（telegram/slack/discord都可以）

第三步：配置待办提醒

待办事项可以有两个来源：

方案 A：手动写入配置

在上面的 daily.yaml 里直接写死：

- tag: div
  text:
    content: |
      ✅ 今日待办
      1. [项目] 完成需求评审
      2. [会议] 下午3点周会
      3. [个人] 健身30分钟

方案 B：读取外部文件

创建 tasks/morning-report/todo.txt：

1. [项目] 完成需求评审
2. [会议] 下午3点周会
3. [个人] 健身30分钟

在 daily.yaml 中读取：

actions:
  - plugin: fs
    method: read
    params:
      file: ./todo.txt
    alias: todo
  
  - plugin: feishu
    # ... 使用 ${todo.content}

这样你每天只需要修改 todo.txt，早报就会自动同步。

第四步：测试与调试

配置写完了，别急着睡觉，先测试一遍：

# 手动触发一次
openclaw run tasks/morning-report/daily.yaml

如果配置没问题，你会立刻收到一条飞书消息。

常见问题排查

问题
可能原因
解决思路
收不到消息
Webhook 地址错误
确认机器人 Webhook 格式正确
天气显示异常
城市名称不支持
尝试用拼音，如 beijing
资讯抓取失败
网站反爬/结构变化
换其他资讯源或用 RSS

进阶优化：让 AI 帮你整理资讯

如果你不想只抓原始内容，可以让 AI 帮忙「精简摘要」。

创建一个 summarize.yaml：

name: summarize_news
description: AI 摘要资讯

actions:
  - plugin: llm
    model: minimax-cn/MiniMax-M2.5
    params:
      prompt: |
        请帮我把以下资讯精简成3句话摘要，每句不超过50字：
        
        ${news.output}
      max_tokens: 300
    alias: summary

然后在 daily.yaml 里引用 ${summary}，早报会更精简易读。

还能做什么？

早报只是一个起点。基于同样的思路，你可以搭建：

  • • 周报汇总：每周五自动收集本周工作内容，生成周报
  • • 股票盯盘：每天收盘前推送持仓股票涨跌
  • • 快递提醒：定时查询快递状态，有更新立刻通知
  • • 生日祝福：朋友生日当天自动发送祝福（联动飞书联系人）

总结

今天我们学到了：

  1. 1. 定时任务：cron 表达式控制执行时间
  2. 2. 任务编排：多个子任务如何组合成一个完整工作流
  3. 3. 多渠道推送：一条配置兼容飞书/微信/Telegram/Slack
  4. 4. 变量引用：任务间如何传递数据

OpenClaw 的设计理念就是「配置即代码」——不需要写复杂的程序，只需要声明式配置，AI 助手就能帮你干活。
